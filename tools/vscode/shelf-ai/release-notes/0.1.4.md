# 0.1.4

## 中文说明

- 本次发布把最后一批项目级知识库聚合依赖从正式主链中移除，运行时入口、运行时摘要、前端/领域 validator、以及知识库前后端辅助逻辑都改为从 package exports 与 runtime projection 派生。
- `scripts/validate_strict_mapping.py` 现在正式提供 `--json` 机器输出，配套报告脚本已同步到当前校验接口，不再依赖过期参数契约。
- 仓库已完成修复后的二次全量审查与二次全量验证，并基于该状态重新打包 Shelf AI VSIX。

对使用者可见的影响：

- Shelf AI 扩展现在与仓库当前的 export-driven runtime 主链保持一致
- 依赖严格映射校验结果的自动化报告链重新可用
- GitHub Release 资产应提供 `shelf-ai-0.1.4.vsix`

安装或获取方式：

1. 下载发布资产 `shelf-ai-0.1.4.vsix`
2. 在 VSCode 中执行 `Extensions: Install from VSIX...`
3. 选择该文件完成安装

注意事项：

- `projects/knowledge_base_basic` 仍只用于本地验证、演示和回归测试，不属于公开发布物
- 仓库仍要求 `projects/*/project.toml` 作为唯一项目入口

## English Notes

- This release removes the last project-wide knowledge-base aggregate assumptions from the formal runtime path. The runtime app, runtime summaries, frontend/domain validators, and knowledge-base frontend/backend helpers now derive from package exports and runtime projection.
- `scripts/validate_strict_mapping.py` now provides a supported `--json` machine output, and the report generators are synced to the current validation contract instead of relying on the stale interface.
- The repository was re-audited and revalidated end-to-end after the fix set, and the Shelf AI VSIX was rebuilt from that validated state.

User-visible impact:

- The Shelf AI extension now stays aligned with the repository's export-driven runtime mainline
- Automation that depends on strict-mapping validation payloads works again with the supported validator contract
- The GitHub Release asset should provide `shelf-ai-0.1.4.vsix`

Install / access:

1. Download the release asset `shelf-ai-0.1.4.vsix`
2. Run `Extensions: Install from VSIX...` in VSCode
3. Select the file to complete installation

Notes:

- `projects/knowledge_base_basic` remains a local validation, demo, and regression sample only; it is not a public release deliverable
- The repository still requires `projects/*/project.toml` to remain the only project entrypoint
