# 置物架框架标准（L2）

## 0. 规范关系
本文件为置物架领域标准，必须遵循：
- `standards/L0/规范总纲与树形结构.md`
- `standards/L1/框架设计核心标准.md`
- `standards/L1/可追溯性标准.md`
- `standards/L1/可删减性标准.md`
本文件位于 `L2`，承接 `L1` 方法标准并向 `L3` 实现层传导。


## 1. 目标（Goal）
定义可拆卸组装的开放式立体堆叠装置，在给定占地面积条件下提升物品存取效率。

## 2. 边界定义（Boundary）
- `N`：层数（`BoundaryDefinition.layers_n`）
- `P`：每层承重（`BoundaryDefinition.payload_p_per_layer`）
- `x`：单层净宽（`BoundaryDefinition.space_s_per_layer.width`）
- `y`：单层净深（`BoundaryDefinition.space_s_per_layer.depth`）
- `h`：单层高度（`BoundaryDefinition.space_s_per_layer.height`），同时作为侧面支撑板高度上限（当前验证场景可取 `h=30`）
- `O`：开口尺寸（`BoundaryDefinition.opening_o`）
- `A`：占地面积（`BoundaryDefinition.footprint_a`）


## 3. 模块（最小可行基，Module）
- `M1` 杆（`Module.ROD`）：结构骨架与承力通道
- `M2` 连接接口（`Module.CONNECTOR`）：定义连接语义、对齐与载荷传递
- `M3` 板（`Module.PANEL`）：提供承载面，并可在规则允许时作为支撑件参与传力


## 4. 组合原则（Combination Principles）
组合对象：
- `ShelfStructure`：单个置物架整体结构，由支撑件（杆或板）、层板、连接接口构成。

统一规则（7条）：
- `R1`：`ShelfStructure` 的支撑结构允许使用 `Module.ROD` 或 `Module.PANEL`，并在实例级明确支撑件类型与支撑角色。
- `R2`：每一层承载板必须按“等效支撑计数”满足 `E_total = 4`：`E(杆)=1`，`E(作为支撑件的板)=2`；单个支撑板必须覆盖同层两个相邻角点，且其高度满足 `0 < h_panel <= h`。每层用于支撑的侧板数量不超过 2；若两块侧板在同层相邻并形成 `90°` 支撑角，其共享角点按一次计入等效支撑（不重复计数），并需由支撑杆补足未覆盖角点。四角形成有效支撑为强制条件，不允许出现缺角支撑。
- `R3`：`ShelfStructure` 必须是立体结构（`span_x > 0`, `span_y > 0`, `span_z > 0`），并形成按高度单调分层的承载体系；杆件长度不设上限，可跨层连续布置。
- `R4`：任意支撑件-层板、支撑件-支撑件连接必须通过 `Module.CONNECTOR` 定义接口实现，包含支撑板与支撑板的 `90°` 连接；`90°` 板-板连接仅表示连接合法，不可替代缺失角点的支撑约束；禁止非法穿透、非法相交、悬空连接。
- `R5`：每层层板轮廓必须与四角支撑件形成闭合支撑框架（四边均有有效连接）。
- `R6`：全结构必须单连通，不允许孤立构件（孤立杆、孤立板、孤立连接件均视为无效）。
- `R7`：所有承载层必须存在连续传力路径，并满足边界中的层数、`x/y/h` 空间量化、开口与占地约束；且每层至少一个侧面保持开放（前/后/左/右任一侧不得被板件完全阻挡）。

组合结果必须可判定为“有效/无效”，并可枚举有效组合集合（参数空间由边界定义给定）。

## 5. 验证（Verification）
通过条件：
- 边界有效
- 组合属于有效组合集
- `target_efficiency > baseline_efficiency`
