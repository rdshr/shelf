# 置物架框架标准（L2）

## 1. 目标（Goal）
定义可拆卸组装的开放式立体堆叠装置，以提升单位占地面积下的存取效率。

### 1.1 单位占地面积下的存取效率定义（目标指标）
统一采用以下计算方式：

- `panel_projection_area_sum = Σ(project_to_ground(panel_i).area)`（全部隔板在地面的投影面积之和）
- `footprint_projection_boundary_area = area(boundary(project_to_ground(whole_shelf)))`（整个置物架在地面投影后的外边界围成面积）
- `target_efficiency = panel_projection_area_sum / footprint_projection_boundary_area`
- `baseline_efficiency = 1.0`（基线参照：物品直接放地面，单位占地面积存取效率定义为 1）

其中：
- `project_to_ground(x)`：将三维对象 `x` 沿竖直方向正交投影到地面（俯视平面）
- `boundary(·)`：投影点集的最外层边界（用于围成占地投影区域）
- 按模块形态定义参与面积计算：
  - `Module.ROD` 为线（宽度为0），自身投影面积记为 0，但其投影线段参与外边界提取
  - `Module.CONNECTOR` 为点（无尺寸），自身投影面积记为 0，但其投影点参与外边界提取
  - `Module.PANEL` 为矩形平面，按 `length * width` 计入投影面积，并参与外边界提取

## 2. 边界定义（Boundary）
- `N`：层数（`BoundaryDefinition.layers_n`）
- `P`：每层承重（`BoundaryDefinition.payload_p_per_layer`）
- `S`：每层空间（`BoundaryDefinition.space_s_per_layer`）
- `O`：开口尺寸（`BoundaryDefinition.opening_o`）
- `A`：占地面积（`BoundaryDefinition.footprint_a`）

## 3. 模块（最小可行基，Module）
- `M1` 杆（`Module.ROD`）：
  - 功能：结构骨架与承力通道
  - 形态：直杆，长度为 `L`、宽度为0、高度为0
- `M2` 连接接口（`Module.CONNECTOR`）：
  - 功能：对接关系，传力规则与装配语义
  - 形态：连接点，长度为0、宽度为0、高度为0
- `M3` 隔板（`Module.PANEL`）：
  - 功能：提供承载面与把载荷分给连接点
  - 形态：矩形平面，长度为 `L`、宽度为 `W`、高度为0

## 4. 组合原则（Combination Principles）
- `R1`：组合中任意两个模块直接至少存在一条可达路径（即从一个模块出发，能到达另一个模块）
- `R2`：组合必须包含连接接口模块
- `R3`：隔板与隔板、隔板与杆、杆与杆之间必须通过连接接口模块连接
- `R4`：每个连接接口模块至少连接两个模块（即每个连接接口模块必须与至少一个隔板或杆相连）
- `R5`：隔板必须水平放置
- `R6`：杆必须垂直放置

## 5. 验证（Verification）
### 5.1 穷举与去重
- 穷举方式必须由组合原则驱动：仅在满足 `## 4. 组合原则` 的前提下扩展结构候选。
- 禁止“先无约束穷举、再用组合原则筛掉”的流程。
- 去重要求：相似结构视为同一结构，只保留一个代表结构。
- 相似结构判定应基于结构同构签名（例如节点/连接关系/面板支撑关系的拓扑签名），不得仅以组合编号区分。

### 5.2 连续三次验证（顺序强制）
按以下顺序执行，前一阶段不通过则停止后续阶段：

1. 组合原则一致性复核  
本阶段仅做一致性复核：候选来自“组合原则驱动穷举”，因此应全部满足 `## 4. 组合原则`（R1-R6）；若出现不满足，视为生成流程错误。

2. 边界验证（仅占地面积）  
仅验证 `footprint_projection_boundary_area` 是否满足边界约束 `A`；本阶段不验证其他边界维度。

3. 目标验证（仅开放式、存取效率）  
仅验证以下两项：
- 开放式：结构存在可直接存取的开放面，不应被全封闭围合。
- 存取效率：`target_efficiency > baseline_efficiency`（即 `target_efficiency > 1.0`）。

### 5.3 通过条件
- 三阶段验证均通过时，结构判定为通过。
