本文是对 `置物架框架化v1.2.md` 的工程化翻译，目标是将自然语言描述重构为可实现、可验证、可追溯的设计输入。

规范与映射来源：
- `standards/L0/规范总纲与树形结构.md`

术语统一（v1.2 -> L2/L3）：
- `M1: 线` -> `Module.ROD`
- `M2: 面` -> `Module.PANEL`
- `M3: 接口` -> `Module.CONNECTOR`

说明：为对齐当前实现，v1.3 使用 `Module.ROD / Module.CONNECTOR / Module.PANEL` 作为唯一规范命名。

## 1. 目标（Goal）[MAP-G]

### 1.1 目标陈述
定义一种可拆卸组装的开放式立体堆叠装置，在给定占地面积下提升物品存取效率。

### 1.2 工程化目标对象
- 数据对象：`Goal.statement`
- 推荐值：`"Increase storage access efficiency per footprint area"`

### 1.3 目标指标
- `baseline_efficiency`：基线存取效率（对照方案）
- `target_efficiency`：目标存取效率（置物架方案）
- 目标通过条件：`target_efficiency > baseline_efficiency`

## 2. 边界定义（Boundary）[MAP-B]

### 2.1 核心参数（与 L3 字段一一对应）

| 边界项 | L3 字段 | 类型 | 单位 | 硬约束 |
| --- | --- | --- | --- | --- |
| 层数 N | `BoundaryDefinition.layers_n` | `int` | 层 | `> 0` |
| 单层承重 P | `BoundaryDefinition.payload_p_per_layer` | `float` | kg | `> 0` |
| 单层空间 S | `BoundaryDefinition.space_s_per_layer` (`Space3D`) | `float,float,float` | cm | `width/depth/height > 0` |
| 开口 O | `BoundaryDefinition.opening_o` (`Opening2D`) | `float,float` | cm | `width/height > 0` |
| 占地 A | `BoundaryDefinition.footprint_a` (`Footprint2D`) | `float,float` | cm | `width/depth > 0` |

### 2.2 从 v1.2 翻译出的约束表达
- 层数范围：`N_min <= layers_n <= N_max`
- 层高范围：`H_min <= space_s_per_layer.height <= H_max`
- 单层面积范围：`A_min <= (space_s_per_layer.width * space_s_per_layer.depth) <= A_max`
- 开口范围：`O_min <= (opening_o.width * opening_o.height) <= O_max`
- 可逆连接约束：装配连接关系必须支持拆卸反向操作。

### 2.3 派生量（用于验证与优化）
- 总可用承载面近似：`layers_n * (space_s_per_layer.width * space_s_per_layer.depth)`
- 开口面积比：`(opening_o.width * opening_o.height) / (footprint_a.width * footprint_a.depth)`

## 3. 模块（最小可行基，Module）[MAP-M]

### 3.1 最小可行模块集

| 模块 | 枚举 | 职责 | 最小几何/连接属性 |
| --- | --- | --- | --- |
| 纵向支撑杆 | `Module.ROD` | 骨架支撑与传力通道 | `length`, `direction_vector`, `head_point` |
| 连接接口 | `Module.CONNECTOR` | 定义连接语义、对齐与载荷传递 | `connector_type`, `supported_joint_types` |
| 横向承载板 | `Module.PANEL` | 提供承载面并向连接点分配载荷 | `width/depth` 或 `polygon_vertices` |

### 3.2 角色映射（对应 L3）
- `MODULE_ROLE[Module.ROD]`: load-bearing support
- `MODULE_ROLE[Module.CONNECTOR]`: joint between structural members
- `MODULE_ROLE[Module.PANEL]`: placement surface

### 3.3 可选扩展属性（非最小基）
- 材质、质量、抗压/抗弯/抗剪指标
- 表面处理、防腐等级、制造公差

## 4. 组合原则（Combination Principles）[MAP-COMBINATION]

### 4.1 核心强制规则（与当前 L3 实现保持一致）
- `R1`: 模块组合不得孤立，`len(combo) >= 2`
- `R2`: 可用组合必须包含连接接口，`Module.CONNECTOR in combo`

### 4.2 工程强制规则（由 v1.2 细化得到）
- `E1`：至少两层平行 `Module.PANEL` 形成分层承载结构。
- `E2`：每层 `Module.PANEL` 至少连接两个支撑单元（`Module.ROD` 或可承载的等效支撑件）。
- `E3`：任意 `Module.ROD` 与 `Module.PANEL` 的连接必须通过 `Module.CONNECTOR`。
- `E4`：全结构必须是单连通图，不允许孤立子结构。
- `E5`：层数、层高、单层面积必须落在边界定义范围内。
- `E6`：禁止非接口定义的穿透、非法相交和悬空连接。

### 4.3 推荐规则（可删减项）
- `S1`：`Module.ROD` 与 `Module.PANEL` 优先保持垂直关系。
- `S2`：每层优先 4 支撑点（常见四角布局）。
- `S3`：`Module.CONNECTOR` 优先落在板角点或预定义安装位。
- `S4`：多层轮廓优先对齐（同形或小偏移）。
- `S5`：开口方向与比例按使用场景偏好配置（前开口/侧开口）。

### 4.4 规则删减评估接口（对应可删减性标准）
每条规则维护：
- `impact_scope`：影响目标/边界/模块列表
- `before_after_metric`：删减前后效率与稳定性对比
- `rollback_plan`：失败回滚路径

## 5. 验证（Verification）[MAP-V]

### 5.1 L3 快速判定（程序化门禁）
输入对象：`VerificationInput`
- `boundary`
- `combo`
- `valid_combinations`
- `baseline_efficiency`
- `target_efficiency`

输出对象：`VerificationResult`
- `boundary_valid`
- `combination_valid`
- `efficiency_improved`
- `passed`
- `reasons`

通过条件（与 `verify()` 对齐）：
- 边界有效
- 组合属于有效组合集合
- `target_efficiency > baseline_efficiency`

### 5.2 工程试验验证（由 v1.2 操作化）
- `V-DETACH` 可拆卸验证：抽样连接点执行拆卸与复装，不损坏且可恢复。
- `V-OPEN` 开放式验证：存取流程不依赖开门/抽屉/翻盖等额外通道使能动作。
- `V-STATIC` 静载验证：在额定承重下静置指定时长，无明显形变或破坏。
- `V-DYNAMIC` 动载验证：施加横向扰动，不发生倒塌或侧翻。
- `V-EFF` 效率验证：同占地面积条件下，对比步骤数与耗时。

### 5.3 失败原因归档（`reasons` 建议编码）
- `BND_*`：边界参数不合法
- `COMBO_*`：组合不在有效集合
- `EFF_*`：效率目标未超过基线
- `MECH_*`：静载/动载失效
- `DETACH_*`：不可逆拆装或复装失败

## 7. 工程落地顺序（建议）
1. 固化 `Goal` 与 `BoundaryDefinition` 参数模板。
2. 先实现 `R1/R2` 的组合门禁，再逐步引入 `E1~E6` 几何与拓扑校验。
3. 将 `V-DETACH/V-OPEN/V-STATIC/V-DYNAMIC/V-EFF` 形成标准测试用例集。
4. 每次规则调整执行删减评估与回滚演练，避免规则膨胀。
