# 置物架框架化

### 目标：

定义一种可拆卸组装的开放式的立体堆叠装置，用于增加单位占地面积的存取物品的效率



### 边界：

装置具有一定承载重量限制

装置每层具有一定高度限制

装置层数需要有范围限制

装置每层需要面积限制

装置对外通道开口面积有范围限制

**理由：**

1.所有的实体装置需要考虑其体积（占地面积＋高度，装置的高度则等于层高*层数）

2.所有的用于存储的装置，一定要考虑其最大承载量。

3.该装置需要增加单位面积存取效率，必定是通过增加表面积的形式，因为纵向堆积物体才能增加堆积效率，而纵向堆积物体的存取需要遵循LIFO原则，增加表面积近似等于增加LIFO的管道，也就是增加存取效率。因此装置必定是分层式装置。

4.可拆卸组装的结构代表一定是由有限个零件组成的。

5.开放式表示装置对外的通道一定是有开口的，不能是封闭的。

6.该装置仅用于存储物品，不需要其他功能，因此无其他限制。


### 最小完备基：

1.M1：线
用于将承载系统架高，为承载系统提供抵抗重力的条件
​属性：长度和方向（以向量的形式体现），头部坐标

2.M2：面
横向承载系统：用于承载物品，用于拦截重力，实现物体在纵向空间上的堆叠，和增加直接可达面积。
属性：顶点数量，每个顶点的端点坐标

3.M3：接口
用于将纵向支撑系统和横向承载系统进行组合的中间件。
属性：接口类型

**理由：**

1.需要实现目标，该装置需要承载物品，因此需要承载系统。

2.该装置为多层结构，层与层之间需要进行隔离（撑高），因此需要支撑系统。

3.承载系统与支撑系统为独立的模块，其连接需要靠链接系统进行链接和固定（接口转换、对齐）

4.最简单的承载系统就是平面板，而想要将平面板支撑起来的最简单的物体就是支撑柱，分别抽象为一个平面和一个线段。

纵向支撑系统通过头部坐标和方向向量来确定尺寸和位置。

### 组合法则：

组合法则用于规定基和基之间的组合方式，最终组合结果满足目标和边界要求
存在强制性法则和非强制性法则
其中强制性法则用于组成目标的基础功能，必须存在，若缺少了则结果不可定义为目标。（用于定义置物架的概念）
非强制性法则用于规定一些可有可无的要求。（用于定义置物架的形态，和其他属性如承重能力等）

**强制性法则：**

最小完备基不能单独使用
M1，M2与其他的M1、M2连接需要使用M3。

有N个互相平行的M2（命名为M_BEARING）
有M个垂直于M2_BEARING的M1或M2（命名为M_SUPPORT）
M3存在于M1和M2的边界上
每个M_BEARING至少与2个M_SUPPORT相连
每个M_SUPPORT在与其相连的M_BEARING的法向量上的延伸长度一致（确保最终M2_BEARING平行与地面平行）
层数N需要在目标定义范围内（N_min <= N <= N_max）
相邻两层M_BEARING之间的层高需要在目标定义范围内（H_min <= H_i <= H_max）
每一层M_BEARING需要满足面积限制（A_min <= A_i <= A_max）
每个M_BEARING的M_SUPPORT连接数量和连接位置模式必须符合目标定义（例如角点连接或角点+中点连接）
所有M_BEARING之间必须通过M_SUPPORT和M3形成单连通结构（不存在孤立子结构）
多层M_BEARING之间需要满足轮廓对应关系约束（投影对齐、偏移量限制或同形约束）


**非强制性法则**
M_SUPPORT仅由M1组成
M_SUPPORT与M_BEARING垂直
每个M_BEARING与4个M_SUPPORT相连
M3存在与M1和M2的顶点上
任意M1、M2之间不允许出现非接口定义的几何穿透、非法相交或悬空连接
开放面方向、开口比例和开口位置满足目标形态偏好（如前开口、侧开口比例范围）

**简化速查版（便于快速判断）**

强制性法则（5条）
1. 至少有2层互相平行的M_BEARING（M2）
2. 每层M_BEARING至少连接2个M_SUPPORT
3. 所有M1与M2连接都必须通过M3
4. 所有层必须连成单一连通结构
5. 层数、层高、单层面积都在目标范围内

非强制性法则（5条）
1. M_SUPPORT优先仅由M1组成，并与M_BEARING垂直
2. 每层优先连接4个M_SUPPORT（常见四角）
3. M3优先设置在M1与M2的顶点处
4. 多层轮廓优先对齐（同形或小偏移）
5. 避免非法穿透/悬空，开口方向与比例按目标形态偏好设置




**理由：**

最小完备基如果单独使用则没有意义，无法实现任何功能。

两个功能模块之间互联一定有两个接口，分别位于两个功能模块

### 如何验证：

首先检查是否满足组合规则。若满足则进行下一步验证。

可拆卸组装功能验证：

拆卸组装功能通过链接系统实现，因此通过反向验证，每个链接节点都可以进行拆卸操作

开放式功能验证：

该装置中进行物品存取时不需要类似“开门”“拉抽屉”“翻盖”等与存取通道使能相关的操作。

立体堆叠功能验证：

按照规定的最大承载重量，和高度限制。将物品放到置物时架上进行静置测试和动态测试。

静置测试：在一定时间段内，装置无明显形变和损坏现象。

动态测试：对装置任意位置施加横向作用力，装置不发生倒塌、侧翻现象。

存取物品的效率验证：

在装置中放入足量的物品。

另取一块相同占地面积的空地，将同样的物品以重叠的方式进行堆叠。

对比在两种方式下，存取同一个物品所需要进行的操作步骤数量和时间。
