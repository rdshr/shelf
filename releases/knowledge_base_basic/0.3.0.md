## 中文说明

本次发布对应仓库完成主路径重构后的 `knowledge_base_basic` 样例。

当前主链是：

`framework markdown -> package registry -> project.toml -> code/runtime -> canonical evidence`

本次变更摘要：

- `projects/knowledge_base_basic/project.toml` 成为唯一项目配置入口
- `canonical_graph.json` 成为唯一机器真相源
- `generation_manifest.json`、`governance_manifest.json`、`governance_tree.json` 与各类报告全部降级为 canonical 派生视图
- 默认运行入口保持为知识库工作台

安装或获取方式：

1. 获取本目录下的 `knowledge-base-basic-0.3.0.tar.gz`
2. 解压后进入目录
3. 执行 `uv sync`
4. 执行 `uv run python src/main.py serve`
5. 浏览器访问 `http://127.0.0.1:8000/knowledge-base`

注意事项：

- 若你修改了框架文档或 `project.toml`，必须重新执行 `uv run python scripts/materialize_project.py`

## English Notes

This release corresponds to the rewritten `knowledge_base_basic` sample after the repository switched to its new mainline architecture.

The active chain is:

`framework markdown -> package registry -> project.toml -> code/runtime -> canonical evidence`

Summary:

- `projects/knowledge_base_basic/project.toml` is now the only project config entrypoint
- `canonical_graph.json` is now the sole machine truth
- `generation_manifest.json`, `governance_manifest.json`, `governance_tree.json`, and reports are all canonical-derived views
- The default runtime entry remains the knowledge-base workbench

Install / access:

1. Download `knowledge-base-basic-0.3.0.tar.gz` from this directory
2. Extract the archive and enter the folder
3. Run `uv sync`
4. Run `uv run python src/main.py serve`
5. Open `http://127.0.0.1:8000/knowledge-base`

Notes:

- If you change framework markdown or `project.toml`, rerun `uv run python scripts/materialize_project.py`
