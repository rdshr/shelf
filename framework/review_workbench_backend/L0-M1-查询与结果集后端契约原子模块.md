# 查询与结果集后端契约原子模块:ReviewWorkbenchBackendQueryAndResultSetServiceContracts

@framework

本模块定义 `review_workbench_backend` 中与“查询结果集”直接相关的最小稳定契约。它回答“当前场景下的范围、搜索词、筛选、分页与排序最少如何整理成查询请求，以及结果列表、空态、分页、查询回显与摘要统计最少如何稳定返回”。

本模块默认建立在 `review_workbench_backend.L0.M0` 之上，不重新发明请求上下文、目标对象、结果状态与回流影响的公共骨架，而是在公共骨架之上继续回答查询类后端契约语义本身。

## 1. 能力声明（Capability Statement）
- `C1` 查询请求整理能力：把当前场景、当前范围、搜索词、固定筛选、分页与排序整理成稳定查询请求。
- `C2` 结果集返回整理能力：把结果列表、结果总数、分页结果与空态整理成稳定结果集返回。
- `C3` 查询上下文回显能力：把当前查询请求中的关键上下文以稳定方式回显到返回中。
- `C4` 结果摘要表达能力：把局部统计、命中概况、结果摘要与必要汇总整理成稳定返回。
- `N1` 非职责声明：不直接定义对象打开、查看内容、动作载荷、范围结构改动、历史差异或具体协议实现。

## 2. 边界定义（Boundary / 参数）
- `QUERY_REQUEST_SCOPE` 查询请求边界：任一查询请求允许随场景实例不同而带有不同范围、搜索词、筛选条件、分页与排序，但必须至少能说明“当前在哪个场景下”“当前在什么范围下”“按什么条件查询”，不允许脱离当前场景与当前范围形成匿名自由查询。
- `RESULT_SET_SCOPE` 结果集返回边界：任一查询返回允许因为场景不同而返回不同类型对象列表，但必须至少能说明结果列表、列表顺序、总量关系或空态信息，不允许只返回无法解释来源的一组对象片段。
- `QUERY_ECHO_SCOPE` 查询回显边界：任一查询返回允许按场景需要回显不同粒度的查询条件，但必须至少回显当前范围、关键筛选或搜索状态中的必要组合，不允许让结果返回与请求上下文脱节。
- `SUMMARY_SCOPE` 摘要统计边界：任一查询返回允许带局部统计、命中摘要、计数概览或分组汇总中的不同组合，但只要当前场景依赖结果概况来辅助判断，就必须至少带一组可解释摘要，不允许只有裸列表没有任何结果概况。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 查询请求基：来源：`C1 + QUERY_REQUEST_SCOPE`。
- `B2` 结果集基：来源：`C2 + RESULT_SET_SCOPE`。
- `B3` 查询回显基：来源：`C3 + QUERY_ECHO_SCOPE`。
- `B4` 摘要统计基：来源：`C4 + SUMMARY_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 查询请求必须附着当前场景与当前范围
  - `R1.1` 参与基：`L0.M0[B1] + B1`。
  - `R1.2` 组合方式：任一查询请求都必须说明自己属于 `review_workbench` 的哪个当前场景、当前附着在哪个范围、并在此基础上附着搜索词、固定筛选、分页或排序；不能把查询组织成脱离场景与范围语义的匿名列表抓取。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`QUERY_REQUEST_SCOPE`。
- `R2` 查询请求必须对应可解释结果集返回
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：任一查询请求都必须能换回一组可解释结果集；结果集中必须能看出当前返回的是哪些对象、这些对象按什么列表关系组织、是完整命中、分页命中还是空结果。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`QUERY_REQUEST_SCOPE/RESULT_SET_SCOPE`。
- `R3` 结果集返回必须回显必要查询上下文
  - `R3.1` 参与基：`B2 + B3`。
  - `R3.2` 组合方式：任一结果集返回都必须至少回显当前范围、关键筛选、搜索状态、分页或排序中的必要组合，使调用方能够解释“为什么看到这组结果”。
  - `R3.3` 输出能力：`C2 + C3`。
  - `R3.4` 边界绑定：`RESULT_SET_SCOPE/QUERY_ECHO_SCOPE`。
- `R4` 结果集返回必须在需要时带出结果摘要
  - `R4.1` 参与基：`B2 + B4`。
  - `R4.2` 组合方式：只要当前场景依赖命中数量、分组统计、处理概况、状态计数或其他结果概览来辅助判断，就必须稳定带出至少一组摘要信息。
  - `R4.3` 输出能力：`C2 + C4`。
  - `R4.4` 边界绑定：`RESULT_SET_SCOPE/SUMMARY_SCOPE`。

## 5. 验证（Verification）
- `V1` 查询请求有效性：任一查询请求都必须能说明当前场景、当前范围以及关键查询条件。
- `V2` 结果集返回有效性：任一查询返回都必须能说明结果列表、空态、总量关系或分页语义中的必要组合。
- `V3` 查询回显有效性：任一查询返回都必须至少回显当前范围与关键查询上下文中的必要组合。
- `V4` 摘要统计有效性：任一依赖结果概况的场景中，查询返回都必须至少带一组可解释摘要。
- `V5` 最小必要性：移除任一 `B*` 后，`V1~V4` 至少一项失效。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
