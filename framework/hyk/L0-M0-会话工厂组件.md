# 会话工厂组件:SessionFactory

@framework

## 1. 能力声明（Capability Statement）

- `C1` 连接目标接入能力：接收合法连接目标并形成可生成会话的输入。
- `C2` 会话源构造能力：根据连接目标、会话源策略和连接池策略构造可复用或可独立使用的会话源。
- `C3` 会话生成能力：从会话源生成可用会话实例。
- `C4` 源释放能力：释放由工厂持有的会话源资源。
- `C5` 非能力项：不负责按句柄追踪已借出会话，不负责 `close(handle)`、`snapshot(handles)` 这类受管语义。

## 2. 边界定义（Boundary / 参数）

- `TG` 连接目标（target）：必须足以唯一定位数据库连接。来源：`C1 + C2`。
- `MD` 会话模式（mode）：只允许 `sync | async`。来源：`C1 + C3`。
- `SP` 会话源策略（source_policy）：只允许 `shared | isolated`。来源：`C2 + C4`。
- `PP` 连接池策略（pool_strategy）：只允许 `pooled | nullpool`。来源：`C2 + C4`。
- `IN` 输入限制（constraint）：禁止把业务对象、业务事务规则和业务初始化逻辑直接放入工厂组件。来源：`C1 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 连接目标单元（自有基）：提供数据库目标和会话模式的统一承载结构。来源：`R5 + C1 + TG + MD + IN`。
- `B2` 会话源单元（自有基）：提供连接资源、会话源、会话源策略和连接池策略的统一承载结构。来源：`R5 + C2 + C4 + TG + MD + SP + PP`。
- `B3` 会话实例单元（自有基）：提供由会话源生成的单个会话实例承载结构。来源：`R5 + C3 + MD + SP + PP`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 连接目标进入组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：所有会话生成请求必须先形成合法连接目标；连接目标不合法时不得进入后续源构造。
  - `R1.3` 输出结构：`TARGET_SPEC`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`TG/MD/IN`。
- `R2` 会话源构造组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：会话源只能由合法连接目标构造；`SP` 负责决定源是否复用，`PP` 负责决定源内部连接如何管理；两条策略轴必须同时可判定，且不得互相替代。
  - `R2.3` 输出结构：`SOURCE_UNIT`。
  - `R2.4` 输出能力：`C2`。
  - `R2.5` 边界绑定：`TG/MD/SP/PP`。
- `R3` 会话生成组合
  - `R3.1` 参与基：`B2 + B3`。
  - `R3.2` 组合方式：会话实例只能从会话源单元生成；不得绕过会话源直接构造会话。
  - `R3.3` 输出结构：`SESSION_INSTANCE`。
  - `R3.4` 输出能力：`C3`。
  - `R3.5` 边界绑定：`MD/SP/PP`。
- `R4` 源释放组合
  - `R4.1` 参与基：`B2`。
  - `R4.2` 组合方式：源释放只能作用于工厂持有的会话源；组件不得假定自己知道已借出会话实例的状态。
  - `R4.3` 输出结构：`SOURCE_RELEASE_RESULT`。
  - `R4.4` 输出能力：`C4`。
  - `R4.5` 边界绑定：`SP/PP`。
- `R5` 完整工厂组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：先形成连接目标，再构造会话源，最后生成会话实例或释放会话源；组件只负责生成和源释放，不追踪借出实例。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R5.4` 边界绑定：`TG/MD/SP/PP/IN`。
- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3`。
  - `R6.2` 组合方式：没有连接目标直接构造会话源、没有会话源直接生成会话、把已借出会话追踪逻辑塞入工厂的组合均无效。
  - `R6.3` 输出能力：`C5`。
  - `R6.4` 边界绑定：`TG/MD/SP/PP/IN`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R5` 输出能力并集必须覆盖 `C1~C4`。
- `V4` 边界符合性：所有有效组合都必须满足连接目标、模式、会话源策略、连接池策略和输入限制边界。
- `V5` 最小必要性：移除任一 `B*` 后，目标接入、会话源构造、会话生成或源释放至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。

## 6. 能力暴露（Capability Exposure）

- `E1` 语义落点：本文件 `1. 能力声明`。
- `E2` 机器落点：组件包中的 `package.py::capabilities`。
- `E3` 接口落点：组件包中的 `api.py`。
- `E4` 运行时落点：组件包中的 `component.py`。
- `E5` 一致性要求：`E1~E4` 必须一一对应；新增能力时四处同时修改。
