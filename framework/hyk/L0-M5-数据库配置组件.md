# 数据库配置组件:DatabaseConfig

@framework

## 1. 能力声明（Capability Statement）

- `C1` 配置承载能力：接收数据库配置输入并形成合法配置快照。
- `C2` 配置解析能力：对外提供可直接消费的数据库连接信息。
- `C3` 配置观察能力：输出当前数据库配置快照。
- `C4` 非能力项：不负责会话创建、不负责 SQL 执行、不负责事务生命周期。

## 2. 边界定义（Boundary / 参数）

- `URL` 连接地址（database_url）：必须是合法且可消费的数据库连接地址。来源：`C1 + C2`。
- `PF` 配置标识（profile_name）：必须是非空配置名。来源：`C1 + C3`。
- `DK` 数据库类型（database_kind）：必须是非空类型标识。来源：`C1 + C3`。
- `IN` 输入限制（constraint）：组件只接收配置数据，不接收业务对象或会话策略。来源：`C1 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 配置单元（自有基）：提供数据库地址、配置标识和数据库类型的统一结构。来源：`R5 + C1 + URL + PF + DK`。
- `B2` 解析单元（自有基）：提供配置合法化和连接地址输出。来源：`R5 + C1 + C2 + URL + IN`。
- `B3` 观察出口（自有基）：提供配置快照的对外暴露。来源：`R5 + C2 + C3 + PF + DK`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 配置进入组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：所有配置输入必须先形成配置单元，再在解析单元中完成合法化；非法地址不得进入后续输出。
  - `R1.3` 输出结构：`CONFIG_SPEC`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`URL/PF/DK/IN`。
- `R2` 配置解析组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：解析单元只能消费合法配置单元，并输出稳定连接地址。
  - `R2.3` 输出结构：`DATABASE_TARGET`。
  - `R2.4` 输出能力：`C2`。
  - `R2.5` 边界绑定：`URL/IN`。
- `R3` 配置观察组合
  - `R3.1` 参与基：`B1 + B3`。
  - `R3.2` 组合方式：观察出口只能暴露当前合法配置快照，不得引入会话状态。
  - `R3.3` 输出结构：`CONFIG_VIEW`。
  - `R3.4` 输出能力：`C3`。
  - `R3.5` 边界绑定：`PF/DK/URL`。
- `R4` 完整数据库配置组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：先形成配置单元，再完成解析，最后暴露配置快照。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`URL/PF/DK/IN`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：没有合法配置直接输出地址、把会话状态混入配置快照、把业务对象放入配置组件的组合均无效。
  - `R5.3` 输出能力：`C4`。
  - `R5.4` 边界绑定：`URL/PF/DK/IN`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R4` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效组合都必须满足数据库地址、配置标识、数据库类型和输入限制边界。
- `V5` 最小必要性：移除任一 `B*` 后，配置承载、配置解析或配置观察至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
- `V7` 能力落点一致性：`C1~C3` 必须同时出现在 `L3 文档能力声明`、`package.py capabilities` 和 `api.py/component.py` 的公开出口中。

## 6. 能力暴露（Capability Exposure）

- `E1` 语义落点：本文件 `1. 能力声明`。
- `E2` 机器落点：组件包中的 `package.py::capabilities`。
- `E3` 接口落点：组件包中的 `api.py`。
- `E4` 运行时落点：组件包中的 `component.py`。
- `E5` 一致性要求：`E1~E4` 必须一一对应；新增能力时四处同时修改。
