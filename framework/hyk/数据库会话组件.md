# 数据库会话管理组件:DatabaseSessionManager

@framework

## 1. 能力声明（Capability Statement）

- `C1` 受管打开能力：通过会话工厂生成会话，并为该次借出建立可追踪句柄。
- `C2` 受管执行能力：只能通过有效句柄在已借出会话上执行语句。
- `C3` 受管关闭能力：按句柄关闭单个会话，或清理全部受管会话和相关会话源。
- `C4` 管理观察能力：输出当前受管句柄和活跃会话源快照。
- `C5` 非能力项：不负责数据库目标解析细节，不直接承担会话生成细节，不负责 ORM 语义。

## 2. 边界定义（Boundary / 参数）

- `SF` 会话工厂来源（reference）：必须绑定 `Ref L3.SessionFactory`。来源：`C1 + C3 + C4`。
- `TG` 连接目标（target）：可显式给出，也可回退到会话工厂默认目标。来源：`C1`。
- `MD` 会话模式（mode）：只允许 `sync | async`。来源：`C1 + C2`。
- `PO` 保留策略（policy）：只允许 `shared | one-shot`。来源：`C1 + C3`。
- `PM` 策略映射（policy_mapping）：管理层必须把 `PO` 映射到会话工厂的源策略，且满足 `shared -> shared`、`one-shot -> isolated`。来源：`C1 + C3`。
- `CO` 关闭责任（owner）：只允许 `component | caller`。来源：`C1 + C3`。
- `HP` 句柄标识（handle）：受管执行、关闭和观察都必须以句柄为定位依据。来源：`C2 + C3 + C4`。
- `AI` 管理组装接口约束（assembly_interface）：所有受管动作都必须通过本层组装接口接入会话工厂和句柄记录，不允许绕过。来源：`C1 + C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话工厂组件基（引用基）：`Ref L3.SessionFactory`
  - 提供连接目标接入、会话源构造、会话生成和源释放能力。
  - 来源：`L2[R4,R5] + C1 + C3 + C4 + SF + TG + MD + PO + PM`。

- `B2` 持有关系单元（自有基）
  - 提供 `handle -> session -> source -> owner` 的统一承载结构。
  - 负责记录已借出会话、来源策略和关闭责任。
  - 来源：`L2[R5] + C1 + C2 + C3 + C4 + HP + PO + CO`。

- `B3` 管理组装接口基（组装接口基）
  - 提供受管打开、执行、关闭、清理和快照的统一管理入口。
  - 负责把 `B1` 的生成能力和 `B2` 的持有关系接成受管语义。
  - 来源：`L2[R4,R5] + C1 + C2 + C3 + C4 + AI + HP + PO + CO`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话工厂规则继承组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：会话管理组件必须继承 `Ref L3.SessionFactory` 的连接目标、会话源和会话生成规则；不得在本层重写工厂语义；当管理层接收 `PO` 时，必须先按 `PM` 映射到工厂源策略。
  - `R1.3` 引用规则：`RefRule L3.SessionFactory.R1~R6`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`SF/TG/MD/PO/PM`。

- `R2` 受管打开组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：会话打开时，必须先通过 `B1` 解析连接目标并生成会话，再由 `B3` 为该次借出建立唯一句柄，并把句柄、会话、会话源和关闭责任写入 `B2`。
  - `R2.3` 输出结构：`MANAGED_HANDLE`。
  - `R2.4` 输出能力：`C1`。
  - `R2.5` 边界绑定：`SF/TG/MD/PO/PM/CO/HP/AI`。

- `R3` 受管执行组合
  - `R3.1` 参与基：`B2 + B3`。
  - `R3.2` 组合方式：语句执行只能作用于 `B2` 中存在的有效句柄；不得绕过句柄直接操作会话实例。
  - `R3.3` 输出结构：`MANAGED_EXECUTION_RESULT`。
  - `R3.4` 输出能力：`C2`。
  - `R3.5` 边界绑定：`HP/AI`。

- `R4` 受管关闭与清理组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：关闭动作必须先按句柄从 `B2` 定位会话；当保留策略为 `one-shot` 时，还必须通过 `B1` 释放对应会话源；整体清理必须关闭全部句柄并释放活跃会话源。
  - `R4.3` 输出结构：`CLOSE_RESULT + CLEANUP_RESULT`。
  - `R4.4` 输出能力：`C3`。
  - `R4.5` 边界绑定：`SF/PO/PM/CO/HP/AI`。

- `R5` 管理观察组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：快照必须同时来源于 `B2` 的句柄记录和 `B1` 的活跃会话源；不得凭空推断未记录状态。
  - `R5.3` 输出结构：`HANDLE_SNAPSHOT + SOURCE_SNAPSHOT`。
  - `R5.4` 输出能力：`C4`。
  - `R5.5` 边界绑定：`SF/HP/AI`。

- `R6` 完整会话管理组合
  - `R6.1` 参与基：`B1 + B2 + B3`。
  - `R6.2` 组合方式：先继承会话工厂规则，再通过 `B3` 完成受管打开、执行、关闭、清理和观察；所有受管动作都必须先经过 `B2` 的句柄关系。
  - `R6.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R6.4` 边界绑定：`SF/TG/MD/PO/PM/CO/HP/AI`。

- `R7` 禁止组合
  - `R7.1` 参与基：`B1 + B2 + B3`。
  - `R7.2` 组合方式：跳过会话工厂直接造会话、没有句柄直接执行、没有句柄直接关闭、存在受管语义却没有持有关系记录、绕过组装接口直接拼接工厂和句柄的组合均无效。
  - `R7.3` 输出能力：`C5`。
  - `R7.4` 边界绑定：`SF/TG/MD/PO/PM/CO/HP/AI`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1/B2/B3` 都必须能由至少一个 `C*` 和一个参数项推导得到。
- `V2` 引用一致性：`B1` 必须明确引用已有组件，且引用规则可追溯到对应 `L3` 文档。
- `V3` 规则一致性：每个 `R*` 必须明确参与基、组合方式、引用规则、输出能力、边界绑定。
- `V4` 目标覆盖性：`R1~R6` 输出能力并集必须覆盖 `C1~C4`。
- `V5` 边界符合性：所有有效组合都必须满足会话工厂引用、连接目标、保留策略、策略映射、关闭责任、句柄标识和管理组装接口约束。
- `V6` 最小必要性：移除 `B1` 后会话生成失效；移除 `B2` 后受管定位失效；移除 `B3` 后管理组合失效。
- `V7` 结论表达：逐条输出 `R* -> C* / 参数边界 / 引用规则` 的通过或失败结论。
- `V8` 引用落点一致性：`Ref/RefRule`、`package.py references`、`ProjectConfig.bindings` 和 `component.py` 中的组装逻辑必须一一对应。
