# 后端项目框架:Backend

@framework

## 1. 能力声明（Capability Statement）

- `C1` 请求接入能力：接收外部请求并进入后端处理上下文。
- `C2` 配置装配能力：按项目配置把基础组件包装配成后端服务。
- `C3` 统一输出能力：把装配结果统一暴露为 HTTP 接口面。
- `C4` 非能力项：不承担基础组件内部语义实现。

## 2. 边界定义（Boundary / 参数）

- `REQ` 请求接口（set）：所有外部能力必须通过 HTTP 接口进入。来源：`C1 + C3`。
- `PC` 项目配置（config）：必须声明要挂载的组件包及其配置；当组件包存在引用契约时，还必须声明 `bindings`。来源：`C2`。
- `RG` 组件注册表（registry）：后端只能从注册表中选择组件包。来源：`C2`。
- `CTX` 应用上下文（context）：装配结果必须进入统一后端上下文。来源：`C1 + C2 + C3`。
- `HC` 接口约束（constraint）：只能暴露已装配组件的接口。来源：`C2 + C3`。
- `OBS` 可观察约束（constraint）：项目必须能输出当前装配的组件、能力和接口挂载结果。来源：`C2 + C3`。
- `BD` 绑定约束（constraint）：组合型组件只能通过绑定复用已挂载组件，且绑定必须满足引用契约。来源：`C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 接入面：L1.M0[R1] + L0.M0[R1]。来源：`C1 + REQ + CTX`。
  - 补充说明：提供后端请求进入点。
- `B2` 装配根：L1.M0[R5] + L0.M7[R6]。来源：`C2 + PC + RG + CTX + HC + BD`。
  - 补充说明：提供组件注册、实例化、绑定解析和挂载。
- `B3` 输出面：L1.M0[R3] + L0.M4[R5]。来源：`C3 + REQ + HC + OBS`。
  - 补充说明：提供统一 HTTP 输出与装配观察。

## 4. 基组合原则（Base Combination Principles）

- `R1` 请求进入组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：所有外部请求必须先进入接入面，再进入后端上下文。
  - `R1.3` 输出结构：`REQ_CTX`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`REQ/CTX`。
- `R2` 项目装配组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：装配根必须根据项目配置从注册表选择组件包；当组件包存在引用契约时，先解析绑定关系，再完成实例化和接口挂载。
  - `R2.3` 输出结构：`SERVICE_GRAPH`。
  - `R2.4` 输出能力：`C2`。
  - `R2.5` 边界绑定：`PC/RG/CTX/HC/BD`。
- `R3` 接口交付组合
  - `R3.1` 参与基：`B2 + B3`。
  - `R3.2` 组合方式：输出面只能暴露装配根已经挂载的接口，并提供当前装配结果的观察接口；观察结果至少包含组件包标识、挂载标识、能力清单、接口前缀和绑定关系。
  - `R3.3` 输出结构：`HTTP_SURFACE + COMPOSITION_VIEW`。
  - `R3.4` 输出能力：`C3`。
  - `R3.5` 边界绑定：`REQ/CTX/HC/OBS`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：路由层直接 new 组件、项目层直接 import 组件内部结构、未在配置中声明的组件进入运行态，均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`PC/RG/CTX/HC`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R3` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效项目都必须满足请求、项目配置、注册表、接口、绑定和可观察约束。
- `V5` 最小必要性：移除任一 `B*` 后，请求接入、配置装配或统一输出至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
- `V7` 装配可见性：项目必须能稳定输出“当前挂载了哪些组件、每个组件声明了哪些能力、这些能力被挂到哪些接口前缀”。
