# 数据库访问组件:DatabaseAccess

@framework

## 1. 能力声明（Capability Statement）

- `C1` 配置化会话打开能力：基于数据库配置组件输出的连接信息打开数据库会话。
- `C2` 配置化执行能力：在已打开会话上执行数据库语句。
- `C3` 配置化关闭能力：关闭由本组件打开的数据库会话。
- `C4` 配置观察能力：输出当前数据库访问所绑定的配置快照。
- `C5` 非能力项：不负责配置来源内部实现，不负责会话内核实现，不负责事务作用域，不负责 ORM 语义。

## 2. 边界定义（Boundary / 参数）

- `DC` 数据库配置来源（reference）：必须绑定 `Ref L3.DatabaseConfig`。来源：`C1 + C4`。
- `DS` 数据库会话来源（reference）：必须绑定 `Ref L3.DatabaseSessionManager`。来源：`C1 + C2 + C3`。
- `MD` 会话模式（mode）：只允许 `sync | async`。来源：`C1`。
- `PO` 保留策略（policy）：只允许 `shared | one-shot`。来源：`C1 + C3`。
- `CO` 关闭责任（owner）：只允许 `component | caller`。来源：`C1 + C3`。
- `AI` 组装接口约束（assembly_interface）：数据库配置输出的连接信息只能通过本层组装接口进入数据库会话，不允许绕过。来源：`C1 + C2 + C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 数据库配置组件基（引用基）：`Ref L3.DatabaseConfig`。来源：`R4 + R5 + C1 + C4 + DC`。
  - 提供数据库配置承载、解析和观察能力。
- `B2` 数据库会话管理组件基（引用基）：`Ref L3.DatabaseSessionManager`。来源：`R4 + R5 + C1 + C2 + C3 + DS + MD + PO + CO`。
  - 提供会话规格、会话建立、会话关闭和执行能力。
- `B3` 数据库访问组装接口基（组装接口基）：提供配置到会话规格的映射接口。来源：`R4 + R5 + C1 + C2 + C3 + C4 + AI`。
  - 负责把 `B1` 的连接信息接入 `B2` 的会话能力。

## 4. 基组合原则（Base Combination Principles）

- `R1` 配置规则继承组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：数据库访问组件必须继承 Ref L3.DatabaseConfig 的配置解析和观察规则；不得在本层重写配置来源语义。
  - `R1.3` 引用规则：RefRule L3.DatabaseConfig.R1~R5。
  - `R1.4` 输出能力：`C1 + C4`。
  - `R1.5` 边界绑定：`DC`。
- `R2` 会话规则继承组合
  - `R2.1` 参与基：`B2`。
  - `R2.2` 组合方式：数据库访问组件必须继承 Ref L3.DatabaseSessionManager 的会话打开、执行、关闭规则；不得在本层重写会话生命周期。
  - `R2.3` 引用规则：RefRule L3.DatabaseSessionManager.R1~R7。
  - `R2.4` 输出能力：`C1 + C2 + C3`。
  - `R2.5` 边界绑定：`DS/MD/PO/CO`。
- `R3` 配置化会话打开组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：会话打开时，必须先从 `B1` 解析配置，再由 `B3` 映射为会话规格，最后交给 `B2` 建立会话。
  - `R3.3` 输出结构：`CONFIGURED_SESSION_HANDLE`。
  - `R3.4` 输出能力：`C1`。
  - `R3.5` 边界绑定：`DC/DS/MD/PO/CO/AI`。
- `R4` 配置化执行与关闭组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：语句执行和会话关闭都只能作用于通过 `R3` 打开的会话；配置快照只能由 `B1` 提供，不得由 `B2` 反向推断。
  - `R4.3` 输出结构：`EXECUTION_RESULT + CLOSE_RESULT + CONFIG_VIEW`。
  - `R4.4` 输出能力：`C2 + C3 + C4`。
  - `R4.5` 边界绑定：`DC/DS/AI/PO/CO`。
- `R5` 完整数据库访问组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：先继承配置规则和会话规则，再通过 `B3` 完成配置到会话的映射；所有会话都必须显式来源于配置组件。
  - `R5.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R5.4` 边界绑定：`DC/DS/MD/PO/CO/AI`。
- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3`。
  - `R6.2` 组合方式：跳过配置组件直接开会话、跳过组装接口直接把配置塞进会话、由配置组件直接执行 SQL 的组合均无效。
  - `R6.3` 输出能力：`C5`。
  - `R6.4` 边界绑定：`DC/DS/MD/PO/CO/AI`。

## 5. 验证（Verification）

- `V1` 推导一致性：`B1/B2/B3` 都必须能由至少一个 `C*` 和一个参数项推导得到。
- `V2` 引用一致性：`B1` 和 `B2` 必须明确引用已有组件，且引用规则可追溯到对应 `L3` 文档。
- `V3` 规则一致性：每个 `R*` 必须明确参与基、组合方式、引用规则、输出能力、边界绑定。
- `V4` 目标覆盖性：`R1~R5` 输出能力并集必须覆盖 `C1~C4`。
- `V5` 边界符合性：所有有效组合都必须满足引用来源、会话模式、保留策略、关闭责任和组装接口约束。
- `V6` 最小必要性：移除 `B1` 后配置化来源失效；移除 `B2` 后会话能力失效；移除 `B3` 后组合失效。
- `V7` 结论表达：逐条输出 `R* -> C* / 参数边界 / 引用规则` 的通过或失败结论。
- `V8` 引用落点一致性：`Ref/RefRule`、`package.py references`、`ProjectConfig.bindings` 和 `assembly.py/component.py` 的组装逻辑必须一一对应。

- `E2` 引用落点：本文件 `3. 最小可行基` 和 `4. 基组合原则` 中的 `Ref L3.* / RefRule L3.*`。
- `E3` 机器落点：组件包中的 `package.py::capabilities` 与 `package.py::references`。
- `E4` 接口落点：组件包中的 `api.py`。
- `E5` 运行时落点：组件包中的 `component.py` 与 `assembly.py`。
- `E6` 一致性要求：`E1~E5` 必须一一对应；新增引用基或新增本层规则时必须同步修改。
