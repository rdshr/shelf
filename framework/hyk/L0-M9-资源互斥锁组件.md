# 资源互斥锁组件:Lock

@framework

## 1. 能力声明（Capability Statement）

- `C1` 资源定位能力：把同一资源请求汇聚到同一把锁上。
- `C2` 独占持有能力：保证同一时刻同一资源只能有一个有效持有者。
- `C3` 正确释放能力：只允许当前这次持有被正确释放。
- `C4` 非能力项：不负责自动过期、续期、等待队列和分布式栅栏等扩展语义。

## 2. 边界定义（Boundary / 参数）

- `K` 资源标识（key）：申请阶段必填。来源：`C1`。
- `O` 持有者标识（owner）：申请阶段必填。来源：`C2`。
- `HT` 持有凭证（token）：每次成功持有生成一次，释放阶段必填。来源：`C2 + C3`。
- `SS` 状态集合（enum）：只允许 `空闲/已持有`。来源：`C1 + C2 + C3`。
- `AT` 切换约束（rule）：占用和释放都必须一次完成。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 资源位置：提供同一资源的唯一锁位置。来源：`R1,R4 + C1 + K + SS`。
- `B2` 持有记录：提供当前持有状态、持有者和持有凭证记录。来源：`R2,R4 + C2 + O + HT + SS`。
- `B3` 原子切换点：提供状态从空闲到持有、从持有到空闲的一次性切换。来源：`R2,R4 + C2 + C3 + AT + HT + SS`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 定位组合
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：所有加锁请求都必须先落到唯一资源位置上，不允许为同一资源创建多个竞争点。
  - `R1.3` 输出结构：`SLOT_SET`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`K/SS`。
- `R2` 占用组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：只有资源位置当前为空闲时，才允许通过原子切换点写入持有记录，并生成新的持有凭证。
  - `R2.3` 输出结构：`HOLD_SET`。
  - `R2.4` 输出能力：`C2`。
  - `R2.5` 边界绑定：`K/O/HT/SS/AT`。
- `R3` 释放组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：释放动作必须携带当前持有凭证，并且只能把当前这次持有原子地切回空闲状态。
  - `R3.3` 输出结构：`RELEASE_SET`。
  - `R3.4` 输出能力：`C3`。
  - `R3.5` 边界绑定：`K/O/HT/SS/AT`。
- `R4` 完整持有组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：先执行 `R1` 定位，再执行 `R2` 占用，最后执行 `R3` 释放；释放不能绕过占用事实。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`K/O/HT/SS/AT`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3`。
  - `R5.2` 组合方式：没有资源位置直接加锁、没有当前持有凭证直接释放、同一资源同时写入多个持有记录、非原子切换的组合均无效。
  - `R5.3` 输出能力：`C4`。
  - `R5.4` 边界绑定：`K/O/HT/SS/AT`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R4` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效组合都必须满足资源标识、持有者、持有凭证、状态和切换约束边界。
- `V5` 最小必要性：移除任一 `B*` 后，定位、独占持有或正确释放至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
- `V7` 能力落点一致性：`C1~C3` 必须同时出现在 `L3 文档能力声明`、`package.py capabilities` 和 `api.py/component.py` 的公开出口中。

## 6. 能力暴露（Capability Exposure）

- `E1` 语义落点：本文件 `1. 能力声明`。
- `E2` 机器落点：组件包中的 `package.py::capabilities`。
- `E3` 接口落点：组件包中的 `api.py`。
- `E4` 运行时落点：组件包中的 `component.py` 公开方法。
- `E5` 一致性要求：`E1~E4` 必须一一对应；新增能力时四处同时增量修改，不允许隐式暴露。
