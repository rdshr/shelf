# 消息流转队列组件:Queue

@framework

## 1. 能力声明（Capability Statement）

- `C1` 消息进入能力：接收消息内容并放入等待流转。
- `C2` 消息领取能力：把等待中的消息交给某一次处理。
- `C3` 消息结束能力：把某一次领取明确结束。
- `C4` 非能力项：不负责分流、延时、重试、死信等扩展语义。

## 2. 边界定义（Boundary / 参数）

- `M` 消息内容（value）：发送阶段必填，且必须可保存。来源：`C1`。
- `ID` 消息编号策略（strategy）：由系统生成且单条唯一。来源：`C1 + C2`。
- `QO` 排队顺序（order）：等待消息必须按单一顺序排列。来源：`C1 + C2`。
- `CT` 领取凭证（token）：每次领取生成一次，结束阶段必填。来源：`C2 + C3`。
- `ST` 状态集合（enum）：只允许 `等待/已领取/已结束`。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 消息单元：提供单条消息的身份和内容边界。来源：`R1,R4 + C1 + M + ID`。
- `B2` 等待区：提供等待消息的顺序承载位置。来源：`R2,R4 + C1 + C2 + QO + ST`。
- `B3` 领取记录：提供某次领取的事实记录和领取凭证。来源：`R2,R4 + C2 + CT + ST`。
- `B4` 结束记录：提供某次领取已经结束的最终记录。来源：`R3,R4 + C3 + CT + ST`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 入队组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：消息先形成消息单元，再按排队顺序进入等待区，并绑定初始状态为等待。
  - `R1.3` 输出结构：`WAIT_SET`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`M/ID/QO/ST`。
- `R2` 领取组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：只能从等待区中领取消息；每次领取都必须生成新的领取凭证，并把状态从等待切换到已领取。
  - `R2.3` 输出结构：`CLAIM_SET`。
  - `R2.4` 输出能力：`C2`。
  - `R2.5` 边界绑定：`ID/QO/CT/ST`。
- `R3` 结束组合
  - `R3.1` 参与基：`B1 + B3 + B4`。
  - `R3.2` 组合方式：结束动作只能作用于已存在的领取记录，并且必须带当前领取凭证，之后把状态写成已结束。
  - `R3.3` 输出结构：`DONE_SET`。
  - `R3.4` 输出能力：`C3`。
  - `R3.5` 边界绑定：`ID/CT/ST`。
- `R4` 完整流转组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：先执行 `R1` 入队，再执行 `R2` 领取，最后执行 `R3` 结束，三步不可颠倒。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`M/ID/QO/CT/ST`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：没有消息单元就入队、没有领取凭证就结束、跳过等待区直接领取、状态越界切换的组合均无效。
  - `R5.3` 输出能力：`C4`。
  - `R5.4` 边界绑定：`M/ID/QO/CT/ST`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R4` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效组合都必须满足消息内容、顺序、领取凭证和状态边界。
- `V5` 最小必要性：移除任一 `B*` 后，入队、领取或结束至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
- `V7` 能力落点一致性：`C1~C3` 必须同时出现在 `L3 文档能力声明`、`package.py capabilities` 和 `api.py/component.py` 的公开出口中。

## 6. 能力暴露（Capability Exposure）

- `E1` 语义落点：本文件 `1. 能力声明`。
- `E2` 机器落点：组件包中的 `package.py::capabilities`。
- `E3` 接口落点：组件包中的 `api.py`。
- `E4` 运行时落点：组件包中的 `component.py` 公开方法。
- `E5` 一致性要求：`E1~E4` 必须一一对应；新增能力时四处同时增量修改，不允许隐式暴露。
