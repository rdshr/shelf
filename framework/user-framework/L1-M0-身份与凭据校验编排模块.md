# 身份与凭据校验编排模块:IdentityCredentialVerificationOrchestration

@framework

本模块把用户定位和凭据证明接成一条身份校验线。它只负责 3 件事：先把用户定位出来、再把凭据校验出来、最后给上层输出可继续登录的身份凭据结果；不负责失败锁定、令牌签发和权限判定。

## 1. 能力声明（Capability Statement）

- `C1` 用户身份定位能力：把登录标识、主用户记录和当前状态接成稳定身份定位段。
- `C2` 凭据校验能力：把密码规则、摘要比对和通过/失败结果接成稳定凭据校验段。
- `C3` 非能力项：不负责失败锁定、令牌签发和角色权限判定。

## 2. 边界定义（Boundary / 参数）

- `USERKEY` 用户标识边界：同一登录标识只能指向一个主用户。来源：`C1 + C2`。
- `PWDRULE` 密码规则边界：密码复杂度和复用窗口必须显式声明。来源：`C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 用户身份定位段基：L0.M0[R1]：把用户主键位、登录标识位和状态位接成可定位主体，能根据登录标识输出唯一且当前可用的用户记录。来源：`C1 + USERKEY`。
- `B2` 凭据校验段基：L0.M0[R3]：把可定位主体、密码规则位和密码证明位接到同一用户记录上，能根据当前输入输出凭据通过或失败结果。来源：`C1 + C2 + USERKEY + PWDRULE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 身份定位成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定一个唯一主体和当前可用状态，形成稳定身份定位段。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`USERKEY`。
- `R2` 凭据校验成立
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在已定位主体上补齐密码规则和摘要比对结果，形成稳定凭据校验段。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`USERKEY/PWDRULE`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：主体未定位、主体不可用或凭据规则缺失时仍继续输出校验结果的组合无效。
  - `R3.3` 输出能力：`C3`。
  - `R3.4` 边界绑定：`USERKEY/PWDRULE`。

## 5. 验证（Verification）

- `V1` 身份定位有效性：用户身份定位段必须满足 `USERKEY` 约束。
- `V2` 凭据校验有效性：凭据校验段必须满足 `PWDRULE` 约束。
- `V3` 最小必要性：移除任一 `B*` 后，`V2` 至少一项失败。
- `V4` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
