# 风险锁定与安全记账编排模块:RiskLockingSecurityLedgerOrchestration

@framework

本模块把失败锁定、关键变更记账和状态汇总接成一条安全状态线。它只负责 4 件事：失败锁定怎么成立、关键变更怎么记账、当前安全状态怎么汇总、上层怎么拿到稳定安全结果；不负责密码校验和角色权限判定。

## 1. 能力声明（Capability Statement）

- `C1` 失败锁定能力：把失败次数、锁定状态和解锁落点接成稳定锁定线。
- `C2` 安全记账能力：把签发、续签、撤销和角色变更接成稳定记账线。
- `C3` 安全状态汇总能力：把失败锁定事实和关键变更记录汇总为稳定安全状态。
- `C4` 非能力项：不负责密码校验和角色权限判定。

## 2. 边界定义（Boundary / 参数）

- `USERKEY` 归属边界：失败锁定事实和安全事件都必须归到一个明确用户。来源：`C1 + C2 + C3`。
- `FAILMAX` 登录失败次数边界：连续失败阈值必须显式声明。来源：`C1 + C3`。
- `LOCKTIME` 锁定时长边界：锁定持续时长和解锁落点必须显式声明。来源：`C1 + C3`。
- `RTOKTTL` 刷新令牌时长边界：续签、轮换和撤销事件必须能回指到刷新令牌寿命事实。来源：`C2 + C3`。
- `ROLESET` 角色集合边界：角色变更事件必须能回指到声明过的角色集合。来源：`C2 + C3`。
- `EVENTTYPE` 事件类型边界：登录失败、锁定、解锁、签发、续签、撤销与角色变更只能落到声明过的事件类型中。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 失败锁定段基：L0.M2[R1]。来源：`C1 + FAILMAX + LOCKTIME`。
  说明：先由失败计数位和锁定状态位收敛出一条失败锁定线。
- `B2` 变更记账段基：L0.M1[R1,R2] + L0.M2[R2]。来源：`C2 + USERKEY + RTOKTTL + ROLESET + EVENTTYPE`。
  说明：令牌事实、角色权限关系和事件记录接上后，才能稳定记下关键安全变更。
- `B3` 安全状态汇总段基：L0.M2[R3]。来源：`C1 + C2 + C3 + USERKEY + FAILMAX + LOCKTIME + EVENTTYPE`。
  说明：失败锁定事实和事件记录汇总后，才有可供上层读取的安全状态。

## 4. 基组合原则（Base Combination Principles）

- `R1` 失败锁定成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定失败阈值、锁定状态和解锁落点，形成稳定失败锁定线。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`FAILMAX/LOCKTIME`。
- `R2` 安全记账成立
  - `R2.1` 参与基：`B2`。
  - `R2.2` 组合方式：把令牌变更事实、角色变更事实和事件记录接到同一用户上，形成稳定记账线。
  - `R2.3` 输出能力：`C2`。
  - `R2.4` 边界绑定：`USERKEY/RTOKTTL/ROLESET/EVENTTYPE`。
- `R3` 安全状态成立
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：把失败锁定线、记账线和安全状态汇总段接成一条完整安全状态线。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`USERKEY/FAILMAX/LOCKTIME/RTOKTTL/ROLESET/EVENTTYPE`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：锁定状态未建立、关键变更未记账或安全状态无法回看时仍继续输出安全结果的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`USERKEY/FAILMAX/LOCKTIME/RTOKTTL/ROLESET/EVENTTYPE`。

## 5. 验证（Verification）

- `V1` 失败锁定有效性：失败锁定段必须满足 `FAILMAX/LOCKTIME` 约束。
- `V2` 安全记账有效性：变更记账段必须满足 `USERKEY/RTOKTTL/ROLESET/EVENTTYPE` 约束。
- `V3` 安全状态汇总有效性：同一用户必须能同时回看失败锁定结果和最近关键变更。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
