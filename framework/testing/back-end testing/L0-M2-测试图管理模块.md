# 测试图管理模块:TestGraphManagement

@framework

## 1. 能力声明（Capability Statement）

- `C1` 测试获取草稿图列表能力：测试系统获取当前处于草稿状态（未正式发布）的所有图列表的能力。
- `C2` 测试获取指定图配置详情能力：测试根据指定的图 ID (graph_id)，获取单个图详细配置数据的能力。
- `C3` 测试更新图配置能力：测试根据指定的图 ID，修改、更新并保存已有图配置信息的能力。
- `C4` 测试获取图的历史发布版本能力：测试根据基础图 ID (base_id)，查询该图所有已经发布的历史版本列表的能力。
- `C5` 测试获取最新版本图配置能力：测试根据基础图 ID，直接快捷拉取该图最新一个已发布版本的详细配置数据的能力。
- `C6` 测试彻底删除图及其所有版本能力：测试根据基础图 ID，将该图及其关联的所有历史版本记录从系统中整体清除的能力。
- `C7` 测试删除特定图版本能力：测试根据基础图 ID 和具体的版本号，仅删除该图某个特定历史版本，而不影响其他版本的能力。
- `C8` 测试创建新图能力：测试在系统中新建并初始化一个空白或初始状态图的能力。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责图的实际运行与调度。

## 2. 边界定义（Boundary / 参数）

- `GETGRAPHLIST` 边界：name (属性: 可选，类型: string | (string | null)，有效范围: 已存在的图名称并支持模糊查询) 
                      limit (属性: 可选，类型: integer，默认值: 10，有效范围:[1,100])
                      page (属性: 可选，类型: integer，默认值: 1，有效范围:≥1) 来源：`C1`。
- `GETGRAPHID` 边界：graph_id (属性: 必选，类型: string, 有效范围: 已存在的 graph_id) 来源：`C2`。
- `PUTGRAPHID` 边界：graph_id (属性: 必选，类型: string, 有效范围: 已存在的 graph_id)
                     name (属性: 必选，类型: string)
                     description (属性: 必选，类型: string)
                     status (属性: 必选，类型: string)
                     config_json (属性: 必选，类型: object) 来源：`C3`。
- `GETGRAPHBASEIDVERSIONS` 边界：base_id (属性: 必选，类型: string, 有效范围: 已存在的 base_id) 来源：`C4`。
- `GETGRAPHBASEIDLATEST` 边界：base_id (属性: 必选，类型: string, 有效范围: 已存在的 base_id) 来源：`C5`。
- `DELETEGRAPHBASEID` 边界：base_id (属性: 必选，类型: string, 有效范围: 已存在的 base_id) 来源：`C6`。
- `DELETEGRAPHBASEIDVERSION` 边界：base_id (属性: 必选，类型: string, 有效范围: 已存在的 base_id)
                                  version (属性: 必选，类型: integer, 有效范围: 已存在的 version) 来源：`C7`。
- `POSTGRAPH` 边界：name (属性: 必选，类型: string)
                    description (属性: 必选，类型: string)
                    status (属性: 必选，类型: string)
                    config_json (属性: 必选，类型: object)
                    base_id (属性: 必选，类型: string) 来源：`C8`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 基础入口组合
  - `R1.1` 参与基：`B1`、`B8`。
  - `R1.2` 组合方式：`B1`（草稿列表）与 `B8`（新建图）可独立调用，作为图管理测试入口；`B8` 产出的标识用于后续依赖链。
  - `R1.3` 输出能力：`C1 + C8`。
  - `R1.4` 边界绑定：`GETGRAPHLIST + POSTGRAPH`。

- `R2` graph_id 依赖组合
  - `R2.1` 参与基：`B1 + B2`、`B8 + B2`、`B8 + B3`、`B1 + B3`。
  - `R2.2` 组合方式：`B2/B3` 必须依赖有效 `graph_id`；`graph_id` 来源限定为同轮 `B8` 返回值或同轮 `B1` 列表选定记录。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`GETGRAPHLIST + GETGRAPHID + PUTGRAPHID + POSTGRAPH`。

- `R3` base_id 版本查询组合
  - `R3.1` 参与基：`B8 + B4`、`B8 + B5`、`B2 + B4`、`B2 + B5`。
  - `R3.2` 组合方式：`B4/B5` 依赖有效 `base_id`；`base_id` 来源为同轮 `B8` 返回值或同轮 `B2` 已确认配置记录。
  - `R3.3` 输出能力：`C4 + C5`。
  - `R3.4` 边界绑定：`POSTGRAPH + GETGRAPHID + GETGRAPHBASEIDVERSIONS + GETGRAPHBASEIDLATEST`。

- `R4` 特定版本删除组合
  - `R4.1` 参与基：`B4 + B7`、`B5 + B7`。
  - `R4.2` 组合方式：`B7` 执行前需先获得 `base_id` 与目标 `version`；`version` 必须来自同轮 `B4` 返回版本集合或可验证的最新版本上下文。
  - `R4.3` 输出能力：`C7`。
  - `R4.4` 边界绑定：`GETGRAPHBASEIDVERSIONS + GETGRAPHBASEIDLATEST + DELETEGRAPHBASEIDVERSION`。

- `R5` 全量版本删除闭环组合
  - `R5.1` 参与基：`B4 + B6`、`B5 + B6`、`B4 + B7 + B6`。
  - `R5.2` 组合方式：`B6` 必须作为该轮删除链最后一步；可先执行 `B7` 删除特定版本，再执行 `B6` 清理剩余版本。
  - `R5.3` 输出能力：`C6 + C7`。
  - `R5.4` 边界绑定：`GETGRAPHBASEIDVERSIONS + GETGRAPHBASEIDLATEST + DELETEGRAPHBASEIDVERSION + DELETEGRAPHBASEID`。

- `R6` 随机有效组合约束
  - `R6.1` 参与基：`{B1, B2, B3, B4, B5, B6, B7, B8}` 的随机子集。
  - `R6.2` 组合方式：合法组合必须满足：
  - `R6.2.a` 含 `B2/B3` 时必须有同轮 `graph_id` 来源。
  - `R6.2.b` 含 `B4/B5/B6/B7` 时必须有同轮 `base_id` 来源。
  - `R6.2.c` 含 `B7` 时必须有可验证 `version` 来源。
  - `R6.2.d` 含 `B6` 时 `B6` 必须最后执行。
  - `R6.3` 输出能力：在约束下实现对 `C1~C8` 的高覆盖随机组合。
  - `R6.4` 边界绑定：`GETGRAPHLIST + GETGRAPHID + PUTGRAPHID + GETGRAPHBASEIDVERSIONS + GETGRAPHBASEIDLATEST + DELETEGRAPHBASEID + DELETEGRAPHBASEIDVERSION + POSTGRAPH`。

- `R7` 无效组合判定
  - `R7.1` 参与基：`B2`、`B3`、`B4`、`B5`、`B6`、`B7` 或任意包含这些依赖基的组合。
  - `R7.2` 组合方式：无 `graph_id/base_id/version` 来源直接调用依赖基、删除顺序违规、跨轮复用失效标识，均判定无效。
  - `R7.3` 失效结论：`N1`。
  - `R7.4` 边界绑定：`GETGRAPHID + PUTGRAPHID + GETGRAPHBASEIDVERSIONS + GETGRAPHBASEIDLATEST + DELETEGRAPHBASEID + DELETEGRAPHBASEIDVERSION + POSTGRAPH + GETGRAPHLIST`。

## 5. 验证（Verification）

- `V1` 规则预校验：通过条件：每轮组合满足 `R1~R6` 且不触发 `R7`；失败原因：依赖链或顺序约束不成立；证据：组合判定日志。

- `V2` 标识来源校验：通过条件：`graph_id/base_id/version` 可回溯到同轮上游基输出；失败原因：标识缺失、来源不明或跨轮串用；证据：标识追踪映射表。

- `V3` 参数边界校验：通过条件：请求参数与请求体字段满足边界；失败原因：字段缺失、类型错误或越界；证据：请求快照与字段校验报告。

- `V4` 调用结果校验：通过条件：有效组合中参与基调用状态码均为 `2000`；失败原因：接口调用失败或返回码异常；证据：响应日志。

- `V5` 删除顺序校验：通过条件：含 `B6/B7` 组合满足 `R4~R6` 顺序约束（`B6` 末位）；失败原因：先全删后查或顺序颠倒；证据：执行步骤序列。

- `V6` 能力导出校验：通过条件：`C1~C8` 均至少由一条合法规则导出；失败原因：存在能力未被任何合法组合覆盖；证据：`R -> C` 导出覆盖报告。

- `V7` 随机覆盖校验：通过条件：随机执行覆盖全部有效组合类型，且 `B1~B8` 至少各执行一次；失败原因：覆盖不全或组合偏斜；证据：组合命中统计。

- `V8` 稳定性校验：通过条件：多轮随机执行中有效组合成功率为 `100%`；失败原因：依赖漂移、顺序漂移或环境污染；证据：轮次成功率与失败归因报告。

- `V9` 结论判定：通过条件：`V1~V8` 全部通过；失败原因：任一验证项失败；证据：汇总判定单（通过/失败、原因、证据路径）。
