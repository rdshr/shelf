# 段落分解模板:ParagraphDecompositionTemplate

@framework

## 1. 能力声明（Capability Statement）

- `C1` 段界观测轴声明能力：提供声明段界观测粒度的能力。
- `C2` 段界匹配模式声明能力：提供声明段界匹配模式与模式值的能力。
- `C3` 分界效果声明能力：提供声明命中边界后分解效果与优先级的能力。
- `C4` 段落分解模板装配能力：提供将观测轴、匹配模式与分界效果装配为完整模板的能力。
- `C5` 模板序列输出能力：提供将多个合法模板组织为稳定模板序列的能力。
- `C6` 非能力项：不负责扫描原文、执行切分、构造段落块或标题判型。

## 2. 边界定义（Boundary / 参数）

- `MATCH_UNIT_SCOPE` 观测轴边界：`match_unit` 必须来自已声明粒度，例如 `line`、`char`。来源：`C1 + C4`。
- `PATTERN_KIND_SCOPE` 模式类型边界：`pattern_kind` 必须来自已声明模式类型，例如 `blank_line`、`regex`、`delimiter`、`indent`。来源：`C2 + C4`。
- `PATTERN_VALUE_SCOPE` 模式值边界：`pattern_value` 必须与 `pattern_kind` 兼容，且必须可校验。来源：`C2 + C4`。
- `EFFECT_SCOPE` 分界效果边界：`effect` 必须来自已声明效果值域，例如 `split`、`split_and_drop`。来源：`C3 + C4`。
- `PRIORITY_SCOPE` 优先级边界：`priority` 必须形成可判定的总顺序。来源：`C3 + C5`。
- `TEMPLATE_FORM` 模板形态边界：每个完整模板必须绑定 `match_unit`、`pattern_kind`、`pattern_value`、`effect`、`priority`。来源：`C4 + C5`。
- `ORDER` 模板顺序边界：模板序列输出必须保持稳定顺序，不得出现不可裁决冲突。来源：`C3 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 段界观测轴：观测轴单元，包含 `MATCH_UNIT`。来源：`C1 + MATCH_UNIT_SCOPE`。
- `B2` 段界匹配模式：模式单元，包含 `PATTERN_KIND`，`PATTERN_VALUE`。来源：`C2 + PATTERN_KIND_SCOPE + PATTERN_VALUE_SCOPE`。
- `B3` 分界效果模板：效果单元，包含 `EFFECT`，`PRIORITY`。来源：`C3 + EFFECT_SCOPE + PRIORITY_SCOPE`。

说明：
- `段落分解模板` 不是基，它是 `B1 + B2 + B3` 的合法组合输出。
- `B1/B2/B3` 是 `B2` 在更低层的三条相对正交结构轴。

## 4. 基组合原则（Base Combination Principles）

- `R1` 观测轴-匹配模式组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：每个 `MATCH_UNIT` 必须与兼容的 `PATTERN_KIND/PATTERN_VALUE` 配对，形成可判定的段界匹配规格。
  - `R1.3` 输出结构：`BoundaryMatchSpec_set`（元素：`BMS`，每个 `BMS` 必须包含 `match_unit`、`pattern_kind`、`pattern_value`）。
  - `R1.4` 边界绑定：`MATCH_UNIT_SCOPE/PATTERN_KIND_SCOPE/PATTERN_VALUE_SCOPE`。
- `R2` 模板装配组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：每个合法 `BoundaryMatchSpec` 必须绑定一个合法 `EFFECT/PRIORITY` 组合，形成一个完整的段落分解模板。
  - `R2.3` 输出结构：`PDT_set`（元素：`PDT`，每个 `PDT` 必须包含 `match_unit`、`pattern_kind`、`pattern_value`、`effect`、`priority`）。
  - `R2.4` 边界绑定：`MATCH_UNIT_SCOPE/PATTERN_KIND_SCOPE/PATTERN_VALUE_SCOPE/EFFECT_SCOPE/PRIORITY_SCOPE/TEMPLATE_FORM`。
- `R3` 模板序列输出组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：全部合法 `PDT` 必须按 `priority` 输出为稳定模板序列；同优先级且不可裁决的模板组合无效。
  - `R3.3` 输出结构：`OrderedTemplate_set`（元素：`OT`，每个 `OT` 必须包含 `match_unit`、`pattern_kind`、`pattern_value`、`effect`、`priority`、`order_index`）。
  - `R3.4` 边界绑定：`PRIORITY_SCOPE/TEMPLATE_FORM/ORDER`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：`match_unit` 与 `pattern_kind` 不兼容、`pattern_value` 不可校验、`effect` 缺失、`priority` 不可排序、或同优先级模板产生不可裁决冲突的组合均无效。
  - `R4.3` 输出结构：`Invalid_set`（元素：`IR`，每个 `IR` 必须明确记录失效基、失效关系与失效原因）。
  - `R4.4` 边界绑定：`MATCH_UNIT_SCOPE/PATTERN_KIND_SCOPE/PATTERN_VALUE_SCOPE/EFFECT_SCOPE/PRIORITY_SCOPE/TEMPLATE_FORM/ORDER`。

## 5. 验证（Verification）

- `V1` 观测轴有效性：`B1` 必须满足 `MATCH_UNIT_SCOPE` 约束。
- `V2` 匹配模式有效性：`B2` 必须满足 `PATTERN_KIND_SCOPE/PATTERN_VALUE_SCOPE` 约束。
- `V3` 分界效果有效性：`B3` 必须满足 `EFFECT_SCOPE/PRIORITY_SCOPE` 约束。
- `V4` 匹配规格合法性：`R1` 产出的 `BoundaryMatchSpec_set` 必须满足观测轴与匹配模式兼容约束。
- `V5` 模板装配合法性：`R2` 产出的 `PDT_set` 必须满足 `TEMPLATE_FORM` 约束，且每个模板都必须有唯一的效果声明。
- `V6` 模板序列稳定性：`R3` 产出的 `OrderedTemplate_set` 必须满足 `ORDER/PRIORITY_SCOPE` 约束，且不得存在不可裁决冲突。
- `V7` 最小必要性：移除任一 `B*` 后，`V4~V6` 至少一项失败。
- `V8` 结论表达：逐条输出 `R* -> 有效/无效 / 边界 / 失效原因` 的结论。
