# 段落分界模板:ParagraphDecompositionTemplate

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文本输入能力：支持输入需要匹配某个目标的字符串。
- `C2` 匹配元字符串输入能力：支持输入匹配元字符串。
- `C3` 匹配规则指定能力：能指定采用某种匹配规则。
- `C4` 匹配结果输出能力：能返回匹配结果，包括成功和失败。

## 2. 边界定义（Boundary / 参数）

- `INPUT` 输入边界：输入必须是字符串。来源：`C1`。
- `STRLEN` 输入字符串长度需要进行限制。来源：`C1`。
- `ELEMENT` 匹配元字符串需要明确。来源：`C2`。
- `MATCHRULE` 匹配规则需要进行明确。来源：`C3`。
- `MATCHRESULT` 匹配结果需要采用统一结构，至少包含匹配状态与命中位置。来源：`C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 输入字符串（需要被扫描的完整字符串载体）。来源：`C1 + INPUT + STRLEN`。
- `B2` 匹配元字符串（用于在输入字符串中查找的目标字符串载体）。来源：`C2 + ELEMENT`。
- `B3` 精确匹配（要求按字符完全一致进行命中的规则单元）。来源：`C3 + MATCHRULE`。
- `B4` 匹配结果记录（统一承载匹配状态、命中位置与失败原因的结果单元）。来源：`C4 + MATCHRESULT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 精确匹配结果组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：每个满足 `STRLEN` 的 `B1` 都必须绑定一个明确的 `B2` 和一个声明为精确匹配的 `B3`；执行时只允许在 `B1` 中按原始字符顺序扫描与 `B2` 等长的连续片段，并以“逐字符完全一致”为唯一命中条件；命中后必须把匹配状态与全部命中位置写入同一个 `B4`，未命中也必须输出统一的未命中结果记录。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R1.4` 输出结构：`ExactMatchResult_set`（元素：`EMR`，每个 `EMR` 必须包含 `input_string`、`element_string`、`match_rule`、`matched`、`hit_span_set`、`result_code`）。
  - `R1.5` 边界绑定：`STRLEN/ELEMENT/MATCHRULE/MATCHRESULT`。
- `R2` 禁止组合
  - `R2.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R2.2` 组合方式：输入字符串缺失或超出 `STRLEN`、匹配元字符串为空或未声明、匹配规则不是精确匹配、以及结果记录缺少统一字段的组合均无效；这类组合不得进入执行链，只能被拒绝并记录失效原因。
  - `R2.3` 输出能力：`C3 + C4`。
  - `R2.4` 输出结构：`InvalidMatch_set`（元素：`IM`，每个 `IM` 必须包含 `invalid_base_id_set`、`invalid_reason`、`rejected_stage`）。
  - `R2.5` 边界绑定：`STRLEN/ELEMENT/MATCHRULE/MATCHRESULT`。


## 5. 验证（Verification）

- `V1` 精确匹配能力有效性：`R1` 必须能够在 `STRLEN/ELEMENT/MATCHRULE` 边界内，对同一组 `B1 + B2 + B3` 稳定地产出同一组 `ExactMatchResult_set`，从而兑现 `C1 + C2 + C3`。
- `V2` 结果记录能力有效性：`R1` 产出的每个 `EMR` 都必须稳定保留统一的 `B4` 结构；若 `matched = true`，则 `hit_span_set` 中的命中区间必须落在 `input_string` 范围内且对应切片与 `element_string` 完全一致；若 `matched = false`，则 `hit_span_set` 必须为空且 `result_code` 必须稳定表达未命中状态，从而兑现 `C4`。
- `V3` 禁止组合检出有效性：`R2` 必须能够在 `STRLEN/ELEMENT/MATCHRULE/MATCHRESULT` 边界内，稳定识别输入缺失、目标为空、规则非法或结果记录不完整等无效组合，并输出带明确失效原因的 `InvalidMatch_set`。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失败。
- `V5` 结论表达：验证结果必须逐条输出 `R* -> 通过/失败 / 边界 / 原因` 的结论，而不是只给出汇总布尔值。
