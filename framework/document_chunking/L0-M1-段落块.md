# 段落块:ParagraphBlock

@framework

## 1. 能力声明（Capability Statement）

- `C1` 段落可排序性能力：支持段落块显示排序。
- `C2` 文本承载能力。
- `C3` 原文中位置追溯能力。
- `C4` 归属文档表达能力。
- `C5` 结尾段落身份标识能力。

## 2. 边界定义（Boundary / 参数）

- `INDEX` 顺序边界：段落块中用来表示顺序的变量需明确。来源：`C1`。
- `LENGTH` 长度边界：文本长度需要进行限制。来源：`C2`。
- `TRACE` 空间边界：块的起止位置不能小于0，且起始位置要小于结束位置。来源：`C3`。
- `BELONG` 归属边界：块必须归属某个具体的原文本。来源：`C4`。
- `ENDING` 结束边界：块必须标明是否属于文档的最后一段文本。来源：`C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 顺序：段落块内部的稳定顺序索引。来源：`C1 + INDEX`。
- `B2` 内容：段落块承载的文本内容。来源：`C2 + LENGTH`。
- `B3` 起始位置：原文本中该段落块的开始位置索引。来源：`C3 + TRACE`。
- `B4` 结束位置：原文本中该段落块的结束位置索引。来源：`C3 + TRACE`。
- `B5` 原文归属：段落块对应的原文本 ID。来源：`C4 + BELONG`。
- `B6` 文档结束标识：布尔值，结尾段落身份标识，TRUE为结束，FLASE为未结束。来源：`C5 + ENDING`

## 4. 基组合原则（Base Combination Principles）

- `R1` 顺序内容组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：每段合法文本内容都必须绑定一个稳定顺序索引，形成可排序的文本段单元；缺失顺序索引或内容超出长度边界的组合不构成有效段落块候选。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`INDEX/LENGTH`。
- `R2` 位置归属组合
  - `R2.1` 参与基：`B1 + B3 + B4 + B5`。
  - `R2.2` 组合方式：顺序索引必须与一组合法的起止位置和唯一原文归属同时绑定，形成可回指到单一原文本的段落定位单元；起止位置非法、归属缺失、同一原文内顺序冲突或同一定位单元同时绑定多个原文的组合均无效。
  - `R2.3` 输出能力：`C1 + C3 + C4`。
  - `R2.4` 边界绑定：`INDEX/TRACE/BELONG`。
- `R3` 完整段落块闭合组合
  - `R3.1` 参与基：`B1 + B2 + B3 + B4 + B5 + B6`。
  - `R3.2` 组合方式：只有当顺序、内容、起始位置、结束位置、原文归属与文档结束标识在同一对象上同时闭合时，才构成合法段落块；内容为空、长度越界、位置反转、边界不连续或归属不明的组合均不得视为有效段落块。
  - `R3.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R3.4` 边界绑定：`INDEX/LENGTH/TRACE/BELONG/ENDING`。


## 5. 验证（Verification）

- `V1` 顺序与内容承载能力有效性：`R1` 必须能够在 `INDEX/LENGTH` 边界内，对同一组输入稳定地产出同一组可排序的段落内容单元，保证内容非空、长度合法且顺序索引可稳定进入后续链路，从而兑现 `C1 + C2`。
- `V2` 位置与归属回连能力有效性：`R2` 必须能够在 `INDEX/TRACE/BELONG` 边界内，稳定把顺序索引绑定到合法起止位置与唯一原文归属，保证 `start_offset >= 0`、`end_offset > start_offset`、`document_id` 非空，并能直接回连到单一原文本，从而兑现 `C3 + C4`。
- `V3` 完整段落块闭合能力有效性：`R3` 必须能够在 `INDEX/LENGTH/TRACE/BELONG/ENDING` 边界内，稳定产出完整闭合的段落块对象；每个合法段落块都必须同时保留顺序、内容、位置、归属和结束标识，且非法组合不得闭合为合法段落块，从而兑现 `C1 + C2 + C3 + C4 + C5`。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失败。
- `V5` 结论表达：验证结果必须逐条输出 `R1/R2/R3 -> 通过/失败 / 边界 / 原因` 的结论，而不是只给出汇总布尔值。
