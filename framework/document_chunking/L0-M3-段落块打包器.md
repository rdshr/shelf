# 中文模块名:EnglishName

@framework

## 1. 能力声明（Capability Statement）

- `C1` 对象输入能力：接受带编号、角色、字符串的对象输入。
- `C2` 文本块打包能力：支持将输入对象打包为一个完整的块。
- `C3` 文本块编号能力：将打包的文本块按顺序进行编号。
- `C4` 文本块输出能力：支持输出文本块。
- `C5` 输出条件判定能力：支持判断文本块输出条件。

## 2. 边界定义（Boundary / 参数）

- `PARAMETER` 对象参数边界：对象的参数至少包含编号、角色、字符串。来源：`C1`。
- `STRLEN` 长度边界：对象的字符串长度需要进行限制。来源：`C1`。
- `CONTENT` 内容边界：文本块除了输入的对象，还需要有自己的编号。来源：`C2 + C3`。
- `ORDER` 顺序边界：文本块编号严格按照打包顺序进行排序。来源：`C3`
- `PACKAGING` 打包条件边界：文本块打包条件必须明确。来源：`C2`。
- `CONDITION` 输出条件边界：文本块输出条件必须明确。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 输入对象：待打包的对象。来源：`C1 + PARAMETER`。
- `B2` 文本块：输入对象集合，和自身编号。来源：`C2 + STRLEN + CONTENT + ORDER`。
- `B3` 打包条件：新输入的对象是正文，则打包入当前的文本块。来源：`C3 + PACKAGING`。
- `B4` 输出条件：新输入的对象角色是标题，或文档结束标识为真，则将当前文本块输出，新输入的对象作为新的文本块。来源：`C4 + C5 + CONDITION`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 段落块打包组合
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：输入对象必须先满足 `PARAMETER/STRLEN` 约束并按原始顺序进入打包流程；当前文本块以最近出现的标题对象作为起点，后续连续正文对象在满足 `B3` 的打包条件时并入同一 `B2`；当新输入对象的角色为标题，或文档结束标识为真并满足 `B4` 的输出条件时，当前文本块必须立即闭合输出，并由新的标题对象开启下一个文本块或结束整个打包流程；每个输入对象只能进入一个文本块，文本块编号必须按闭合输出顺序稳定递增。
  - `R1.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R1.4` 输出结构：`PackedTextBlock_set`（元素：`PTB`，每个 `PTB` 必须包含 `text_block_id`、`text_block_order`、`title_object_id`、`body_object_id_set`、`block_content`）。
  - `R1.5` 边界绑定：`PARAMETER/STRLEN/CONTENT/ORDER/PACKAGING/CONDITION`。

## 5. 验证（Verification）

- `V1` 输入合法性：进入 `R1` 的每个 `B1` 都必须至少包含编号、角色、字符串三个参数，且字符串长度必须满足 `STRLEN` 约束。
- `V2` 打包条件合法性：当新输入对象角色为正文时，必须满足 `B3` 的打包条件并被并入当前文本块；正文对象不得跳过当前文本块进入后续块，也不得脱离任何文本块独立存在。
- `V3` 输出条件合法性：当新输入对象角色为标题，或文档结束标识为真时，系统必须依据 `B4` 闭合并输出当前文本块；未闭合的文本块不得提前输出，已闭合文本块不得继续吸收新的输入对象。
- `V4` 编号与内容稳定性：`R1` 产出的每个 `PTB` 都必须按输出顺序稳定编号，且 `block_content` 必须严格由 `title_object_id` 与 `body_object_id_set` 对应对象的字符串按原始顺序拼接得到。
- `V5` 最小必要性：移除 `B1` 后无法提供待打包对象，移除 `B2` 后无法承载文本块，移除 `B3` 或 `B4` 后无法判定并入或输出时机；任一 `B*` 缺失时，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R1 -> 通过/失败 / 边界 / 原因` 的结论。
