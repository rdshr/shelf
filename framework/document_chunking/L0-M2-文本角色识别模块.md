# 文本角色识别模块:RoleIdentification

@framework

## 1. 能力声明（Capability Statement）

- `C1` 字符串输入能力：支持输入一段待识别的字符串。
- `C2` 角色识别能力：判断输入的文档属于预设角色中的哪一个。
- `C3` 结果返回能力：返回识别结果。

## 2. 边界定义（Boundary / 参数）

- `INPUT` 输入边界：输入必须是字符串。来源：`C1`。
- `STRLEN` 长度边界：输入字符串长度需要进行限制。来源：`C1`。
- `ROLE` 角色边界：可选角色必须有限制。来源：`C2`。
- `OUTPUT` 输出边界：有且只有一个输出结果，且属于预设角色之一。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 输入字符串：需要被识别的完整字符串载体。来源：`C1 + INPUT + STRLEN`。
- `B2` 关键字映射表：根据首字母是不是'#'来映射这段字符串是标题或正文。来源：`C2 + C3 + ROLE + OUTPUT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 文本角色识别组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：每个满足 `INPUT/STRLEN` 约束的 `B1` 都必须绑定同一份合法的 `B2`；执行识别时，先读取 `B1` 的首字符，再严格按 `B2` 的映射规则判定角色：首字符为 `#` 时输出 title，否则输出 body；每次识别只能产生一个且仅一个角色结果，且结果必须落在 `ROLE` 约束限定的预设角色集合内。
  - `R1.3` 输出能力：`C1 + C2 + C3`。
  - `R1.4` 输出结构：`RoleIdentificationResult`（必须包含 `input_text`、`identified_role`、`mapping_rule`）。
  - `R1.5` 边界绑定：`INPUT/STRLEN/ROLE/OUTPUT`。

## 5. 验证（Verification）

- `V1` 输入合法性：进入 `R1` 的 `B1` 必须是满足 `INPUT` 约束的字符串，且长度必须落在 `STRLEN` 允许范围内。
- `V2` 映射合法性：`B2` 必须稳定声明“首字符为 `#` 映射为 title，否则映射为 body”这一规则，且 title/body 必须属于 `ROLE` 限定的预设角色集合。
- `V3` 结果唯一性：`R1` 产出的每个 `RoleIdentificationResult` 都只能包含一个 `identified_role`，不得同时返回多个角色或空结果。
- `V4` 识别一致性：当 `input_text` 的首字符为 `#` 时，`identified_role` 必须为 title；当首字符不是 `#` 时，`identified_role` 必须为 body；`mapping_rule` 必须能回溯到本模块声明的关键字映射规则。
- `V5` 最小必要性：移除 `B1` 后无法提供待识别字符串，移除 `B2` 后无法完成角色判定；任一 `B*` 缺失时，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R1 -> 通过/失败 / 边界 / 原因` 的结论。
