# 文档分块:DocumentChunking

@framework

## 1. 能力声明（Capability Statement）

- `C1` 段界识别能力：提供依据显式模板识别纯文本文档段落边界的能力。
- `C2` 段落块构造能力：提供将原文切分为有序、可回溯段落块序列的能力。
- `C3` 标题判型能力：提供依据显式模板判定段落标题角色的能力。
- `C4` 文本块归属记录构造能力：提供依据标题与顺序范围生成文本块归属记录的能力。
- `C5` 表结构导出能力：提供将中间结果导出为稳定表结构的能力。
- `C6` 节文本块输出能力：提供将标题与其归属正文组合为稳定文本块的能力。
- `C7` 非能力项：不负责跨文档聚类、摘要改写、向量化索引、版面渲染或下游检索排序。

## 2. 边界定义（Boundary / 参数）

- `NORM_DOC` 归一文本边界：输入必须为单文档纯文本，且已完成换行、空白与编码归一。来源：`C1 + C2`。
- `PARAGRAPH_TEMPLATE` 分段模板边界：段落切分依据必须以显式模板结构给出，且至少包含 `match_unit`、`pattern_kind`、`pattern_value`、`effect`、`priority`。来源：`C1 + C2`。
- `TITLE_TEMPLATE` 标题模板边界：标题判型依据必须以显式模板结构给出，且至少包含 `feature_kind`、`feature_value`、`role_label`、`priority`。来源：`C3 + C4`。
- `PARAGRAPH_SPAN` 段落回溯边界：每个段落块必须保留原文起止区间或等价行号区间。来源：`C2 + C6`。
- `PARAGRAPH_ROLE` 段落角色边界：经标题判型后，每个段落块必须可判定为 `title`、`body` 或其它允许角色之一。来源：`C3 + C4`。
- `ATTACH_SCOPE` 归属范围边界：标题仅可归属其后的连续正文段落；遇到下一个标题或文档结束时必须终止。来源：`C4 + C6`。
- `ORDER` 原文顺序边界：所有段落块、角色标记、归属记录、输出表与最终文本块都必须保持原文顺序，不得重排。来源：`C2 + C3 + C4 + C5 + C6`。
- `TABLE_FORM` 表结构边界：模块输出的表结构只能显式表达 `段落块表`、`文本块成员关系表` 这两类结果。来源：`C5`。
- `CHUNK_FORM` 节块输出边界：每个输出文本块必须绑定 `chunk_id`、`title_block_id`、`body_block_id_set`、`text`、`span`、`termination_reason` 与 `order_range`。来源：`C6`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 段落块：最小文本单元，包含 `ID`，`TEXT`，`START`，`END`。来源：`C2 + PARAGRAPH_SPAN + ORDER`。
- `B2` 段落分界模板：分界模板单元，包含 `MATCH_UNIT`，`PATTERN_KIND`，`PATTERN_VALUE`，`EFFECT`，`PRIORITY`。来源：`C1 + PARAGRAPH_TEMPLATE`。
- `B3` 标题判型模板：判型模板单元，包含 `FEATURE_KIND`，`FEATURE_VALUE`，`ROLE_LABEL`，`PRIORITY`。来源：`C3 + TITLE_TEMPLATE + PARAGRAPH_ROLE`。

说明：
- `段落块表`、`文本块成员关系表` 都不是基，它们是组合原则的输出结构。
- `ChunkBlock` 也不是基，它仅作为合法组合后的最终输出结构。

## 4. 基组合原则（Base Combination Principles）

- `R1` 段界切分组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：`B2` 只能作用于满足 `NORM_DOC` 的原文；模板按 `priority` 顺序判定段落分界，并生成按原文顺序排列且区间不重叠的 `B1` 序列。
  - `R1.3` 输出结构：`PB_set`（元素：`PB`，每个 `PB` 必须包含 `block_id`、`text`、`span`、`order_index`）；其表结构化结果为 `段落块表`，列为 `BLOCK_ID`，`TEXT`，`START`，`END`，`ORDER_INDEX`。
  - `R1.4` 边界绑定：`NORM_DOC/PARAGRAPH_TEMPLATE/PARAGRAPH_SPAN/ORDER/TABLE_FORM`。
- `R2` 标题判型组合
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：`B3` 只能作用于 `R1` 产出的 `B1` 序列；模板按 `priority` 顺序为段落块附着角色标记，且每个段落块在同一轮判型中只能保留一个最终 `role_label`。
  - `R2.3` 输出结构：`PR_set`（元素：`PR`，每个 `PR` 必须包含 `block_id`、`role_label`、`template_id`、`order_index`）。
  - `R2.4` 边界绑定：`TITLE_TEMPLATE/PARAGRAPH_ROLE/ORDER`。
- `R3` 文本块归属记录输出组合
  - `R3.1` 参与基：`B1 + B3`。
  - `R3.2` 组合方式：仅在 `R2` 中被标记为 `role_label=title` 的段落块允许成为标题锚点；每个标题段落与其后的连续正文段落必须按照 `ATTACH_SCOPE` 生成同一 `chunk_id` 下的归属记录。
  - `R3.3` 输出结构：
    - `AnchorRecord_set`（元素：`AR`，每个 `AR` 必须包含 `chunk_id`、`title_block_id`）
    - `MemberRecord_set`（元素：`MR`，每个 `MR` 必须包含 `rel_id`、`block_id`、`chunk_id`、`member_role`、`member_order`）
    - `BoundaryRecord_set`（元素：`BR`，每个 `BR` 必须包含 `chunk_id`、`termination_reason`、`order_start`、`order_end`）
  - `R3.4` 边界绑定：`PARAGRAPH_ROLE/ATTACH_SCOPE/ORDER`。
- `R4` 表结构导出组合
  - `R4.1` 参与基：`B1 + B3`。
  - `R4.2` 组合方式：`R1` 产出的 `PB_set` 必须导出为 `段落块表`；`R3` 产出的 `MemberRecord_set` 必须导出为 `文本块成员关系表`。
  - `R4.3` 输出结构：
    - `段落块表`：`BLOCK_ID`，`TEXT`，`START`，`END`，`ORDER_INDEX`
    - `文本块成员关系表`：`REL_ID`，`BLOCK_ID`，`CHUNK_ID`，`MEMBER_ROLE`，`MEMBER_ORDER`
  - `R4.4` 边界绑定：`PARAGRAPH_SPAN/PARAGRAPH_ROLE/ATTACH_SCOPE/ORDER/TABLE_FORM`。
- `R5` 节块构造组合
  - `R5.1` 参与基：`B1 + B3`。
  - `R5.2` 组合方式：每个合法 `chunk_id` 必须以 `R3` 产出的锚点记录、成员记录和边界记录为依据，从 `B1` 对应段落中取回标题段与正文段，并按原文顺序拼接为一个 `ChunkBlock`；未被任何标题归属的普通段落必须独立形成单段 `ChunkBlock`。
  - `R5.3` 输出结构：`CK_set`（元素：`CK`，每个 `CK` 必须包含 `chunk_id`、`title_block_id`、`body_block_id_set`、`text`、`span`、`termination_reason`、`order_range`）。
  - `R5.4` 边界绑定：`PARAGRAPH_SPAN/PARAGRAPH_ROLE/ATTACH_SCOPE/ORDER/CHUNK_FORM`。
- `R6` 完整文档分块组合
  - `R6.1` 参与基：`B1 + B2 + B3`。
  - `R6.2` 组合方式：先由 `B2` 生成 `PB_set`，再由 `B3` 生成 `PR_set`，随后生成三类归属记录集，再导出两类表结构，最后导出覆盖整篇文档的 `ChunkBlock` 集合。
  - `R6.3` 输出结构：`DocChunk_set`（元素：`DocChunk`，每个 `DocChunk` 必须保持 `PB_set`、`PR_set`、`AnchorRecord_set`、`MemberRecord_set`、`BoundaryRecord_set`、表结构输出与 `CK_set` 之间的可追溯绑定）。
  - `R6.4` 边界绑定：`NORM_DOC/PARAGRAPH_TEMPLATE/TITLE_TEMPLATE/PARAGRAPH_SPAN/PARAGRAPH_ROLE/ATTACH_SCOPE/ORDER/TABLE_FORM/CHUNK_FORM`。

## 5. 验证（Verification）

- `V1` 分段模板有效性：`B2` 必须满足 `PARAGRAPH_TEMPLATE` 约束，且模板优先级必须可判定。
- `V2` 标题模板有效性：`B3` 必须满足 `TITLE_TEMPLATE` 约束，且角色标记结果必须满足 `PARAGRAPH_ROLE` 约束。
- `V3` 段落块表一致性：`R1` 产出的 `PB_set` 与 `段落块表` 必须满足 `PARAGRAPH_SPAN/ORDER/TABLE_FORM` 约束，且块区间不得重叠。
- `V4` 标题标记合法性：`R2` 产出的 `PR_set` 必须满足 `PARAGRAPH_ROLE/ORDER` 约束，且每个段落块只能对应一个最终角色标记。
- `V5` 归属记录合法性：`R3` 产出的 `AnchorRecord_set`、`MemberRecord_set`、`BoundaryRecord_set` 必须满足 `ATTACH_SCOPE/ORDER` 约束，且 `title_block_id` 只能引用标题段落。
- `V6` 表结构合法性：`R4` 产出的 `段落块表` 与 `文本块成员关系表` 必须满足 `TABLE_FORM/ORDER` 约束，且 `chunk_id` 分组下必须存在唯一标题成员。
- `V7` 节块覆盖性：`R5` 产出的 `CK_set` 必须覆盖全部 `PB_set`，且每个 `CK` 都能回溯到唯一的 `title_block_id/body_block_id_set/order_range`。
- `V8` 完整链路可追溯性：`R6` 必须保持 `PB_set -> PR_set -> Record_set -> Table_set -> CK_set` 的稳定绑定关系。
- `V9` 最小必要性：移除任一 `B*` 后，`V3~V8` 至少一项失败。
- `V10` 结论表达：逐条输出 `R* -> 有效/无效 / 边界 / 失效原因` 的结论。
