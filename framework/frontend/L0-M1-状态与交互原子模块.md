# 前端状态与交互原子模块:FrontendStateInteractionAtoms

@framework

本模块定义前端最底层的状态与交互结构。它把值流、动作出口和状态相位压到同一原子层，不再把“值通道”“动作通道”“状态集合”机械拆成多份平行基。

直观示例：搜索输入的状态与交互闭环

输入关键词
  -> 更新当前值
  -> 触发提交
  -> 进入处理中
  -> 返回成功或失败
  -> 显示反馈 / 清空 / 回退
  -> 进入下一轮输入

这个例子说明：本模块不是把值流、动作通道和状态集合分开描述，而是要求它们在同一个前端交互单元中形成稳定闭环，才能成为可复用的底层状态与交互原子。

## 1. 能力声明（Capability Statement）

- `C1` 值流组织能力：把默认值、当前值、提交出口与复位出口收敛为稳定交互流结构。
- `C2` 动作承接能力：把触发动作、切换动作与回退动作收敛为稳定动作结构。
- `C3` 状态相位表达能力：把悬停、聚焦、按下、加载、成功、失败、禁用与反馈相位收敛为稳定状态结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责具体组件外形、宿主元素与主题令牌取值。

## 2. 边界定义（Boundary / 参数）

- `VALUE` 值流边界：默认值、当前值、值变化顺序与值更新归属关系需稳定。来源：`C1 + C2`。
- `ACTION` 动作边界：触发出口、提交出口、切换出口与动作结果类型需稳定。来源：`C1 + C2`。
- `STATE` 状态边界：默认、悬停、聚焦、按下、加载、成功、失败与禁用相位集合、状态迁移关系与反馈窗口需稳定；其中交互前态、操作中态与结果态的切换条件需可判定。来源：`C2 + C3`。
- `RESET` 回退边界：清空、撤销、回退落点与回退后的值恢复条件需稳定。来源：`C1 + C3`。
- `FEEDBACK` 反馈边界：成功提示、失败提示、处理中反馈位与反馈归属关系需稳定。来源：`C2 + C3`。
- `TRANSFER` 传递边界：流入顺序、流出顺序、外部同步位与内外状态同步条件需稳定。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 值流动作结构基：由默认值、当前值、触发出口、提交出口与复位出口构成的交互流结构。来源：`C1 + C2 + VALUE + ACTION + RESET + TRANSFER`。
- `B2` 状态相位反馈结构基：由交互前态、状态集合、相位切换、反馈出口、禁用位与对外暴露位构成的状态结构。来源：`C3 + STATE + FEEDBACK + TRANSFER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 值流先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定值流归属关系、动作出口类型与回退链，再允许更高层组件附着状态语义。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`VALUE + ACTION + RESET + TRANSFER`。
- `R2` 状态闭合
  - `R2.1` 参与基：`B1 + B2`。
- `R2.2` 组合方式：在值流动作结构上叠加交互前态、状态迁移关系、反馈出口与对外同步条件，形成稳定前端交互闭环。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`STATE + FEEDBACK + TRANSFER + VALUE + ACTION + RESET`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：没有值流归属关系就声明状态相位、动作结果与反馈出口不可解释、回退链断裂或内外同步条件缺位的方案均无效。
  - `R3.3` 失效结论：`N1`。
  - `R3.4` 边界绑定：`VALUE + ACTION + STATE + RESET + FEEDBACK + TRANSFER`。

## 5. 验证（Verification）

- `V1` 值流完整性：值流结构必须满足 `VALUE + ACTION` 约束；默认值、当前值与动作出口归属关系可判定，且值更新顺序不破坏动作闭合。
- `V2` 状态闭合有效性：状态结构必须满足 `STATE + FEEDBACK` 约束；默认、悬停、聚焦、按下、加载、成功、失败与禁用等状态集合可解释，状态迁移关系明确，且反馈出口能附着合法状态相位。
- `V3` 回退与传递一致性：回退与对外同步结构必须满足 `RESET + TRANSFER` 约束；复位与回退路径完整，且流入流出顺序与内外状态同步条件可解释。
- `V4` 最小必要性：移除任一 `B*` 后，`V1 ~ V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。
