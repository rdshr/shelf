# 前端面板与视口原子模块:FrontendPanelViewportAtoms

@framework

本模块定义面板与视口这一层的组件原子。它在更底层的槽位壳体、状态通道、宿主绑定与令牌主题之上，收束出阅读与浏览常用的承载原子。

## 1. 能力声明（Capability Statement）

- `C1` 面板外壳承载能力：沉淀面板的头部、主体、尾部与边框外形结构。
- `C2` 视口滚动承载能力：沉淀滚动区、分节槽位与空态承载结构。
- `C3` 绑定承载能力：沉淀面板与视口和前端框架宿主元素、slot/children、滚动接口之间的稳定结构。
- `C4` 非能力项：不负责领域布局语义、动画实现与具体主题值。

## 2. 边界定义（Boundary / 参数）

- `PANELFRAME` 外形边界：头部、主体、尾部、边框粗细角色、圆角形态与阴影角色需稳定。来源：`C1 + C3`。
- `PANELTONE` 视觉边界：面板背景、描边、分隔线与层级强调需以语义角色表达。来源：`C1 + C2`。
- `VIEWPORT` 视口边界：滚动区、固定头尾、分节锚点与滚动范围需稳定。来源：`C2 + C3`。
- `EMPTY` 空态边界：无内容、加载中与占位态承载需统一。来源：`C2 + C3`。
- `PANELBIND` 绑定边界：宿主元素、slot/children、scroll ref 与滚动回调接口需稳定。来源：`C1 + C3`。
- `PANELA11Y` 可访问边界：区域标题、阅读顺序与滚动焦点回路需达标。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 面板外壳结构基：L0.M0[R2,R3] + L0.M3[R1,R2]。来源：`C1 + PANELFRAME + PANELTONE`。
- `B2` 视口分节结构基：L0.M0[R3] + L0.M1[R2,R3]。来源：`C1 + C2 + VIEWPORT + EMPTY`。
- `B3` 面板绑定结构基：L0.M2[R1,R3]。来源：`C2 + C3 + PANELBIND + PANELA11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 外壳先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定面板头部、主体、尾部与边框外形，再允许更上层结构附着内容语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`PANELFRAME/PANELTONE/PANELA11Y`。
- `R2` 视口闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将面板外壳与滚动视口、分节锚点和空态承载对齐，形成稳定阅读与浏览面。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`VIEWPORT/EMPTY/PANELFRAME`。
- `R3` 框架绑定闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将面板外壳和视口结构映射到宿主元素、slot/children 与滚动接口，形成可被 React 等前端框架直接承接的面板视口原子。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`PANELBIND/PANELA11Y/VIEWPORT/EMPTY`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：跳过面板外壳直接定义滚动视口、面板边框不可解释或宿主元素与滚动接口错配的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`PANELFRAME/PANELTONE/VIEWPORT/EMPTY/PANELBIND/PANELA11Y`。

## 5. 验证（Verification）

- `V1` 外壳完整性：头部、主体、尾部与边框外形必须完整。
- `V2` 视口完整性：滚动区、分节锚点与空态承载必须满足 `VIEWPORT/EMPTY` 约束。
- `V3` 绑定有效性：宿主元素、slot/children 与滚动接口必须满足 `PANELBIND` 约束并能映射到具体前端框架。
- `V4` 可访问符合性：有效组合必须满足 `PANELA11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
