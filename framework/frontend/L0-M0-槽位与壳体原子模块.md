# 前端槽位与壳体原子模块:FrontendSlotShellAtoms

@framework

本模块定义前端组件在更深层上的共相结构。按钮、输入框、文本域、文件选择、文本块、面板、列表与反馈，都会先落到“槽位如何承载内容、壳体如何限定边界、分区如何组织空态”这一层，再继续长成具体组件原子。

## 1. 能力声明（Capability Statement）

- `C1` 槽位承载能力：沉淀主内容槽位、辅助槽位与前后缀槽位的稳定组织结构。
- `C2` 壳体限定能力：沉淀边框、盒模型、分区轮廓与区域边界的稳定结构。
- `C3` 分区空态承载能力：沉淀局部分区、空态占位与槽位顺序关系的稳定结构。
- `C4` 非能力项：不负责领域语义、具体主题值与宿主框架接口。

## 2. 边界定义（Boundary / 参数）

- `SLOT` 槽位边界：主槽位、辅助槽位、前后缀槽位与说明槽位需稳定。来源：`C1 + C3`。
- `SHELL` 壳体边界：边框轮廓、圆角角色、阴影角色与盒模型外形需稳定。来源：`C1 + C2`。
- `REGION` 分区边界：头部、主体、尾部、侧区与局部视口分区需明确。来源：`C2 + C3`。
- `ORDER` 顺序边界：槽位前后关系、分区读写顺序与插入位置需稳定。来源：`C1 + C3`。
- `EMPTY` 空态边界：无内容、占位中与待附着结构需统一承载。来源：`C2 + C3`。
- `A11Y` 可访问边界：槽位说明、区域标题与阅读顺序需达标。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 槽位集结构基：由主槽位、辅助槽位与前后缀槽位构成的槽位集结构。来源：`C1 + SLOT + ORDER`。
- `B2` 壳体轮廓结构基：由边框轮廓、盒模型与分区边界构成的壳体结构。来源：`C1 + C2 + SHELL + REGION`。
- `B3` 分区空态结构基：由区域顺序、空态占位与阅读提示构成的分区结构。来源：`C2 + C3 + REGION + EMPTY + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 槽位先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定主槽位、辅助槽位与前后缀槽位，再允许更高层附着动作、值与主题语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SLOT/ORDER/A11Y`。
- `R2` 壳体闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将槽位集装入边框轮廓和盒模型，使槽位在稳定壳体中成立。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`SHELL/REGION/SLOT/ORDER`。
- `R3` 分区承载闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在壳体与槽位之上叠加分区顺序和空态占位，形成可供上层组件复用的共相承载结构。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`REGION/EMPTY/A11Y/SLOT/SHELL`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失槽位关系、壳体轮廓不可解释或区域顺序不可达的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SLOT/SHELL/REGION/ORDER/EMPTY/A11Y`。

## 5. 验证（Verification）

- `V1` 槽位完整性：主槽位、辅助槽位与前后缀槽位必须完整。
- `V2` 壳体完整性：边框轮廓、盒模型与分区边界必须满足 `SHELL/REGION` 约束。
- `V3` 分区有效性：空态占位与阅读顺序必须满足 `ORDER/EMPTY/A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
