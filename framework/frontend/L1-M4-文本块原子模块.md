# 前端文本块原子模块:FrontendTextBlockAtoms

@framework

本模块定义文本块这一层的组件原子。它把更底层的槽位壳体、宿主绑定与令牌主题收束成标题、正文、摘要和辅助文案可复用的文本原子。

## 1. 能力声明（Capability Statement）

- `C1` 文本内容承载能力：沉淀标题、正文、摘要与辅助文案的稳定文本结构。
- `C2` 文本语义层级能力：沉淀文本层级、强调关系与截断规则的稳定结构。
- `C3` 绑定承载能力：沉淀文本块与前端框架 children/slot、语义标签和可访问标记之间的稳定结构。
- `C4` 非能力项：不负责领域文案本身、富文本解析与具体主题值。

## 2. 边界定义（Boundary / 参数）

- `TEXTCONTENT` 内容边界：主文本、辅助文本、前后缀与说明文本槽位需稳定。来源：`C1 + C2`。
- `TEXTTYPO` 排版边界：标题级别、字号角色、字重角色与行高角色需稳定。来源：`C1 + C2`。
- `TEXTTONE` 视觉边界：主文本色、辅助文本色、强调色与禁用色需以语义角色表达。来源：`C1 + C3`。
- `TEXTCLAMP` 截断边界：单行、多行、换行与省略规则需稳定。来源：`C2 + C3`。
- `TEXTBIND` 绑定边界：宿主标签、children/slot、内联 HTML 策略与 ref 接口需稳定。来源：`C2 + C3`。
- `TEXTA11Y` 可访问边界：标题层级、朗读顺序与说明关联需达标。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文本内容结构基：L0.M0[R1,R3]。来源：`C1 + TEXTCONTENT + TEXTA11Y`。
- `B2` 文本排版语义基：L0.M1[R3] + L0.M3[R1,R3]。来源：`C1 + C2 + TEXTTYPO + TEXTTONE + TEXTCLAMP`。
- `B3` 文本绑定结构基：L0.M2[R1,R3]。来源：`C2 + C3 + TEXTBIND + TEXTA11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 内容先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定文本内容槽位与说明关系，再允许更上层结构附着领域语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TEXTCONTENT/TEXTA11Y`。
- `R2` 排版语义闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在文本内容上绑定层级、色调语义和截断规则，形成稳定文本块。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`TEXTTYPO/TEXTTONE/TEXTCLAMP`。
- `R3` 框架绑定闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将文本内容和排版语义映射到宿主标签、children/slot 与可访问标记，形成可被 React 等前端框架直接承接的文本块原子。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`TEXTBIND/TEXTA11Y/TEXTTYPO/TEXTCLAMP`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：跳过文本内容直接定义排版、标题层级不可解释或宿主标签与绑定接口错配的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`TEXTCONTENT/TEXTTYPO/TEXTTONE/TEXTCLAMP/TEXTBIND/TEXTA11Y`。

## 5. 验证（Verification）

- `V1` 内容完整性：主文本、辅助文本与说明槽位必须完整。
- `V2` 排版语义完整性：层级、色调语义与截断规则必须满足 `TEXTTYPO/TEXTTONE/TEXTCLAMP` 约束。
- `V3` 绑定有效性：宿主标签、children/slot 与可访问标记必须满足 `TEXTBIND` 约束并能映射到具体前端框架。
- `V4` 可访问符合性：有效组合必须满足 `TEXTA11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
