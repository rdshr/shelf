# 前端宿主绑定与可达原子模块:FrontendHostBindingA11yAtoms

@framework

本模块定义前端组件在更深层上的绑定共相。无论是按钮、输入框还是列表树，最终都要落到宿主元素、属性接口、事件出口、焦点回路和可访问说明上；这层结构应被抽出来，而不是散落在每个组件里各写一套。

## 1. 能力声明（Capability Statement）

- `C1` 宿主承载能力：沉淀组件与宿主标签、宿主元素和宿主角色之间的稳定映射结构。
- `C2` 接口绑定能力：沉淀属性接口、事件接口、引用句柄与 slot/children 绑定结构。
- `C3` 可达回路能力：沉淀焦点顺序、语义提示与阅读可达关系的稳定结构。
- `C4` 非能力项：不负责具体组件外形、主题实例值与领域业务动作。

## 2. 边界定义（Boundary / 参数）

- `HOST` 宿主边界：button、input、textarea、section、listitem 等宿主角色需稳定。来源：`C1 + C2`。
- `PROP` 属性边界：value、type、disabled、aria-*、data-* 与 slot/children 入口需稳定。来源：`C1 + C2`。
- `EVENT` 事件边界：click、change、submit、focus、blur、scroll 与 expand 等出口需稳定。来源：`C2 + C3`。
- `REF` 引用边界：ref、handle 与滚动/聚焦引用句柄需稳定。来源：`C1 + C2`。
- `FOCUS` 焦点边界：聚焦顺序、回退顺序与焦点回路需明确。来源：`C2 + C3`。
- `A11Y` 可访问边界：语义说明、阅读提示、标签关联与状态播报需达标。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 宿主角色结构基：由宿主元素、语义角色与宿主替换位构成的宿主结构。来源：`C1 + HOST + PROP`。
- `B2` 接口事件结构基：由属性入口、事件出口与引用句柄构成的绑定结构。来源：`C1 + C2 + PROP + EVENT + REF`。
- `B3` 焦点可达结构基：由焦点顺序、阅读提示与状态播报位构成的可达结构。来源：`C2 + C3 + FOCUS + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 宿主先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定宿主角色和属性入口，再允许更高层组件附着值、动作与主题语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`HOST/PROP/REF`。
- `R2` 接口闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将宿主角色与属性入口绑定到稳定事件出口和引用句柄，形成可挂接宿主框架的接口结构。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`PROP/EVENT/REF/HOST`。
- `R3` 可达闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在宿主和接口结构上叠加焦点顺序、语义提示与状态播报，形成完整可达回路。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`FOCUS/A11Y/HOST/PROP/EVENT/REF`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：宿主角色与事件不匹配、无焦点回路或无语义提示的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`HOST/PROP/EVENT/REF/FOCUS/A11Y`。

## 5. 验证（Verification）

- `V1` 宿主一致性：宿主角色和属性入口必须满足 `HOST/PROP` 约束。
- `V2` 绑定有效性：事件出口和引用句柄必须满足 `EVENT/REF` 约束。
- `V3` 可达符合性：焦点顺序与语义提示必须满足 `FOCUS/A11Y` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
