# 前端文本与承载原子模块:FrontendTextCarrierAtoms

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文本承载能力：沉淀标题、正文、摘要等文本块的稳定结构。
- `C2` 区块承载能力：沉淀面板、卡片、分区等承载面的稳定结构。
- `C3` 阅读视口承载能力：沉淀滚动视口、节段锚点与阅读区域的稳定结构。
- `C4` 非能力项：不负责领域字段语义、主题实例值与对话状态流。

## 2. 边界定义（Boundary / 参数）

- `TEXT` 文本边界：标题、正文、摘要与辅助文案的承载职责需稳定。来源：`C1 + C3`。
- `PANEL` 面板边界：头部、主体、尾部与分节槽位需明确。来源：`C1 + C2`。
- `VIEWPORT` 视口边界：滚动范围、节段锚点与阅读主区需稳定。来源：`C2 + C3`。
- `SECTION` 分节边界：区块层级、折叠关系与主辅信息位置需稳定。来源：`C1 + C2`。
- `EMPTY` 空态边界：无内容、无正文与占位承载需统一。来源：`C2 + C3`。
- `A11Y` 可访问边界：文本顺序、标题层级与阅读焦点需达标。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文本块结构基：由主文本槽位、辅助文本槽位与层级位构成的文本承载结构。来源：`C1 + TEXT + SECTION`。
- `B2` 面板结构基：由头部槽位、主体槽位与尾部槽位构成的区块承载结构。来源：`C1 + C2 + PANEL + SECTION`。
- `B3` 视口节段结构基：由滚动视口、节段锚点与空态承载构成的阅读结构。来源：`C2 + C3 + VIEWPORT + EMPTY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 文本承载成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定文本槽位和层级位，再允许更上层结构附着领域语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TEXT/SECTION/A11Y`。
- `R2` 区块分层成立
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将文本块嵌入面板槽位，形成稳定主辅信息分层结构。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`PANEL/SECTION/TEXT/A11Y`。
- `R3` 阅读视口闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合文本块、面板和视口节段结构，形成可滚动、可锚点定位的阅读承载面。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`VIEWPORT/EMPTY/PANEL/TEXT/SECTION/A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失文本层级、面板槽位不可解释或视口锚点中断的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`TEXT/PANEL/VIEWPORT/SECTION/EMPTY/A11Y`。

## 5. 验证（Verification）

- `V1` 文本完整性：文本槽位与层级位必须完整。
- `V2` 面板完整性：头部、主体与尾部槽位必须满足 `PANEL/SECTION` 约束。
- `V3` 视口有效性：滚动视口、节段锚点与空态承载必须满足 `VIEWPORT/EMPTY` 约束。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
