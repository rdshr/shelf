# 前端文件选择原子模块:FrontendFilePickerAtoms

@framework

本模块定义文件选择这一层的组件原子。它在更底层的槽位壳体、动作和值通道、宿主绑定之上，约束文件入口和选择状态如何成立。

## 1. 能力声明（Capability Statement）

- `C1` 文件入口承载能力：沉淀文件选择触发面、选择结果承载位与替换取消出口结构。
- `C2` 文件集合状态承载能力：沉淀单文件、多文件、空选择与错误提示状态结构。
- `C3` 绑定承载能力：沉淀文件选择与前端框架宿主元素、accept/multiple、change 回调之间的稳定结构。
- `C4` 非能力项：不负责上传后存储实现、解析算法与具体主题值。

## 2. 边界定义（Boundary / 参数）

- `PICKTRIGGER` 入口边界：选择入口、替换入口与取消出口需稳定。来源：`C1 + C2`。
- `PICKACCEPT` 接受边界：accept 规则、多选规则与目录选择能力需稳定。来源：`C1 + C3`。
- `PICKSELECTION` 选择边界：已选文件槽位、空选择位与错误信息位需稳定。来源：`C1 + C2`。
- `PICKSTATE` 状态边界：默认、已选、禁用、上传中与错误状态需稳定。来源：`C2 + C3`。
- `PICKBIND` 绑定边界：宿主元素、files 接口、onChange/onCancel/ref 和拖放映射接口需稳定。来源：`C2 + C3`。
- `PICKA11Y` 可访问边界：可聚焦性、键盘触发、文件选择说明与错误提示需达标。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件入口结构基：L0.M0[R2,R3] + L0.M1[R1,R2]。来源：`C1 + PICKTRIGGER + PICKA11Y`。
- `B2` 文件集合状态基：L0.M1[R2,R3] + L0.M3[R1,R2]。来源：`C1 + C2 + PICKSELECTION + PICKSTATE`。
- `B3` 文件绑定结构基：L0.M2[R1,R3]。来源：`C2 + C3 + PICKACCEPT + PICKBIND + PICKA11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 入口先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定文件选择、替换和取消出口，再允许更上层结构附着领域语义。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`PICKTRIGGER/PICKA11Y`。
- `R2` 选择状态闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将入口结构与文件集合状态对齐，形成稳定文件选择结构。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`PICKSELECTION/PICKSTATE/PICKTRIGGER`。
- `R3` 框架绑定闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将文件入口和选择状态映射到宿主元素、accept/multiple 配置与 change 回调，形成可被 React 等前端框架直接承接的文件选择原子。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`PICKACCEPT/PICKBIND/PICKSTATE/PICKA11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：跳过选择入口直接绑定文件集合、accept 规则不可解释或宿主元素与文件接口错配的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`PICKTRIGGER/PICKACCEPT/PICKSELECTION/PICKSTATE/PICKBIND/PICKA11Y`。

## 5. 验证（Verification）

- `V1` 入口完整性：选择入口、替换入口与取消出口必须完整。
- `V2` 选择状态完整性：已选文件槽位、空选择位与状态集合必须满足 `PICKSELECTION/PICKSTATE` 约束。
- `V3` 绑定有效性：accept/multiple 配置、宿主元素与回调接口必须满足 `PICKBIND` 约束并能映射到具体前端框架。
- `V4` 可访问符合性：有效组合必须满足 `PICKA11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
