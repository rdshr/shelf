# 前端列表与层级原子模块:FrontendListHierarchyAtoms

@framework

本模块定义列表与层级这一层的组件原子。它把更底层的槽位壳体、状态通道与宿主绑定收束成顺序列表、树层级和焦点回路。

## 1. 能力声明（Capability Statement）

- `C1` 顺序列表承载能力：沉淀顺序项集合与浏览路径的稳定结构。
- `C2` 层级树承载能力：沉淀父子节点、展开关系与目录层次的稳定结构。
- `C3` 选择焦点保持能力：沉淀当前选中项、展开项与空态的稳定结构。
- `C4` 非能力项：不负责领域节点语义、后端分页策略与样式实例值。

## 2. 边界定义（Boundary / 参数）

- `LIST` 列表边界：顺序项、当前位置与浏览主链需稳定。来源：`C1 + C3`。
- `TREE` 树边界：父子层级、展开关系与折叠入口需稳定。来源：`C2 + C3`。
- `ITEM` 节点边界：节点标题、辅助信息与操作槽位需明确。来源：`C1 + C2`。
- `FOCUS` 焦点边界：当前选中项、当前展开项与返回路径需稳定。来源：`C1 + C3`。
- `EMPTY` 空态边界：无项、无子节点与未加载节点需统一承载。来源：`C2 + C3`。
- `A11Y` 可访问边界：键盘浏览、树级提示与选中提示需达标。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 列表结构基：L0.M0[R3] + L0.M1[R2,R3]。来源：`C1 + LIST + EMPTY + FOCUS`。
- `B2` 树结构基：L0.M0[R2,R3] + L0.M1[R1,R3]。来源：`C2 + TREE + FOCUS`。
- `B3` 节点项结构基：L0.M0[R1,R2] + L0.M2[R1,R3]。来源：`C1 + C2 + ITEM + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 顺序列表成立
  - `R1.1` 参与基：`B1 + B3`。
  - `R1.2` 组合方式：将节点项结构装入顺序列表中，形成稳定浏览主链。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`LIST/ITEM/EMPTY/A11Y`。
- `R2` 层级树成立
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：将节点项结构装入父子层级关系中，形成可展开层级树。
  - `R2.3` 输出能力：`C2`。
  - `R2.4` 边界绑定：`TREE/ITEM/FOCUS/A11Y`。
- `R3` 选择焦点闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合顺序列表、层级树和节点项结构，使选中项、展开项与空态保持在同一焦点回路内。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`LIST/TREE/ITEM/FOCUS/EMPTY/A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失节点项、焦点无法回溯或空态不可承载的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`LIST/TREE/ITEM/FOCUS/EMPTY/A11Y`。

## 5. 验证（Verification）

- `V1` 列表完整性：顺序项、当前位置与空态承载必须完整。
- `V2` 树完整性：父子节点与展开关系必须满足 `TREE/FOCUS` 约束。
- `V3` 焦点有效性：选中项、展开项与返回路径必须构成稳定回路。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
