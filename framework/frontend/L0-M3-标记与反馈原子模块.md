# 前端标记与反馈原子模块:FrontendMarkerFeedbackAtoms

@framework

## 1. 能力声明（Capability Statement）

- `C1` 标签标记承载能力：沉淀标签、徽记与小型状态标记的稳定结构。
- `C2` 回合气泡承载能力：沉淀消息气泡、引用气泡与提示气泡的稳定结构。
- `C3` 状态反馈承载能力：沉淀提示条、占位态与结果反馈的稳定结构。
- `C4` 非能力项：不负责领域解释文本、后端告警机制与主题实例值。

## 2. 边界定义（Boundary / 参数）

- `TAG` 标记边界：标签文本、附着位置与状态差异需稳定。来源：`C1 + C3`。
- `BUBBLE` 气泡边界：消息主体、来源提示与展开关系需稳定。来源：`C2 + C3`。
- `STATUS` 状态边界：成功、失败、处理中与提醒状态需统一表达。来源：`C1 + C3`。
- `EMPTY` 空态边界：无结果、无消息与占位提示需统一承载。来源：`C2 + C3`。
- `EVENT` 事件边界：展开、关闭、重试与跳转出口需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：状态提示、消息阅读与标签说明需达标。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 标签结构基：由标签文本槽位、附着位与状态位构成的标记结构。来源：`C1 + TAG + STATUS`。
- `B2` 气泡结构基：由主体槽位、辅助说明位与展开位构成的消息结构。来源：`C2 + BUBBLE + EVENT`。
- `B3` 提示占位结构基：由状态说明槽位、空态槽位与关闭出口构成的反馈结构。来源：`C2 + C3 + EMPTY + STATUS`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 标签附着成立
  - `R1.1` 参与基：`B1 + B3`。
  - `R1.2` 组合方式：将标签结构与提示占位结构对齐，形成稳定附着标记面。
  - `R1.3` 输出能力：`C1 + C3`。
  - `R1.4` 边界绑定：`TAG/STATUS/EMPTY/A11Y`。
- `R2` 回合反馈成立
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：将气泡结构与提示占位结构并轨，形成稳定消息和结果反馈面。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`BUBBLE/STATUS/EMPTY/EVENT/A11Y`。
- `R3` 状态表达闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合标签、气泡与提示占位结构，使状态表达、消息表达和附着标记形成统一反馈回路。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`TAG/BUBBLE/STATUS/EMPTY/EVENT/A11Y`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失状态承载、消息反馈不可展开或标签无法附着的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`TAG/BUBBLE/STATUS/EMPTY/EVENT/A11Y`。

## 5. 验证（Verification）

- `V1` 标签完整性：标签文本、附着位与状态位必须完整。
- `V2` 气泡有效性：消息主体、辅助说明与展开关系必须满足 `BUBBLE/EVENT` 约束。
- `V3` 反馈有效性：提示与占位结构必须满足 `STATUS/EMPTY` 约束。
- `V4` 可访问符合性：有效组合必须满足 `A11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
