# 前端状态与通道原子模块:FrontendStateChannelAtoms

@framework

本模块定义前端组件在更深层上的状态共相。按钮和文件选择依赖动作通道，输入框和文本域依赖值通道，几乎所有组件都依赖相位状态和切换回路；这些不该重复在每个组件里各自发明。

## 1. 能力声明（Capability Statement）

- `C1` 值通道承载能力：沉淀默认值、当前值、派生值与清空回路的稳定结构。
- `C2` 动作通道承载能力：沉淀提交、重置、展开、关闭与切换动作的稳定结构。
- `C3` 状态相位承载能力：沉淀默认、激活、加载、禁用、错误与恢复状态的稳定结构。
- `C4` 非能力项：不负责具体组件外形、宿主标签与实例主题值。

## 2. 边界定义（Boundary / 参数）

- `VALUE` 值边界：默认值、当前值、派生值与只读值需可区分。来源：`C1 + C3`。
- `ACTION` 动作边界：提交、重置、展开、关闭、选择与重试动作类型需稳定。来源：`C2 + C3`。
- `STATE` 状态边界：默认、聚焦、激活、禁用、加载、成功与错误相位需稳定。来源：`C1 + C3`。
- `TRANSFER` 传递边界：值流入、动作触发、状态切换与回退顺序需稳定。来源：`C1 + C2`。
- `RESET` 复位边界：清空、取消、回退与重试出口需明确。来源：`C1 + C2`。
- `FEEDBACK` 暴露边界：状态向上游暴露给主题或提示结构的方式需稳定。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 值流结构基：由默认值、当前值与流入流出顺序构成的值通道结构。来源：`C1 + VALUE + TRANSFER`。
- `B2` 动作流结构基：由动作类型、触发出口与复位出口构成的动作通道结构。来源：`C1 + C2 + ACTION + RESET + TRANSFER`。
- `B3` 状态相位结构基：由状态集合、切换顺序与向外暴露位构成的相位结构。来源：`C2 + C3 + STATE + FEEDBACK + RESET`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 通道先行
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定值流和动作流，再允许更高层组件选择以值为主还是以动作为主。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`VALUE/ACTION/TRANSFER/RESET`。
- `R2` 相位闭合
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：将值流或动作流映射到稳定状态集合和切换顺序，形成可观测相位结构。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`STATE/TRANSFER/FEEDBACK/RESET`。
- `R3` 通道状态统一
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合值流、动作流和相位结构，使组件在输入、触发和恢复阶段共享同一状态骨架。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`VALUE/ACTION/STATE/TRANSFER/RESET/FEEDBACK`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：值流和动作流顺序冲突、状态切换无复位出口或反馈位不可解释的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`VALUE/ACTION/STATE/TRANSFER/RESET/FEEDBACK`。

## 5. 验证（Verification）

- `V1` 通道完整性：值流和动作流必须至少具备一个稳定输入与一个稳定输出。
- `V2` 相位完整性：状态集合与切换顺序必须满足 `STATE/TRANSFER/RESET` 约束。
- `V3` 暴露有效性：上游必须能够读取稳定状态暴露位。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
