# 前端标记与反馈原子模块:FrontendMarkerFeedbackAtoms

@framework

模块短标签：附着反馈
典型落地对象：badge / tag / inline status / empty hint / alert bubble
不包括：锚点附着提示、输入内反馈闭环、文件摄取反馈闭环、媒体预览反馈闭环

本模块定义前端最小标记与反馈原子。它承接标签、状态标记、消息气泡和空态提示这类“附着在主体结构上、但不改写主体主关系的反馈表达”，不把锚点提示面、主体结构专属状态闭环或页面级通知流程混进同一模块。

## 1. 能力声明（Capability Statement）

- `C1` 状态标记能力：把标签、状态标记与摘要提示收敛为稳定标记结构。
- `C2` 附着反馈能力：把消息气泡、失败提示与空态说明收敛为稳定附着反馈结构。
- `C3` 可达反馈能力：把朗读关系、提示顺序与动作说明收敛为稳定可访问反馈结构。

### 非职责声明（Non-Responsibility Statement）

- `N1` 非职责声明：不负责文本编辑、集合浏览与页面路由编排。
- `N2` 非职责声明：不负责围绕稳定锚点短时附着的提示、说明与注释结构。
- `N3` 非职责声明：不负责输入、文件摄取、日期时间输入与媒体预览等主体结构内部的专属状态闭环。
- `N4` 非职责声明：不负责局部任务接管、页面级通知流与长期驻留帮助面板。

## 2. 边界定义（Boundary / 参数）

- `TAG` 标记边界：标签、状态芯片、摘要标记位与标记归属关系需稳定。来源：`C1 + C3`。
- `BUBBLE` 气泡边界：消息气泡、附着反馈文本、动作位与反馈附着关系需稳定。来源：`C2 + C3`。
- `EMPTY` 空态边界：空列表、空面板、无结果占位与空态归属条件需稳定。来源：`C1 + C2`。
- `STATUS` 状态边界：成功、失败、处理中、禁用说明等状态集合、状态映射关系与反馈窗口需稳定。来源：`C1 + C2`。
- `EVENT` 事件边界：展开、关闭、确认、撤销等事件出口与事件结果类型需稳定。来源：`C2 + C3`。
- `A11Y` 可访问边界：提示朗读、状态说明、动作说明与辅助技术可感知关系需可达。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 状态标记结构基：L0.M0[R1,R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C1 + TAG + STATUS + A11Y`。
- `B2` 附着反馈结构基：L0.M0[R1,R2] + L0.M1[R2] + L0.M2[R1,R2]。来源：`C2 + C3 + BUBBLE + EMPTY + EVENT + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 标记先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定标签、状态芯片、摘要提示与标记归属关系，再允许更高层结构挂接附着反馈动作。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TAG + STATUS + A11Y`。
- `R2` 附着反馈闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将气泡、空态说明、事件出口与状态反馈附着到稳定标记结构或主体结构边缘，形成统一附着反馈原子，不改写主体结构主关系。
  - `R2.3` 输出能力：`C1 + C2 + C3`。
  - `R2.4` 边界绑定：`BUBBLE + EMPTY + EVENT + A11Y + TAG + STATUS`。
- `R3` 禁止组合
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：反馈不附着稳定标记或主体边缘、标记归属关系不可解释、提示顺序不可解释，或把锚点提示面、主体结构内部专属状态闭环、页面级通知流误写为标记反馈原子的方案均无效。
  - `R3.3` 失效结论：`N2 + N3 + N4`。
  - `R3.4` 边界绑定：`TAG + BUBBLE + EMPTY + STATUS + EVENT + A11Y`。

## 5. 验证（Verification）

- `V1` 标记完整性：标记结构必须满足 `TAG + STATUS` 约束；标签、状态芯片与摘要提示完整，且标记归属关系与状态映射关系可判定。
- `V2` 反馈闭合有效性：附着反馈结构必须满足 `BUBBLE + EMPTY + EVENT` 约束；气泡、空态说明与事件出口可解释，且反馈附着关系不改写主体结构主关系。
- `V3` 可达一致性：可访问反馈结构必须满足 `A11Y + STATUS` 约束；朗读关系、状态说明与动作说明完整，且对辅助技术可感知。
- `V4` 主体附着稳定性：反馈必须附着于稳定主体结构，而不是替代主体结构本身。
- `V5` 最小必要性：移除任一 `B*` 后，`V1 ~ V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* + 边界绑定` 或 `R* -> N* + 边界绑定` 的通过或失败结论，并附对应边界证据。
