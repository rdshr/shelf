# 前端按钮原子模块:FrontendButtonAtoms

@framework

本模块定义按钮这一层的组件原子。它建立在更底层的槽位壳体、状态通道、宿主绑定与令牌主题之上，把这些共相结构收束成可直接实例化的按钮原子。

## 1. 能力声明（Capability Statement）

- `C1` 按钮外形承载能力：沉淀按钮的内容槽位、边框轮廓与盒模型外形。
- `C2` 按钮触发承载能力：沉淀按钮的动作语义、事件出口与状态切换结构。
- `C3` 按钮绑定承载能力：沉淀按钮与前端框架宿主元素、属性接口和可访问标记之间的稳定绑定结构。
- `C4` 非能力项：不负责领域动作语义、具体颜色值与运行时业务逻辑。

## 2. 边界定义（Boundary / 参数）

- `BTNCONTENT` 内容边界：文本槽位、图标槽位与图标前后关系需稳定。来源：`C1 + C2`。
- `BTNFRAME` 外形边界：按钮高度、内边距、边框有无、边框粗细角色、圆角形态与块级/行内形态需稳定。来源：`C1 + C3`。
- `BTNCHROME` 视觉边界：填充、描边、文字色与强调等级需以语义角色表达。来源：`C1 + C2`。
- `BTNACTION` 动作边界：提交、重置、导航、弹出与普通点击等动作类型需稳定。来源：`C2 + C3`。
- `BTNSTATE` 状态边界：默认、悬停、按下、禁用与加载等状态集需稳定。来源：`C1 + C2`。
- `BTNBIND` 绑定边界：宿主元素、框架属性接口、事件回调、引用句柄与 children/slot 绑定方式需稳定。来源：`C2 + C3`。
- `BTNA11Y` 可访问边界：可聚焦性、键盘触发、aria 标签与焦点提示需达标。来源：`C1 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 按钮外壳结构基：L0.M0[R2,R3] + L0.M3[R1,R2]。来源：`C1 + BTNCONTENT + BTNFRAME + BTNCHROME`。
- `B2` 按钮状态语义基：L0.M1[R1,R3] + L0.M3[R2,R3]。来源：`C1 + C2 + BTNCHROME + BTNACTION + BTNSTATE`。
- `B3` 按钮绑定结构基：L0.M2[R1,R3]。来源：`C2 + C3 + BTNBIND + BTNA11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 外壳先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定内容槽位、边框轮廓、圆角形态与盒模型，再允许更上层结构附着语义动作。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`BTNCONTENT/BTNFRAME/BTNA11Y`。
- `R2` 状态语义闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在按钮外壳上绑定视觉语义、动作类型与状态集合，形成稳定可实例化按钮。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`BTNCHROME/BTNACTION/BTNSTATE/BTNFRAME`。
- `R3` 框架绑定闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将按钮外壳和状态语义映射到宿主元素、属性接口、事件出口与可访问标记，形成可被 React 等前端框架直接承接的按钮原子。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`BTNBIND/BTNA11Y/BTNACTION/BTNSTATE/BTNCONTENT`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：跳过按钮外壳直接定义动作、边框轮廓不可解释或宿主元素与事件出口错配的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`BTNCONTENT/BTNFRAME/BTNCHROME/BTNACTION/BTNSTATE/BTNBIND/BTNA11Y`。

## 5. 验证（Verification）

- `V1` 外壳完整性：内容槽位、边框轮廓与盒模型必须完整。
- `V2` 状态完整性：动作类型、视觉语义与状态集合必须满足 `BTNACTION/BTNSTATE/BTNCHROME` 约束。
- `V3` 绑定有效性：宿主元素、属性接口与事件出口必须满足 `BTNBIND` 约束并能映射到具体前端框架。
- `V4` 可访问符合性：有效组合必须满足 `BTNA11Y` 约束。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
