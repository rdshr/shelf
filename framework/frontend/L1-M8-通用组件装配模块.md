# 前端通用组件装配模块:FrontendGenericComponentAssembly

@framework

本模块位于前端组件原子层内部，承担“共相结构到通用组件面”的装配职责。它不直接复写按钮、输入框、文本域、面板和列表这些 L1 组件原子，而是把更底层的槽位壳体、状态通道、宿主绑定、令牌主题和运行底座收敛成稳定装配支撑面，供更高层前端标准统一承接。

## 1. 能力声明（Capability Statement）

- `C1` 通用装配面能力：把槽位壳体、状态通道和宿主绑定收敛成稳定装配骨架。
- `C2` 主题支撑能力：把装配骨架与令牌、主题和覆盖回退结构绑定到同一组件面。
- `C3` 运行承接能力：把装配骨架与挂载、交付和治理底座对齐，形成稳定通用组件支撑面。
- `C4` 非能力项：不负责领域对象语义与项目实例值。

## 2. 边界定义（Boundary / 参数）

- `COMPOSE` 装配边界：槽位、壳体、通道与绑定关系需形成稳定装配顺序。来源：`C1 + C2`。
- `SURFACE` 承载面边界：表单面、阅读面、列表面与反馈面需统一表达。来源：`C1 + C3`。
- `TOKEN` 主题边界：装配面只能消费语义令牌，不得写死实例样式。来源：`C1 + C2`。
- `RUNTIME` 运行边界：挂载、缓存、回滚与治理支撑需覆盖装配面。来源：`C2 + C3`。
- `EVENT` 事件边界：点击、提交、选择、展开与关闭出口需一致。来源：`C1 + C3`。
- `A11Y` 可访问边界：装配后的通用组件面必须具备稳定焦点和阅读关系。来源：`C1 + C2`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 装配骨架结构基：L0.M0[R2,R3] + L0.M1[R1,R3] + L0.M2[R1,R3]。来源：`C1 + COMPOSE + SURFACE + EVENT`。
- `B2` 主题映射支撑基：L0.M0[R1,R2] + L0.M3[R1,R3]。来源：`C1 + C2 + TOKEN + SURFACE + A11Y`。
- `B3` 运行治理支撑基：L0.M2[R2,R3] + L0.M4[R1,R3]。来源：`C2 + C3 + RUNTIME + EVENT + A11Y`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 装配骨架先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定槽位、通道与绑定的装配顺序，再允许具体组件原子承接这一顺序。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`COMPOSE/SURFACE/EVENT`。
- `R2` 主题映射并轨
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：将装配骨架绑定到语义令牌和主题回退链，形成统一视觉语义面。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`TOKEN/SURFACE/A11Y/COMPOSE`。
- `R3` 运行承接闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在装配骨架和主题映射之上叠加挂载、交付和治理支撑，形成可供更高层前端标准直接承接的通用组件面。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`RUNTIME/EVENT/A11Y/TOKEN/COMPOSE`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：跳过装配骨架直接写主题或缺失运行承接的方案均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`COMPOSE/SURFACE/TOKEN/RUNTIME/EVENT/A11Y`。

## 5. 验证（Verification）

- `V1` 骨架完整性：装配顺序必须同时覆盖槽位、通道与绑定。
- `V2` 主题一致性：装配面必须满足 `TOKEN/A11Y` 约束。
- `V3` 运行符合性：装配面必须满足 `RUNTIME/EVENT` 约束。
- `V4` 最小必要性：移除任一 `B*` 后，`V2~V3` 至少一项失败。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
