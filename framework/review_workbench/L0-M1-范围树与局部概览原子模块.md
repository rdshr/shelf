# 范围树、局部工具位与局部信息原子模块:ReviewWorkbenchScopeTreeLocalActionsAndSummaryAtoms

@framework

本模块定义 `review_workbench` 领域最小的范围树、局部工具位与局部信息结构。推导链为：`framework/frontend/L1-M4-动作-触发-原子模块.md` + `framework/frontend/L1-M9-承载-展示与容器-原子模块.md` + `framework/frontend/L1-M12-浏览-集合浏览-原子模块.md` + `framework/frontend/L1-M15-反馈-标记与反馈-原子模块.md` -> 本模块。也就是说，`review_workbench` 中的范围树、当前范围、范围树局部工具位与局部统计，不单独发明一套新的前端基础结构，而是把前端已存在的集合浏览、展示容器、触发入口和标记反馈约束成“范围限定”这组领域原子。

本模块定义统一审查平台中的范围限定结构原子。它回答“当前结果是在哪个范围下被查看或处理”“该范围下有哪些局部工具位”以及“该范围下有哪些局部信息”这三个最小问题。它不负责查询条件、不负责结果列表，只负责把目录树、分组树、当前节点、范围树局部工具位与局部统计沉成稳定结构。

## 1. 能力声明（Capability Statement）

- `C1` 范围树表达能力：将目录树、分组树、分类树或层级范围结构组织成稳定表达。
- `C2` 当前范围表达能力：将当前节点、当前路径、当前焦点范围组织成稳定上下文。
- `C3` 局部工具位表达能力：将范围树附近的刷新、上传、局部目录动作或局部快捷入口组织成稳定结构。
- `C4` 局部信息表达能力：将当前范围下的统计、计数、摘要信息组织成稳定结构。
- `N1` 非职责声明：不负责查询表单、不负责结果列结构、不负责全局状态条。

## 2. 边界定义（Boundary / 参数）

- `SCOPETREE` 范围树边界：树节点数量、层级深度和展开方式允许变化，但必须保留稳定层级关系，并能识别唯一当前节点。
- `SCOPESTATE` 范围状态边界：当前路径、当前范围和切换后的范围状态允许变化，但任一时刻都必须能唯一解释当前范围及其对应路径。
- `LOCALACTIONS` 局部工具位边界：局部动作入口种类、顺序和展示方式允许变化，但每个入口都必须附着当前范围，并能说明当前作用范围。
- `LOCALSUMMARY` 局部信息边界：计数项、状态统计和摘要卡数量允许变化，但每一项都必须附着当前范围，并能解释其统计来源。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 范围节点基：frontend.L1.M12[R1,R2]。由集合层级、当前项和集合内展开切换关系约束成目录树、分组树与当前节点结构。来源：`C1 + SCOPETREE`。
- `B2` 范围状态基：frontend.L1.M12[R2] + frontend.L1.M15[R1]。由当前路径、当前项提示和状态标记约束成当前范围与切换后的范围状态结构。来源：`C2 + SCOPESTATE`。
- `B3` 局部工具位基：frontend.L1.M4[R1,R2] + frontend.L1.M9[R2] + frontend.L1.M15[R1]。由动作入口、局部承接面和状态标记约束成范围树附近的刷新、上传、局部目录动作与当前作用范围结构。来源：`C3 + LOCALACTIONS`。
- `B4` 局部信息基：frontend.L1.M9[R1,R2] + frontend.L1.M15[R1,R2]。由展示容器、摘要文本和局部附着反馈约束成当前范围下的统计项、摘要项与局部反馈结构。来源：`C4 + LOCALSUMMARY`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 范围树必须导出当前范围
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：在 frontend 的集合浏览结构上，范围树必须同时导出当前节点、当前路径和当前范围状态；不能只展示层级而不给出当前范围状态。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`SCOPETREE/SCOPESTATE`。

- `R2` 局部工具位必须附着当前范围
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：在 frontend 的触发入口、承接面和状态结构上，范围树附近的局部工具位必须附着于当前范围，而不是作为无作用范围的独立按钮存在。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`SCOPESTATE/LOCALACTIONS`。

- `R3` 局部信息必须附着当前范围
  - `R3.1` 参与基：`B2 + B4`。
  - `R3.2` 组合方式：在 frontend 的展示容器和附着反馈结构上，局部信息中的统计与摘要必须附着于当前范围，而不是作为无来源的独立数字存在。
  - `R3.3` 输出能力：`C2 + C4`。
  - `R3.4` 边界绑定：`SCOPESTATE/LOCALSUMMARY`。

- `R4` 范围限定原子闭环
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：范围树、当前范围、局部工具位与局部信息必须共同形成“选中范围 -> 确认当前范围 -> 使用局部工具位 / 查看局部信息”的稳定结构。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`SCOPETREE/SCOPESTATE/LOCALACTIONS/LOCALSUMMARY`。

- `R5` 无效方案
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：以下方案无效：跳过 frontend 集合浏览直接声明范围树；只有树没有当前范围；局部工具位存在但无法说明作用于哪个当前范围；只有统计卡没有范围来源；切换范围后局部工具位或局部信息无法解释对应关系。
  - `R5.3` 失效结论：`N1`。
  - `R5.4` 边界绑定：`SCOPETREE/SCOPESTATE/LOCALACTIONS/LOCALSUMMARY`。

## 5. 验证（Verification）

- `V1` 范围可识别性：范围树中必须能识别唯一当前节点，并导出唯一当前范围与对应路径。
- `V2` 局部工具位可追溯性：任一局部工具位都必须能说明附着的当前范围与作用范围。
- `V3` 局部信息可追溯性：任一局部信息都必须能说明附着的当前范围与统计来源。
- `V4` 前端承接有效性：范围树、当前范围、局部工具位和局部信息必须能追溯到 frontend 的集合浏览、触发入口、展示容器和附着反馈结构。
- `V5` 最小必要性：移除任一 `B*` 后，`V1~V4` 至少一项失效。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
