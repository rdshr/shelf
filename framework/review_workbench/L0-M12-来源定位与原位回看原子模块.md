# 来源定位与原位回看原子模块:ReviewWorkbenchSourceLocationAndInPlaceInspectionAtoms

@framework

本模块定义 `review_workbench` 中最小的来源定位与原位回看结构。推导链为：`framework/review_workbench/L0-M4-对象列表与查看入口共享底座原子模块.md` + `framework/review_workbench/L0-M5-全局反馈与状态汇总原子模块.md` -> 本模块。也就是说，从当前对象、当前块或当前结果进入源文件、定位到原位并回看来源内容这组结构，不单独发明新的查看真相，而是在通用查看入口与反馈结构之上，收束成来源定位与原位回看这组领域原子。

本模块位于 `review_workbench` 的原子层。它负责把来源文件、来源定位、原位回看承接面和定位反馈表达清楚，回答“当前来源查看来自哪个对象或块”“如何在前端定位到源文件中的原位”“看完之后如何保持当前上下文可解释”这几个最小问题；它不替代文件预览主工作面，也不直接承担历史差异回看。

它不是悬空的“来源查看子系统”，而是明确建立在现有 `review_workbench` 共享原子之上：`L0-M4` 提供来源入口、对象来源与查看锚点的共享对象底座，`L0-M5` 提供来源加载与定位后的可观察反馈底座；而当前对象、当前块和当前处理页签上下文则由上层编排模块继续承接。本模块是在这些既有结构上，把“来源定位与原位回看”沉成领域专有原子，而不是另起一套并行查看真相源。凡是“来源查看请求怎样发、查看内容怎样返回、锚点怎样编码”的问题，统一交给外部领域 `review_workbench_backend.L0.M2`。

## 1. 能力声明（Capability Statement）
- `C1` 来源查看入口表达能力：将从当前对象或当前块进入来源查看的入口组织成稳定结构。
- `C2` 来源定位表达能力：将当前对象或块映射到源文件中的原位、片段或页内位置组织成稳定结构。
- `C3` 原位回看承接面表达能力：将源文件回看面、当前定位锚点和回退上下文组织成稳定结构。
- `C4` 来源定位反馈表达能力：将来源加载、定位成功、定位失败或定位变更反馈组织成稳定结构。
- `N1` 非职责声明：不定义具体文件预览控件，不替代文件预览与分块处理主工作面，不承担审核结论或历史差异语义。

## 2. 边界定义（Boundary / 参数）
- `SOURCE_ENTRY_SCOPE` 来源入口边界：入口种类和进入方式允许变化，但任一次来源查看都必须说明来源对象及当前进入上下文，并能回接到当前对象或当前块来源。
- `SOURCE_LOCATION_SCOPE` 来源定位边界：坐标形式、片段边界和锚点类型允许变化，但定位后必须能解释当前来源文件与当前定位锚点。
- `SOURCE_VIEW_SCOPE` 来源回看边界：承接面形式和回退方式允许变化，但进入来源面后必须能解释当前定位内容及回退上下文。
- `SOURCE_FEEDBACK_SCOPE` 来源反馈边界：加载状态、成功提示和失败提示形式允许变化，但来源加载或定位后必须至少出现一处可观察反馈。

## 3. 最小可行基（Minimum Viable Bases）
- `B1` 来源入口与对象附着基：`L0.M4[R2,R3]`。来源：`C1 + SOURCE_ENTRY_SCOPE`。
- `B2` 来源定位与查看锚点基：`L0.M4[R3,R4]`。来源：`C2 + SOURCE_LOCATION_SCOPE`。
- `B3` 原位回看承接面基：`L0.M4[R3,R4] + L0.M5[R1]`。来源：`C3 + SOURCE_VIEW_SCOPE`。
- `B4` 来源反馈基：`L0.M5[R2,R3]`。来源：`C4 + SOURCE_FEEDBACK_SCOPE`。

## 4. 基组合原则（Base Combination Principles）
- `R1` 来源查看必须说明来源对象或来源块
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：任何来源查看都必须能说明当前是由哪个对象或哪个块进入，并说明该对象或块映射到源文件中的哪个原位；不能只有一个来源抽屉而没有来源上下文。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`SOURCE_ENTRY_SCOPE/SOURCE_LOCATION_SCOPE`。
- `R2` 原位回看必须附着稳定定位锚点
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：来源回看承接面必须带出当前定位锚点、定位片段和回退上下文；不能只有源文件内容而无法说明当前看的是源文件哪里。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`SOURCE_LOCATION_SCOPE/SOURCE_VIEW_SCOPE`。
- `R3` 来源定位之后必须有反馈
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：来源加载、定位成功或定位失败之后，至少一处来源反馈区必须给出可解释变化；不能让定位状态只停留在内部。
  - `R3.3` 输出能力：`C3 + C4`。
  - `R3.4` 边界绑定：`SOURCE_VIEW_SCOPE/SOURCE_FEEDBACK_SCOPE`。
- `R4` 来源定位与原位回看闭环
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：来源查看入口、来源定位、原位回看承接面和来源反馈必须共同组成“由当前对象/块进入来源 -> 定位原位 -> 查看来源内容 -> 看到定位反馈”的稳定闭环；其中对象来源与查看锚点继续复用 `L0-M4`，反馈继续复用 `L0-M5`，不得在本模块中重写并行查看底座。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`SOURCE_ENTRY_SCOPE/SOURCE_LOCATION_SCOPE/SOURCE_VIEW_SCOPE/SOURCE_FEEDBACK_SCOPE`。
- `R5` 无效方案
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：以下方案无效：只有来源查看按钮没有来源对象；只有源文件内容没有当前原位；能够打开来源面但无法说明当前定位锚点；定位成功或失败后没有任何可观察反馈；`L0-M12` 自己重写了一套脱离 `L0-M4/L0-M5` 的并行来源查看底座。
  - `R5.3` 失效结论：`N1`。
  - `R5.4` 边界绑定：`SOURCE_ENTRY_SCOPE/SOURCE_LOCATION_SCOPE/SOURCE_VIEW_SCOPE/SOURCE_FEEDBACK_SCOPE`。

## 5. 验证（Verification）
- `V1` 来源入口有效性：任一来源查看都必须能追溯到当前对象或当前块来源。
- `V2` 原位定位有效性：任一来源回看都必须能说明当前来源文件与当前定位锚点。
- `V3` 来源反馈可见性：任一来源加载或定位动作之后，必须至少有一处反馈发生可解释变化。
- `V4` 共享原子承接有效性：来源入口、定位锚点和反馈承接必须能分别追溯到 `L0-M4` 与 `L0-M5`，而不是在本模块中重写并行查看底座。
- `V5` 最小必要性：移除任一 `B*` 后，`V1~V4` 至少一项失效。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
