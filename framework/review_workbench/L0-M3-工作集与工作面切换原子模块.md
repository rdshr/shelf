# 工作集与工作面切换原子模块:ReviewWorkbenchWorksetAndWorkspaceSwitchAtoms

@framework

本模块定义 `review_workbench` 领域最小的工作集与工作面切换结构。推导链为：`framework/frontend/L1-M0-触发与选择原子模块.md` + `framework/frontend/L1-M3-集合与导航原子模块.md` + `framework/frontend/L1-M4-标记与反馈原子模块.md` -> 本模块。也就是说，平台中的固定列表页签、处理页签、审核页签、历史查看页签和关闭回退关系，不单独重写页签基础，而是把前端已有的集合导航、离散切换和状态提示约束成“工作集与工作面切换”这组领域原子。

本模块定义统一审查平台中的工作页签与工作面切换结构原子。它回答“当前工作台里有哪些列表页签、详情页签或回看页签”以及“用户此刻正在处理哪一个工作面内容”这两个最小问题。它不定义对象字段和动作语义，只负责把固定页签、详情页签、多任务面和打开来源沉成稳定切换结构。

## 1. 能力声明（Capability Statement）

- `C1` 工作集集合表达能力：将固定列表页签、已打开的详情页签、审核页签、历史页签或其它工作面集合组织成稳定结构。
- `C2` 当前工作面表达能力：将当前激活工作面、工作面类型、切换结果与关闭后的回退关系组织成稳定状态。
- `N1` 非职责声明：不负责对象字段、不负责动作语义、不负责统计反馈。

## 2. 边界定义（Boundary / 参数）

- `WORKSETSET` 工作集集合边界：固定列表页签、已打开项和可切换项的数量允许变化，但必须共同落在同一工作集集合中，并能识别当前可切换项集合。
- `WORKSETSTATE` 工作面状态边界：当前激活项、工作面类型、切换结果和关闭回退关系允许变化，但任一时刻必须存在唯一当前工作面，且关闭后回退结果必须可解释。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 工作集集合基：frontend.L1.M3[R1,R2]。由集合结构、当前项和导航出口约束成固定列表页签、已打开详情页签与工作面切换入口结构。来源：`C1 + WORKSETSET`。
- `B2` 工作面状态基：frontend.L1.M0[R1,R2] + frontend.L1.M3[R2] + frontend.L1.M4[R1]。由离散切换入口、当前选中项和状态标记约束成当前激活工作面、工作面类型、切换结果与关闭回退关系。来源：`C2 + WORKSETSTATE`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 工作集集合必须导出当前工作面
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：在 frontend 的集合导航与触发选择结构上，工作集集合必须能够导出当前激活项及其工作面类型，不能只有打开列表而没有当前内容。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`WORKSETSET/WORKSETSTATE`。

- `R2` 工作面切换结果必须可解释
  - `R2.1` 参与基：`B2`。
  - `R2.2` 组合方式：在 frontend 的离散动作和状态提示结构上，切换或关闭工作面后，系统必须能解释当前回到了列表页签、审核页签、历史页签还是某个详情页签，以及为什么回到它。
  - `R2.3` 输出能力：`C2`。
  - `R2.4` 边界绑定：`WORKSETSTATE`。

- `R3` 无效方案
  - `R3.1` 参与基：`B1 + B2`。
  - `R3.2` 组合方式：以下方案无效：跳过 frontend 集合导航直接声明工作集；只有标签可见但无当前项；列表页签、详情页签和审核/历史工作面不可区分；关闭当前标签后当前内容不可解释。
  - `R3.3` 失效结论：`N1`。
  - `R3.4` 边界绑定：`WORKSETSET/WORKSETSTATE`。

## 5. 验证（Verification）

- `V1` 当前工作面可识别性：任一时刻都必须能识别唯一当前激活工作面及其类型。
- `V2` 切换可解释性：切换或关闭后必须能解释当前回到哪一个工作面，以及为什么回到它。
- `V3` 前端承接有效性：工作集集合与状态必须能追溯到 frontend 的触发选择、集合导航和标记反馈结构。
- `V4` 最小必要性：移除任一 `B*` 后，`V1~V3` 至少一项失效。
- `V5` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
