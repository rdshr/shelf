# 文档分块模块:DocumentChunkingModule

@framework

## 1. 能力声明（Capability Statement）

- `C1` 文档类型识别与结构读取能力：提供对受支持文档类型的识别，并按类型读取章节结构与正文顺序的能力。
- `C2` 结构锚点识别能力：提供按标题或等价结构锚点识别分块依据的能力。
- `C3` 分块边界生成能力：提供基于结构锚点生成有序内容块边界的能力。
- `C4` 分块结果校验能力：提供块完整性、块顺序与块标识可追踪性的校验能力。
- `C5` 非能力项：不负责 `Tag` 标注、数据提取、结果对比与参数调整。

## 2. 边界定义（Boundary / 参数）

- `FD` 文档类型边界：仅处理 `SupportedDocTypes` 中的文档类型。来源：`C1`。
- `FR` 文档类型识别边界：每个输入文档在分块前必须被识别为唯一且受支持的文档类型。来源：`C1`。
- `HB` 结构依据边界：分块必须基于已识别文档类型下可解释的标题或等价结构锚点。来源：`C1 + C2`。
- `B` 分块数量边界：输出内容块数量必须满足 `B > 1`。来源：`C2 + C3`。
- `ORDER` 顺序边界：内容块顺序必须与原文档结构锚点顺序一致。来源：`C1 + C2 + C3`。
- `COMPLETE` 完整性边界：每个内容块必须完整覆盖其所属结构锚点范围内的正文内容，不得跨块丢失或重复。来源：`C3 + C4`。
- `TRACE` 追踪边界：每个内容块必须绑定稳定的 `block_id` 与来源结构锚点。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文档类型识别与结构读取基。来源：`C1 + FD + FR + ORDER`。
- `B2` 结构锚点识别基。来源：`C2 + HB + ORDER`。
- `B3` 分块边界生成基。来源：`C3 + HB + B + COMPLETE`。
- `B4` 分块结果校验基。来源：`C4 + B + COMPLETE + TRACE + ORDER`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 类型结构锚点组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先识别输入文档类型，再按该类型对应的结构规则读取正文顺序并识别可作为分块依据的标题或等价结构锚点；类型未识别、识别结果不唯一或锚点无法解释时不得进入下游分块。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`FD/FR/HB/ORDER`。
- `R2` 锚点分块组合
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：每个结构锚点只能生成一个顺序确定的内容块边界，块数量必须满足 `B > 1`，且块边界不得交叉。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`HB/B/ORDER/COMPLETE`。
- `R3` 分块校验组合
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：对已生成内容块逐一校验完整性、顺序一致性与 `block_id` 追踪信息；缺失任一校验项则该块集合无效。
  - `R3.3` 输出能力：`C3 + C4`。
  - `R3.4` 边界绑定：`B/COMPLETE/TRACE/ORDER`。
- `R4` 文档分块闭环组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：将文档类型识别与结构读取、结构锚点识别、分块边界生成与结果校验收敛为可供下游标注直接消费的有序块集合。
  - `R4.3` 输出能力：`C1 + C2 + C3 + C4`。
  - `R4.4` 边界绑定：`FD/FR/HB/B/ORDER/COMPLETE/TRACE`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：文档类型不受支持、类型无法识别或识别结果不唯一、缺失结构锚点、输出仅单块、块顺序错乱、块内容重叠/缺失或缺失 `block_id` 的方案无效。
  - `R5.3` 输出能力：`C5`。
  - `R5.4` 边界绑定：`FD/FR/HB/B/ORDER/COMPLETE/TRACE`。

## 5. 验证（Verification）

- `V1` 文档类型识别有效性：输入文档必须满足 `FD` 约束，且识别结果必须满足 `FR` 约束。
- `V2` 结构锚点有效性：分块依据必须满足 `HB` 约束且顺序稳定。
- `V3` 分块数量合法性：输出块数量必须满足 `B > 1`。
- `V4` 分块完整性：所有内容块必须满足 `COMPLETE` 与 `ORDER` 约束。
- `V5` 结果可追踪性：所有内容块必须满足 `TRACE` 约束并可供下游绑定。
- `V6` 最小必要性：移除任一 `B*` 后，`V2~V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
