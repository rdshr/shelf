# 知识库工作台场景模块:KnowledgeBaseWorkbenchScenarios

@framework

本模块是知识库领域通用框架的场景层。它建立在知识库原子结构和知识引用上下文结构之上，同时默认由 frontend `L1` 的触发选择、文本输入、展示容器、集合导航和标记反馈原子族提供更基础的渲染承接结构，而这些结构又建立在 frontend `L0` 基础原子层之上。项目实例应把本模块编译成显式 workbench contract、上下文回路和规则校验，而不是在实例配置中拆散这条主链。默认工作台形态是 chat-first：左侧承接历史会话和知识库入口，中部承接对话，来源细节通过按需打开的引用抽屉与辅助详情页展开。实例配置采用“大类边界 -> 大类 section”约定：`LIBRARY -> [library]`、`PREVIEW -> [preview]`、`CHAT -> [chat]`、`CONTEXT -> [context]`、`RETURN -> [return]`；界面外形与布局 recipe 继续由 frontend 的 `[surface]/[visual]/[route]/[a11y]` 承接。对应的 `product_spec.toml` 与 `implementation_config.toml` 应默认使用中文详细注释，使知识库产品讨论、引用回路审查和实现细化在同一份可读文档中对齐。在 `Framework -> Product Spec -> Implementation Config -> Code` 分层中，本模块只负责知识库工作台场景的共同结构语言；具体产品真相应先收敛为 `Product Spec`，再由 `Implementation Config` 细化到某种实现路径。

## 1. 能力声明（Capability Statement）

- `C1` chat-first 工作台建模能力：将会话侧栏、知识库入口、对话主舞台与引用来源收敛为统一工作台场景。
- `C2` 上下文衔接能力：统一当前文件、当前章节、引用域与回合上下文在场景间的传递结构。
- `C3` 引用对话闭环能力：支持上传文件、选择引用域、基于知识库对话并回看引用来源。
- `C4` 受保护工作台能力：支持在不改写知识库工作台主链的前提下，对受保护页面施加登录前置条件、登录承接面与登录后返回回路。
- `C5` 非能力项：不负责前端底座实现、后端索引算法、身份提供商 SDK 与模型本身推理质量。

## 2. 边界定义（Boundary / 参数）

- `SURFACE` 场景边界：至少覆盖会话侧栏、对话主舞台、引用抽屉与辅助详情页；其布局 recipe 由 frontend `[surface]` section 承接。来源：`C1 + C2`。
- `LIBRARY` 文件库边界：创建、移除、聚焦、筛选与选择引用域动作需统一。来源：`C1 + C3`。
- `PREVIEW` 预览边界：引用抽屉、文档正文、TOC 导航与章节焦点切换需稳定。来源：`C1 + C2`。
- `CHAT` 对话边界：输入、回合顺序、引用附着与回看路径需稳定。来源：`C1 + C3`。
- `CONTEXT` 上下文边界：当前文件、当前章节、当前引用域与当前回合需稳定传递。来源：`C2 + C3`。
- `RETURN` 返回边界：从引用回合返回文件、目录与文档锚点的路径需明确；返回卡片变体与 anchor restore 策略由 `[return]` section 承接。来源：`C2 + C3`。
- `AUTH` 身份边界：登录是否为前置条件、哪些页面受保护、登录承接面与登录后返回目标需稳定。来源：`C1 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话与知识入口场景基：L1.M0[R1,R2]。来源：`C1 + SURFACE + LIBRARY + PREVIEW`。
- `B2` 对话引用场景基：L1.M0[R1,R3] + L1.M1[R1,R2]。来源：`C1 + CHAT + CONTEXT + RETURN`。
- `B3` chat-first 客户端闭环场景基：L1.M0[R3] + L1.M1[R2,R3]。来源：`C1 + C2 + C3 + SURFACE + CONTEXT + RETURN`。
- `B4` 受保护工作台承接基：L1.M0[R1] + auth.L2.M0[R2,R3,R4]。来源：`C1 + C4 + AUTH + SURFACE`。
  说明：本基承接的是 auth 子域已经收敛好的 `SESSIONMODEL / GUARDMODEL / RETURNMODEL / FEEDBACKMODEL` 标准框架边界在工作台入口上的落点，而不是让 knowledge_base 实例重新以场景字段发明认证配置；它只承接工作台访问前的认证守卫、登录后目标恢复与失效提示，不承接知识引用上下文、来源锚点与 citation return 回路。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话知识主链
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定会话侧栏、知识库入口和来源详情展开之间的主链关系，再允许 library 与 preview 大类 section 中的小类字段在该主链内变化。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SURFACE/LIBRARY/PREVIEW`。
- `R2` 对话引用并轨
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在会话与来源主链上接入对话回合、引用域、引用抽屉与文档详情返回路径，形成知识库对话场景；chat、context 与 return 大类 section 只能在既定闭环内取值。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`CHAT/CONTEXT/RETURN/LIBRARY/PREVIEW`。
- `R3` 客户端上下文闭环
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：联合会话与来源主链、对话引用并轨和上下文回路，形成稳定知识库 chat-first 客户端场景。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SURFACE/LIBRARY/PREVIEW/CHAT/CONTEXT/RETURN`。
- `R4` 受保护工作台守卫
  - `R4.1` 参与基：`B1 + B4`。
  - `R4.2` 组合方式：在不改写 chat-first 主链和知识引用闭环的前提下，将 auth.L2.M0 已收敛的认证保护、会话、返回恢复与反馈边界承接到工作台入口；未登录或会话失效时切入认证入口，认证完成后恢复工作台访问目标；认证语义只决定工作台访问放行与恢复，不进入知识引用上下文编排，也不改写 citation return 回路。knowledge_base 实例应通过 auth 子域的大类 section 接入标准框架，而不是直接以 auth 场景流名词作为实例开放边界。
  - `R4.3` 输出能力：`C1 + C4`。
  - `R4.4` 边界绑定：`SURFACE/AUTH/LIBRARY/CHAT`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：缺失会话侧栏、引用抽屉与对话主舞台的稳定关系，或将认证前置条件反向改写产品主链，或绕过 auth.L2.M0 已定义的标准框架边界，在知识库实例里重新散写认证真相，或将登录回跳与 citation return 混写为同一返回语义的方案均无效。
  - `R5.3` 输出能力：`C5`。
  - `R5.4` 边界绑定：`SURFACE/LIBRARY/PREVIEW/CHAT/CONTEXT/RETURN/AUTH`。

## 5. 验证（Verification）

- `V1` 场景完整性：会话侧栏、引用抽屉、辅助详情页与对话区必须全部存在。
- `V2` 上下文一致性：当前文件、当前章节、引用域与回合上下文必须可追踪且一致。
- `V3` 闭环有效性：知识库切换、对话引用与返回来源路径必须形成稳定闭环。
- `V4` 身份守卫承接有效性：当产品声明受保护工作台时，未登录访问受保护页面必须被拦截，并沿 auth.L2.M0 已定义的 `[auth.entry] / [auth.session] / [auth.guard] / [auth.return] / [auth.feedback] / [auth.service]` 收敛边界进入稳定回路；认证完成后恢复的是工作台访问目标，而不是知识引用返回目标。
- `V5` 最小必要性：移除任一 `B*` 后，`V1~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
- `V7` 配置归属稳定性：`[library]/[preview]/[chat]/[context]/[return]/[auth]` 中的字段必须严格归属于对应边界大类；其中 `[auth]` 应以 `[auth.entry] / [auth.session] / [auth.identity] / [auth.guard] / [auth.return] / [auth.feedback] / [auth.service]` 的标准框架大类开放实例真相，只承接工作台访问前的认证入口、会话放行、主体表达、登录后目标恢复与失效提示，不得承接知识引用、来源锚点、citation drawer 或 document detail 的返回回路；`[return]` 只承接知识引用后的来源返回与 anchor restore，不得承接登录回跳、受保护目标放行或会话失效处理。
