# 会话与记载原子模块:SessionAndLedgerAtoms

@framework

本模块定义多渠道助手里“会话线”和“记载页”最底层要长成什么样。它只回答 5 件事：会话主轴怎么立、当前页怎么放、历史页怎么放、沉淀页怎么放、常驻规则页怎么放；不负责具体渠道接入和推理策略。

## 1. 能力声明（Capability Statement）

- `C1` 会话主轴抽象能力：定义一次会话沿时间展开时的稳定主线。
- `C2` 记载落页抽象能力：定义过程记录和沉淀记录分别落到哪些页上。
- `C3` 常驻规则抽象能力：定义长期存在的身份、规则和作用域放在哪些页上。
- `C4` 非能力项：不负责模型选择、渠道协议和时间触发策略。

## 2. 边界定义（Boundary / 参数）

- `CONTINUITY` 连续性边界：同一会话只能沿一条主轴延展，分支只能挂回主轴，不得漂成孤链。来源：`C1 + C2`。
- `APPEND` 追加边界：历史条目和沉淀条目只能追加，不得原位覆盖既有条目。来源：`C2`。
- `INDEX` 索引边界：任一条目都必须能通过稳定索引位回到原位。来源：`C1 + C2`。
- `SCOPE` 作用域边界：当前工作单元可见的常驻规则和页组范围必须显式界定。来源：`C1 + C3`。
- `REFRESH` 刷新边界：沉淀写入只能发生在声明过的刷新点，不得随意插入。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话主轴基：由起点锚、回合链和分支挂点构成。来源：`C1 + CONTINUITY + INDEX`。
  说明：先把这段会话的主线立起来，后面的页和分支都挂在这条线上。
- `B2` 当前账页基：由当前可见带和当前写回口构成。来源：`C2 + INDEX + SCOPE`。
  说明：当前回合直接要读、直接要写的内容先落在这一页。
- `B3` 历史账页基：由追加记录带和索引位构成。来源：`C2 + APPEND + INDEX`。
  说明：原始过程记录沿这条历史页一直往后追加。
- `B4` 沉淀账页基：由稳定事实带和刷新点构成。来源：`C2 + APPEND + REFRESH`。
  说明：整理后的稳定内容单独落在沉淀页，不和原始过程混写。
- `B5` 常驻规则页基：由身份页、规则页和作用域页构成。来源：`C3 + SCOPE + CONTINUITY`。
  说明：长期不跟着单个回合变化的身份、规则和可见范围都放在这里。

## 4. 基组合原则（Base Combination Principles）

- `R1` 会话主轴成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定起点锚、回合链和分支挂点，形成可持续延展的会话主轴。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`CONTINUITY/INDEX`。
- `R2` 当前与历史账页成立
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：把当前账页和历史账页挂到同一会话主轴上，形成可读、可追溯的过程记载面。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`CONTINUITY/APPEND/INDEX/SCOPE`。
- `R3` 沉淀与常驻页成立
  - `R3.1` 参与基：`B1 + B4 + B5`。
  - `R3.2` 组合方式：把沉淀账页和常驻规则页挂到同一会话主轴上，形成可长期带着走的稳定页组。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`CONTINUITY/REFRESH/SCOPE`。
- `R4` 记载面成立
  - `R4.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R4.2` 组合方式：把当前账页、历史账页、沉淀账页和常驻规则页一起挂到同一会话主轴上，形成完整会话记载面。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`CONTINUITY/APPEND/INDEX/SCOPE/REFRESH`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：主轴断裂、历史页可原位覆盖、沉淀页绕过刷新点或常驻规则页越出声明作用域的组合无效。
  - `R5.3` 输出能力：`C4`。
  - `R5.4` 边界绑定：`CONTINUITY/APPEND/INDEX/SCOPE/REFRESH`。

## 5. 验证（Verification）

- `V1` 主轴有效性：会话主轴必须满足 `CONTINUITY/INDEX` 约束。
- `V2` 过程页有效性：当前账页和历史账页必须满足 `APPEND/INDEX/SCOPE` 约束。
- `V3` 稳定页有效性：沉淀账页和常驻规则页必须满足 `REFRESH/SCOPE` 约束。
- `V4` 记载面有效性：同一会话主轴上的各页必须能共同支撑一次稳定装载和一次稳定写回。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
