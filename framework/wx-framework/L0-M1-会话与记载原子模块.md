# 会话与记载原子模块:SessionAndLedgerAtoms

@framework

本模块定义多渠道助手框架中的会话主轴、双层记载账页和常驻约束册页结构。它只回答“工作单元依附在哪条主轴上、记载落在哪些页里、常驻约束以什么册页存在”，不负责具体推理策略与渠道接入。

## 1. 能力声明（Capability Statement）

- `C1` 会话主轴抽象能力：定义会话沿时间展开时的稳定主轴结构。
- `C2` 记载账页抽象能力：定义历史与沉淀条目落入账页时的稳定结构。
- `C3` 常驻约束抽象能力：定义身份、规则与作用域常驻存在时的稳定册页结构。
- `C4` 非能力项：不负责具体模型选择、渠道协议和时间触发策略。

## 2. 边界定义（Boundary / 参数）

- `CONTINUITY` 连续性边界：同一会话必须沿一条稳定主轴延展，分支只能挂回主轴，不得漂移成孤立链。来源：`C1 + C2`。
- `APPEND` 追加边界：记载条目只能追加到账页，不得原位改写既有条目。来源：`C2`。
- `INDEX` 索引边界：任一条目必须能通过稳定索引位或引用位回到原位。来源：`C1 + C2`。
- `SCOPE` 作用域边界：当前工作单元可见的账页切片和常驻册页必须显式界定。来源：`C1 + C3`。
- `REFRESH` 刷新边界：沉淀、归档和写回只能落在声明的刷新点，不得任意插入账页。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 会话主线基：由会话锚点、回合链和分支挂点构成。来源：`C1 + CONTINUITY + INDEX`。
  说明：一段会话先有一条主线，后面的回合和分支都只能挂在这条线上。
- `B2` 会话记录页基：由时序记录带、沉淀记录带和索引位构成。来源：`C2 + APPEND + INDEX + REFRESH`。
  说明：会话记录分成两页，一页放原始过程，一页放整理后的稳定记录。
- `B3` 常驻规则页基：由身份页、规则页和作用域页构成。来源：`C3 + SCOPE + CONTINUITY`。
  说明：代理长期带着走的身份、规则和可见范围，统一放在这组规则页里。

## 4. 基组合原则（Base Combination Principles）

- `R1` 主轴成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定会话锚点、回合链和分支挂点，形成可持续延展的会话主轴。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`CONTINUITY/INDEX`。
- `R2` 册页挂载
  - `R2.1` 参与基：`B1 + B3`。
  - `R2.2` 组合方式：将身份页、规则页和作用域页挂到已建立的会话主轴上，形成可装载的常驻册页面。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`CONTINUITY/SCOPE`。
- `R3` 记载闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将双层记载账页和常驻约束册页共同挂到同一会话主轴上，形成可读、可写、可沉淀的会话记载面。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`CONTINUITY/APPEND/INDEX/SCOPE/REFRESH`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：主轴断裂、账页可被原位改写、条目不可索引或常驻册页越出声明作用域的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`CONTINUITY/APPEND/INDEX/SCOPE/REFRESH`。

## 5. 验证（Verification）

- `V1` 主轴有效性：会话主轴必须满足 `CONTINUITY/INDEX` 约束。
- `V2` 账页有效性：双层记载账页必须满足 `APPEND/INDEX/REFRESH` 约束。
- `V3` 册页有效性：常驻约束册页必须满足 `SCOPE` 约束。
- `V4` 记载闭合性：同一会话主轴上的账页和册页必须可共同支撑一次稳定装载与一次稳定写回。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
