# 接入插口与执行插槽原子模块:PortAndExecutionSlotAtoms

@framework

本模块定义多渠道助手里“从哪进、从哪出、在哪工作、从哪被唤醒”这些最底层位置件。它只回答 5 件事：入站插口、出站插口、当前回合槽位、后台续跑槽位、触发窗口怎么立；不负责具体厂商、协议库和实例值。

## 1. 能力声明（Capability Statement）

- `C1` 接入插口抽象能力：定义外部进入和离开系统时的固定口子。
- `C2` 工作槽位抽象能力：定义系统内部当前回合和后台续跑分别在哪些槽位里工作。
- `C3` 触发窗口抽象能力：定义后台计划、唤醒和恢复落在哪些窗口上。
- `C4` 非能力项：不负责渠道名单、提供商名单、权限实例值和桥接实现细节。

## 2. 边界定义（Boundary / 参数）

- `BIND` 绑定位边界：每个插口、槽位或窗口都必须落在一个明确绑定位上，不得悬空。来源：`C1 + C2 + C3`。
- `GATE` 门控边界：每个插口和槽位都必须有显式允许条件或拒绝条件，不得裸露开放。来源：`C1 + C2`。
- `HANDOFF` 交接边界：相邻插口、槽位和窗口之间交接的载荷外形必须一致，不得中途变形。来源：`C1 + C2 + C3`。
- `TRIGGER` 触发边界：每个触发窗口只能对应一种声明过的唤醒来源或计划形式。来源：`C3`。
- `RESUME` 恢复边界：后台续跑离开当前槽位后，必须有显式停止位、重试位或恢复位。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 入站插口基：由入站口和绑定位面构成。来源：`C1 + BIND + GATE + HANDOFF`。
  说明：外部消息从哪进来，先在这里定住。
- `B2` 出站插口基：由出站口和绑定位面构成。来源：`C1 + BIND + GATE + HANDOFF`。
  说明：系统结果从哪送出去，也要单独定住。
- `B3` 当前回合槽位基：由推理槽和动作槽构成。来源：`C2 + BIND + GATE + HANDOFF`。
  说明：当前回合里要做的事，主要都落在这组槽位上。
- `B4` 后台续跑槽位基：由延续槽和任务槽构成。来源：`C2 + BIND + GATE + RESUME`。
  说明：脱离当前回合后继续往下跑的工作，落在这组后台槽位上。
- `B5` 触发窗口基：由计划窗、唤醒窗和恢复窗构成。来源：`C3 + TRIGGER + RESUME + BIND`。
  说明：后台什么时候开始、何时重试、从哪里恢复，都由这组窗口负责。

## 4. 基组合原则（Base Combination Principles）

- `R1` 出入口成立
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定入站插口和出站插口，形成一组受门控的基本出入口。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`BIND/GATE/HANDOFF`。
- `R2` 当前回合槽位成立
  - `R2.1` 参与基：`B3`。
  - `R2.2` 组合方式：把推理槽和动作槽接成当前回合可用的工作槽位。
  - `R2.3` 输出能力：`C2`。
  - `R2.4` 边界绑定：`BIND/GATE/HANDOFF`。
- `R3` 后台续跑槽位成立
  - `R3.1` 参与基：`B4`。
  - `R3.2` 组合方式：把延续槽和任务槽接成后台续跑可用的工作槽位。
  - `R3.3` 输出能力：`C2`。
  - `R3.4` 边界绑定：`BIND/GATE/RESUME`。
- `R4` 触发窗口成立
  - `R4.1` 参与基：`B5`。
  - `R4.2` 组合方式：把计划窗、唤醒窗和恢复窗接成一组完整触发窗口。
  - `R4.3` 输出能力：`C3`。
  - `R4.4` 边界绑定：`BIND/TRIGGER/RESUME`。
- `R5` 交接面闭合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R5.2` 组合方式：让出入口、当前回合槽位、后台续跑槽位和触发窗口以同形载荷完成稳定交接。
  - `R5.3` 输出能力：`C1 + C2 + C3`。
  - `R5.4` 边界绑定：`BIND/GATE/HANDOFF/TRIGGER/RESUME`。
- `R6` 禁止组合
  - `R6.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R6.2` 组合方式：绑定位悬空、门控缺失、交接外形断裂、触发窗口来源不明或续跑后没有恢复落点的组合无效。
  - `R6.3` 输出能力：`C4`。
  - `R6.4` 边界绑定：`BIND/GATE/HANDOFF/TRIGGER/RESUME`。

## 5. 验证（Verification）

- `V1` 出入口有效性：入站插口和出站插口必须满足 `BIND/GATE/HANDOFF` 约束。
- `V2` 当前槽位有效性：当前回合槽位必须满足 `BIND/GATE/HANDOFF` 约束。
- `V3` 后台槽位有效性：后台续跑槽位必须满足 `BIND/GATE/RESUME` 约束。
- `V4` 触发窗口有效性：触发窗口必须满足 `TRIGGER/RESUME` 约束。
- `V5` 交接面有效性：出入口、槽位和窗口之间必须能以同形载荷完成稳定交接。
- `V6` 最小必要性：移除任一 `B*` 后，`V2~V5` 至少一项失败。
- `V7` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
