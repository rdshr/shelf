# 多渠道个人助手框架标准模块:MultiChannelPersonalAssistantFramework

@framework

本模块定义消息总线驱动的多渠道个人助手共同结构语言。它只负责 5 块结构怎么接成一套：渠道收消息入口、当前回合工作面、会话落账面、渠道发消息出口、后台续跑面；具体产品人格、启用渠道、提供商选择、权限实例值和工作区文案都应落到 `Product Spec` 与 `Implementation Config`。

## 1. 能力声明（Capability Statement）

- `C1` 多渠道收发能力：把不同承载面的消息收进来、做完后再发出去。
- `C2` 即时回合能力：把当前回合的装载、处理和写回接成一条稳定工作线。
- `C3` 后台续跑能力：把脱离当前回合的工作挂成一条可续跑、可回投的侧线。
- `C4` 非能力项：不负责具体渠道名单、提供商名单、产品人格真相和领域知识内容。

## 2. 边界定义（Boundary / 参数）

- `MAINLINE` 主线边界：收消息、当前回合处理、会话落账和发消息必须沿同一条前台主线顺序接起。来源：`C1 + C2`。
- `CHANNELDIFF` 渠道差异边界：渠道差异只能留在渠道线层，不得改写当前回合工作面和后台续跑面。来源：`C1 + C2`。
- `SESSIONBACK` 落账边界：任一当前回合结果或后台结果都必须落回唯一会话页组。来源：`C2 + C3`。
- `ASYNC` 续跑边界：后台续跑只能作为主线侧边的独立续跑线存在，不得直接替代前台主线。来源：`C3`。
- `TRACE` 追踪边界：任一消息和任务路径都必须能从入口一路回放到出口或回投口。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 渠道收消息入口基：L1.M0[R1]。来源：`C1 + MAINLINE + CHANNELDIFF + TRACE`。
  说明：前台任何渠道的第一步，都先从这组收消息入口进来。
- `B2` 当前回合工作面基：L1.M1[R2]。来源：`C2 + MAINLINE + SESSIONBACK + TRACE`。
  说明：消息进来以后，这一轮即时处理就在这块工作面上展开。
- `B3` 会话落账面基：L1.M1[R3]。来源：`C2 + C3 + SESSIONBACK + TRACE`。
  说明：无论是当前回合结果还是后台结果，最后都要落回这块会话页面。
- `B4` 渠道发消息出口基：L1.M0[R2]。来源：`C1 + MAINLINE + CHANNELDIFF + TRACE`。
  说明：会话页上可发送的结果，最后都从这组出口回到外部承载面。
- `B5` 后台续跑面基：L1.M2[R4]。来源：`C3 + ASYNC + SESSIONBACK + TRACE`。
  说明：那些不能在当前回合立刻做完的事，从这里挂成侧线继续往后做，并在做完后回到会话落账面。

## 4. 基组合原则（Base Combination Principles）

- `R1` 前台主线成立
  - `R1.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R1.2` 组合方式：把渠道收消息入口、当前回合工作面、会话落账面和渠道发消息出口顺序接起，形成一条完整前台主线。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`MAINLINE/CHANNELDIFF/SESSIONBACK/TRACE`。
- `R2` 后台侧线成立
  - `R2.1` 参与基：`B2 + B3 + B5`。
  - `R2.2` 组合方式：让后台续跑面挂在当前回合工作面侧边，并把结果重新接回会话落账面，形成一条独立后台侧线。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`SESSIONBACK/ASYNC/TRACE`。
- `R3` 全框架成立
  - `R3.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R3.2` 组合方式：把前台主线和后台侧线共同绑定到同一套会话落账规则和追踪规则上，形成完整多渠道个人助手框架。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`MAINLINE/CHANNELDIFF/SESSIONBACK/ASYNC/TRACE`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R4.2` 组合方式：渠道差异侵入当前回合工作面、结果不落账、后台续跑直接顶替前台主线或任一路径无法回放的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`MAINLINE/CHANNELDIFF/SESSIONBACK/ASYNC/TRACE`。

## 5. 验证（Verification）

- `V1` 前台主线有效性：收消息入口、当前回合工作面、会话落账面和发消息出口必须满足 `MAINLINE/CHANNELDIFF` 约束。
- `V2` 落账有效性：任一结果都必须满足 `SESSIONBACK` 约束。
- `V3` 后台侧线有效性：后台续跑面必须满足 `ASYNC/TRACE` 约束。
- `V4` 全链有效性：前台主线和后台侧线都必须能顺序回放到同一会话页组。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
