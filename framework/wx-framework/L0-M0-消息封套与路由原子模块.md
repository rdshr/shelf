# 消息封套与路由原子模块:MessageEnvelopeAndRoutingAtoms

@framework

本模块定义多渠道助手里一条消息最底层要带哪些部件。它只回答 5 件事：消息头怎么放、内容怎么放、挂到哪条会话上、要送到哪里、后续怎么接下去；不负责具体渠道协议、人格文案和推理内容。

## 1. 能力声明（Capability Statement）

- `C1` 消息本体抽象能力：定义一条消息自身最少要带哪些部件。
- `C2` 消息挂线抽象能力：定义一条消息怎样挂到会话线和目标线。
- `C3` 消息接续抽象能力：定义一条消息怎样和后续记录接起来。
- `C4` 非能力项：不负责具体渠道字段名、提供商响应语义和产品文案。

## 2. 边界定义（Boundary / 参数）

- `IDENTITY` 标识边界：同一消息在同一条会话线上只能有一个主标识。来源：`C1 + C2`。
- `DIRECTION` 方向边界：同一消息只能是入站或出站之一，不得同时带双向语义。来源：`C1 + C2`。
- `SESSION` 会话边界：同一消息必须挂到一条明确会话线上，并能排出稳定先后序。来源：`C2 + C3`。
- `TARGET` 目标边界：同一消息必须指向一个明确目标面，缺失则不能路由。来源：`C2`。
- `LINK` 接续边界：任一后继记录都必须指回前一条消息或前一条后继记录，不得断链。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 消息头基：由主标识位和方向位构成。来源：`C1 + IDENTITY + DIRECTION`。
  说明：先把“这是谁”和“这是进来的还是出去的”放稳。
- `B2` 消息内容基：由正文位和附带位构成。来源：`C1 + DIRECTION`。
  说明：真正要传的内容和附带材料都放在这里。
- `B3` 会话挂点基：由会话锚位和顺序位构成。来源：`C2 + SESSION`。
  说明：消息先挂到哪条会话线上、排在什么位置，要在这里定住。
- `B4` 目标挂点基：由来源位和目标位构成。来源：`C2 + TARGET + DIRECTION`。
  说明：消息从哪来、要到哪去，要在这里定住。
- `B5` 后继引用基：由前序位、状态位和终止位构成。来源：`C3 + LINK + SESSION`。
  说明：消息离开当前节点后，后面的记录靠这组引用接上。

## 4. 基组合原则（Base Combination Principles）

- `R1` 消息本体成立
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先把消息头和消息内容接成一个完整消息本体。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`IDENTITY/DIRECTION`。
- `R2` 消息挂线成立
  - `R2.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R2.2` 组合方式：在完整消息本体上补齐会话挂点和目标挂点，使消息能落到一条会话线上并指向一个明确目标面。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`IDENTITY/DIRECTION/SESSION/TARGET`。
- `R3` 消息接续成立
  - `R3.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R3.2` 组合方式：在已挂线消息后补齐后继引用，使同一条消息线能够继续往后接。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`IDENTITY/DIRECTION/SESSION/TARGET/LINK`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3 + B4 + B5`。
  - `R4.2` 组合方式：主标识不唯一、方向冲突、会话挂线缺失、目标缺失或后继引用断链的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`IDENTITY/DIRECTION/SESSION/TARGET/LINK`。

## 5. 验证（Verification）

- `V1` 本体有效性：消息头和消息内容必须满足 `IDENTITY/DIRECTION` 约束。
- `V2` 挂线有效性：会话挂点和目标挂点必须满足 `SESSION/TARGET` 约束。
- `V3` 接续有效性：后继引用必须满足 `LINK/SESSION` 约束。
- `V4` 全链有效性：同一消息及其后继记录必须能沿引用位顺序回放。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
