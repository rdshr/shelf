# 消息封套与路由原子模块:MessageEnvelopeAndRoutingAtoms

@framework

本模块定义多渠道助手框架最底层的消息封套、锚点定位和回执链结构。它只回答“一个消息单元在回路里最少要长成什么样、怎样被定位、怎样留下后继链”，不负责具体渠道协议、人格文案或推理内容。

## 1. 能力声明（Capability Statement）

- `C1` 消息封套抽象能力：定义消息单元的稳定承载外形。
- `C2` 锚点定位抽象能力：定义消息单元落入回路时的稳定定位外形。
- `C3` 回执链抽象能力：定义消息单元离开或续写时的稳定后继外形。
- `C4` 非能力项：不负责具体渠道协议语法、提供商响应语义与产品人格文案。

## 2. 边界定义（Boundary / 参数）

- `IDENTITY` 标识边界：同一消息单元在同一回路内必须只有一个主标识，不得并存两个等价主标识。来源：`C1 + C2`。
- `DIRECTION` 方向边界：同一消息单元必须只属于入站或出站单一方向，不得同时承担双向语义。来源：`C1 + C2`。
- `ANCHOR` 锚点边界：同一消息单元必须能同时落到一个会话锚点和一个目标锚点，缺一则定位无效。来源：`C2`。
- `ORDER` 顺序边界：同一会话锚点上的消息单元和回执单元必须能排出稳定先后序。来源：`C2 + C3`。
- `LINK` 链接边界：任一回执单元必须能指回其前序消息单元或前序回执单元，不得断链。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 消息封套基：由头部区、内容区和附带区构成。来源：`C1 + IDENTITY + DIRECTION`。
  说明：消息先要有一个完整封套，把身份、正文和附带内容放在一起。
- `B2` 消息定位基：由来源锚点位、目标锚点位和会话锚点位构成。来源：`C2 + ANCHOR + ORDER`。
  说明：消息进入系统后，要知道自己从哪来、到哪去、挂在哪条会话线上。
- `B3` 消息回执链基：由前序引用位、状态位和终止位构成。来源：`C3 + LINK + ORDER + DIRECTION`。
  说明：消息走出当前节点后，后续记录要能沿着同一条链继续接下去。

## 4. 基组合原则（Base Combination Principles）

- `R1` 封套骨架成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固定头部区、内容区和附带区的三段骨架，再允许消息单元进入回路。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`IDENTITY/DIRECTION`。
- `R2` 定位闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在封套骨架上挂接来源锚点位、目标锚点位和会话锚点位，形成可定位消息单元。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`IDENTITY/DIRECTION/ANCHOR/ORDER`。
- `R3` 回执闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在已定位消息单元之后挂接回执链路结构，使其能沿稳定先后序继续生成后继单元。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`IDENTITY/DIRECTION/ANCHOR/ORDER/LINK`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：主标识不唯一、消息方向冲突、锚点缺失或回执链无法指回前序单元的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`IDENTITY/DIRECTION/ANCHOR/ORDER/LINK`。

## 5. 验证（Verification）

- `V1` 骨架有效性：每个消息单元都必须满足 `IDENTITY/DIRECTION` 约束。
- `V2` 定位有效性：每个消息单元都必须满足 `ANCHOR/ORDER` 约束。
- `V3` 链路有效性：每个回执单元都必须满足 `LINK/ORDER` 约束。
- `V4` 闭合有效性：同一消息单元及其后继单元必须能沿回执链完整回放。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
