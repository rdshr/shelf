# 后台延续编排模块:BackgroundContinuationOrchestration

@framework

本模块把来源挂点、触发窗口、后台槽位和结果回投口接成一条后台续跑线。它只负责 4 件事：后台续跑从哪起、什么时候被叫醒、在后台哪儿继续做、结果从哪回到主会话；不负责前台渠道接入和回复外形。

## 1. 能力声明（Capability Statement）

- `C1` 续跑起点能力：把后台续跑稳定挂回一个明确来源。
- `C2` 后台展开能力：把脱离当前回合后的工作稳定放到后台槽位里展开。
- `C3` 结果回投能力：把后台结果重新接回主会话和主消息线。
- `C4` 非能力项：不负责前台消息承接、渠道隔离规则和具体提醒文案。

## 2. 边界定义（Boundary / 参数）

- `SOURCE` 来源边界：每个后台续跑单元都必须指回一个明确来源会话或来源回合。来源：`C1 + C2`。
- `WAKE` 唤醒边界：每次后台续跑都必须由一个声明过的计划、事件或恢复条件触发。来源：`C1`。
- `RUNNER` 续跑边界：后台工作必须落在显式后台槽位上，不得悬空展开。来源：`C2`。
- `REJOIN` 回投边界：每个后台结果都必须回到一个明确会话页和一条明确消息线。来源：`C3`。
- `RESUME` 恢复边界：暂停、失败、重试和恢复落点必须显式可见。来源：`C2 + C3`。
- `TRACE` 追踪边界：来源、唤醒原因、后台历程和回投结果必须可顺序回放。来源：`C1 + C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 来源挂点基：L0.M1[R1]。来源：`C1 + SOURCE + TRACE`。
  说明：后台续跑不许无根起步，先要挂回一条明确会话主轴。
- `B2` 唤醒窗口基：L0.M2[R4]。来源：`C1 + WAKE + RESUME`。
  说明：计划、唤醒和恢复条件先落在一组明确窗口里。
- `B3` 后台续跑槽位基：L0.M2[R3] + L0.M1[R3]。来源：`C2 + RUNNER + RESUME + TRACE`。
  说明：`L0.M2[R3]` 给出后台槽位，`L0.M1[R3]` 给出稳定页组，两者接上后后台续跑才有地方继续做。
- `B4` 结果回投口基：L0.M0[R3] + L0.M1[R2] + L0.M1[R3] + L0.M2[R2]。来源：`C3 + REJOIN + RESUME + TRACE`。
  说明：可接续结果、会话页和出站口一起到位后，后台结果才回得来、发得出去。

## 4. 基组合原则（Base Combination Principles）

- `R1` 续跑起点成立
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先把唤醒窗口挂到明确来源挂点上，形成一个能被叫醒的后台起点。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SOURCE/WAKE/RESUME/TRACE`。
- `R2` 后台展开成立
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：让被叫醒的后台起点进入后台续跑槽位，使工作能脱离当前回合继续往后做。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`SOURCE/WAKE/RUNNER/RESUME/TRACE`。
- `R3` 结果回投成立
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：把后台续跑产出的结果接到结果回投口，使结果能回到会话页并重新进入消息线。
  - `R3.3` 输出能力：`C2 + C3`。
  - `R3.4` 边界绑定：`REJOIN/RESUME/TRACE`。
- `R4` 后台续跑线成立
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：把来源挂点、唤醒窗口、后台续跑槽位和结果回投口接成一条完整后台续跑线。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`SOURCE/WAKE/RUNNER/REJOIN/RESUME/TRACE`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：来源挂点缺失、唤醒原因不明、后台续跑无槽位、恢复落点缺失或结果回不到主会话的组合无效。
  - `R5.3` 输出能力：`C4`。
  - `R5.4` 边界绑定：`SOURCE/WAKE/RUNNER/REJOIN/RESUME/TRACE`。

## 5. 验证（Verification）

- `V1` 起点有效性：来源挂点和唤醒窗口必须满足 `SOURCE/WAKE/RESUME` 约束。
- `V2` 续跑有效性：后台续跑槽位必须满足 `RUNNER/RESUME/TRACE` 约束。
- `V3` 回投有效性：结果回投口必须满足 `REJOIN/RESUME/TRACE` 约束。
- `V4` 续跑线有效性：同一后台单元的起点、续跑和回投必须能顺序回放到同一来源会话。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
