# 代理工作编排模块:AgentWorkOrchestration

@framework

本模块把输入消息、会话页组和当前回合槽位接成一轮即时工作线。它只负责 4 件事：输入怎么装载、常驻规则怎么带上、当前回合在哪工作、结果怎么落回会话页；不负责渠道接入和后台计划。

## 1. 能力声明（Capability Statement）

- `C1` 输入装载能力：把当前回合要处理的消息和会话页装成一包可用输入。
- `C2` 当前回合工作能力：把这一轮要做的事放到当前回合槽位里展开。
- `C3` 结果写回能力：把这一轮做出的结果稳定写回会话页。
- `C4` 非能力项：不负责渠道桥接策略、后台唤醒计划和具体提供商实例值。

## 2. 边界定义（Boundary / 参数）

- `LOAD` 装载边界：单个工作回合只能装载一包输入消息和一组明确会话页。来源：`C1 + C3`。
- `OWNERSHIP` 归属边界：同一回合里的中间节点和最终结果不得跨两个会话主轴。来源：`C1 + C2`。
- `ITERATE` 展开边界：中间节点只能沿当前回合往后展开，不得回插更早节点。来源：`C2`。
- `SCOPE` 作用域边界：当前回合可见的规则和页组范围必须显式界定。来源：`C1 + C2 + C3`。
- `WRITEBACK` 写回边界：最终结果只能落回当前会话可见页和沉淀页。来源：`C3`。
- `TRACE` 追踪边界：任一中间节点必须能回到它的输入包和回合主节点。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 输入消息包基：L0.M0[R2] + L0.M1[R2]。来源：`C1 + LOAD + OWNERSHIP`。
  说明：`L0.M0[R2]` 给出已挂线消息，`L0.M1[R2]` 给出当前和历史账页，两者接上后才有这一轮真正能读的输入包。
- `B2` 常驻规则页组基：L0.M1[R3]。来源：`C1 + SCOPE`。
  说明：这一轮长期要带着走的身份、规则和稳定页组，从这里进来。
- `B3` 当前回合槽位基：L0.M2[R2]。来源：`C2 + ITERATE + SCOPE + TRACE`。
  说明：这一轮真正展开工作的地方，就是当前回合槽位。
- `B4` 会话回写口基：L0.M0[R3] + L0.M1[R2] + L0.M1[R3]。来源：`C3 + WRITEBACK + TRACE + OWNERSHIP`。
  说明：`L0.M0[R3]` 给出可接续结果，`L0.M1[R2]` 给出过程页，`L0.M1[R3]` 给出稳定页，三者接上后结果才有地方落回去。

## 4. 基组合原则（Base Combination Principles）

- `R1` 输入装载成立
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先把输入消息包和常驻规则页组装到同一会话主轴上，形成当前回合可用输入面。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`LOAD/OWNERSHIP/SCOPE`。
- `R2` 当前回合展开成立
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：让已装载输入进入当前回合槽位，使这一轮工作能沿当前回合顺序展开。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`LOAD/OWNERSHIP/ITERATE/SCOPE/TRACE`。
- `R3` 结果写回成立
  - `R3.1` 参与基：`B3 + B4`。
  - `R3.2` 组合方式：把当前回合槽位产出的结果接到会话回写口，使结果能落回过程页和稳定页。
  - `R3.3` 输出能力：`C2 + C3`。
  - `R3.4` 边界绑定：`OWNERSHIP/WRITEBACK/TRACE`。
- `R4` 回合工作线成立
  - `R4.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R4.2` 组合方式：把输入装载、当前回合展开和结果写回接成一条完整即时工作线。
  - `R4.3` 输出能力：`C1 + C2 + C3`。
  - `R4.4` 边界绑定：`LOAD/OWNERSHIP/ITERATE/SCOPE/WRITEBACK/TRACE`。
- `R5` 禁止组合
  - `R5.1` 参与基：`B1 + B2 + B3 + B4`。
  - `R5.2` 组合方式：输入包跨两个会话主轴、规则页越出声明范围、当前回合回插旧节点或结果落不到会话页的组合无效。
  - `R5.3` 输出能力：`C4`。
  - `R5.4` 边界绑定：`LOAD/OWNERSHIP/ITERATE/SCOPE/WRITEBACK/TRACE`。

## 5. 验证（Verification）

- `V1` 装载有效性：输入消息包和常驻规则页组必须满足 `LOAD/OWNERSHIP/SCOPE` 约束。
- `V2` 槽位有效性：当前回合槽位必须满足 `ITERATE/SCOPE/TRACE` 约束。
- `V3` 写回有效性：会话回写口必须满足 `WRITEBACK/TRACE` 约束。
- `V4` 工作线有效性：同一回合的装载、展开和写回必须能顺序回放到同一会话主轴。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
