# 代理工作编排模块:AgentWorkOrchestration

@framework

本模块把消息单元、会话记载结构和工作腔位结构组合成代理工作回路。它只负责当前回合怎样装载、怎样展开工作链、怎样沉淀结果，不负责渠道接入治理和具体渠道协议。

## 1. 能力声明（Capability Statement）

- `C1` 上下文装载能力：把当前回合所需的输入封套、会话主轴和常驻册页收敛为稳定装载面。
- `C2` 工作链展开能力：把当前回合的中间节点和工作腔位收敛为稳定工作链。
- `C3` 结果沉淀能力：把当前回合的最终结果稳定写回会话记载面。
- `C4` 非能力项：不负责渠道桥接策略、后台唤醒计划和具体提供商实例值。

## 2. 边界定义（Boundary / 参数）

- `LOAD` 装载边界：单个工作回合必须只装载一个输入封套、一个会话主轴切片和一个常驻册页集合。来源：`C1 + C3`。
- `OWNERSHIP` 归属边界：同一工作回合的中间节点和最终结果不得跨两个会话主轴。来源：`C1 + C2`。
- `ITERATE` 链式边界：中间节点只能沿当前回合形成单向工作链，不得回插更早节点。来源：`C2`。
- `SCOPE` 作用域边界：工作链可使用的腔位必须受当前册页作用域限制。来源：`C1 + C2 + C3`。
- `WRITEBACK` 写回边界：最终结果只能写回当前会话可见账页和沉淀带。来源：`C3`。
- `TRACE` 追踪边界：任一中间节点必须能回到其输入封套和回合主节点。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 当前回合上下文基：L0.M0[R2] + L0.M1[R2]。来源：`C1 + LOAD + OWNERSHIP + SCOPE`。
  说明：把当前输入消息和它所属的会话记录收成这个回合唯一可用的上下文。
- `B2` 当前回合工作链基：L0.M0[R2] + L0.M2[R2]。来源：`C2 + OWNERSHIP + ITERATE + SCOPE + TRACE`。
  说明：这个回合内部一步一步怎么往下走，就落在这条工作链上。
- `B3` 当前回合结果落点基：L0.M0[R3] + L0.M1[R3]。来源：`C3 + WRITEBACK + TRACE + OWNERSHIP`。
  说明：这个回合做完以后，结果最后落到哪里、怎么落回记录里，由这块负责。

## 4. 基组合原则（Base Combination Principles）

- `R1` 上下文包成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先把输入封套、会话主轴切片和常驻册页收成一个回合上下文包，再允许后继工作链附着其上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`LOAD/OWNERSHIP/SCOPE`。
- `R2` 工作链闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在已建立的回合上下文包后挂接工作腔位链，使当前回合沿单向节点链展开。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`LOAD/OWNERSHIP/ITERATE/SCOPE/TRACE`。
- `R3` 回合闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：把最终结果通过结果沉淀面接回当前会话记载面，使一次代理工作在本回合内闭合。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`LOAD/OWNERSHIP/ITERATE/SCOPE/WRITEBACK/TRACE`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：回合装载跨两个会话主轴、工作链回插旧节点、腔位越出作用域或最终结果无法写回当前账页的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`LOAD/OWNERSHIP/ITERATE/SCOPE/WRITEBACK/TRACE`。

## 5. 验证（Verification）

- `V1` 装载有效性：回合上下文包必须满足 `LOAD/OWNERSHIP/SCOPE` 约束。
- `V2` 工作链有效性：回合工作链必须满足 `ITERATE/SCOPE/TRACE` 约束。
- `V3` 沉淀有效性：结果沉淀链必须满足 `WRITEBACK/TRACE` 约束。
- `V4` 回路闭合性：同一回合的装载、工作链和沉淀链必须能顺序回放到同一会话主轴。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
