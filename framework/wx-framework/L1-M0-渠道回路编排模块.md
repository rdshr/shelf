# 渠道回路编排模块:ChannelLoopOrchestration

@framework

本模块把消息封套结构和通道插口结构组合成渠道回路。它只负责多渠道消息怎样汇入、怎样回投、怎样隔离，不负责回复内容本身。

## 1. 能力声明（Capability Statement）

- `C1` 入站汇流能力：把外部消息稳定并入统一渠道回路。
- `C2` 出站回投能力：把系统结果稳定送回目标承载面。
- `C3` 渠道隔离能力：把多渠道并行存在时的隔离关系收敛为稳定结构。
- `C4` 非能力项：不负责推理生成逻辑、会话记忆策略与后台计划值。

## 2. 边界定义（Boundary / 参数）

- `INGRESS` 汇入边界：同一外部消息只能经一个声明过的入站汇流口进入主回路。来源：`C1 + C3`。
- `EGRESS` 回投边界：同一出站结果只能落到一个声明过的目标回路，不得一结果多投。来源：`C2 + C3`。
- `CARRIER` 承载边界：消息进入渠道回路后不得丢失主标识、方向位和锚点位。来源：`C1 + C2`。
- `ISOLATE` 隔离边界：不同渠道和不同会话不得共享同一活动回路。来源：`C1 + C3`。
- `BRIDGE` 侧链边界：桥接段只能作为显式侧链挂接，不得冒充主回路。来源：`C2 + C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 入站通道基：L0.M0[R2] + L0.M2[R1]。来源：`C1 + INGRESS + CARRIER + ISOLATE`。
  说明：把已定位的消息接到接入插口上，形成消息进入系统的入口通道。
- `B2` 出站通道基：L0.M0[R3] + L0.M2[R1]。来源：`C2 + EGRESS + CARRIER + BRIDGE`。
  说明：把已经带着后继链的消息接到离开插口上，形成结果离开系统的出口通道。
- `B3` 渠道分区基：L0.M0[R2,R3] + L0.M2[R1]。来源：`C3 + ISOLATE + BRIDGE + INGRESS + EGRESS`。
  说明：把同一渠道的入口和出口收进同一个分区里，让不同渠道各走各的回路。

## 4. 基组合原则（Base Combination Principles）

- `R1` 入站段成立
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先建立入站通道段，使外部消息通过固定入口落到可定位消息单元上。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`INGRESS/CARRIER/ISOLATE`。
- `R2` 通道闭合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在已建立的入站通道段后接出站通道段，形成一条从进入到离开的完整渠道通路。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`INGRESS/EGRESS/CARRIER/BRIDGE`。
- `R3` 分区闭合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将渠道分区壳包住已闭合的渠道通路，使多条渠道通路能够并行存在而不互相串扰。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`INGRESS/EGRESS/CARRIER/ISOLATE/BRIDGE`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：入站汇流口重复、出站目标歧义、消息承载位丢失或不同渠道共用同一活动回路的组合无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`INGRESS/EGRESS/CARRIER/ISOLATE/BRIDGE`。

## 5. 验证（Verification）

- `V1` 汇流有效性：入站汇流结构必须满足 `INGRESS/CARRIER` 约束。
- `V2` 回投有效性：出站回投结构必须满足 `EGRESS/CARRIER/BRIDGE` 约束。
- `V3` 隔离有效性：渠道隔离结构必须满足 `ISOLATE` 约束。
- `V4` 回路闭合性：同一渠道消息必须能从汇流口追到回投口且途中不丢失主标识和锚点位。
- `V5` 最小必要性：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 边界` 的通过或失败结论。
