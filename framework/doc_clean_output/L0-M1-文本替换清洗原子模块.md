# 文本替换清洗原子模块:TextReplacementCleaningAtoms

@framework

组件目标：读取统一格式的文本片段，按替换规则完成文本清洗，并把结果落为稳定结构化清洗结果。

## 1. 能力声明（Capability Statement）

- `C1` 替换规则承接能力：把待替换文本及其目标文本承接为稳定规则结构。
- `C2` 匹配范围控制能力：把清洗动作稳定约束在可判定的处理范围内。
- `C3` 文本清洗生成能力：把输入文本按规则生成清洗后的正文结果。
- `C4` 结果对位保持能力：把清洗结果稳定绑定到输入片段及其原有顺序锚点，并收敛为结构化结果集合，不在本层创造新的编号或归并规则。
- `C5` 非能力项：不负责文件格式解析，不负责目标文件格式转换。

## 2. 边界定义（Boundary / 参数）

- `TM` 文本模型边界：输入必须是带有片段引用、文本内容和原顺序锚点的统一文本片段集合。来源：`C2 + C3 + C4`。
- `RULE` 规则边界：原文本、替换文本和匹配条件必须稳定。来源：`C1 + C3`。
- `SCOPE` 范围边界：一次处理的文本范围、分隔方式和扫描粒度必须可判定。来源：`C2 + C3`。
- `LIMIT` 体量边界：总文本长度、片段数量和单窗口长度必须可校验。来源：`C2 + C3`。
- `RESULT` 结果边界：清洗后的文本结果必须保留片段对位关系，并落为可继续消费的结构化结果。来源：`C3 + C4`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 规则映射结构基（自有基）：文本替换规则的承载结构，由原文本映射、目标文本映射和匹配条件构成。来源：`C1 + RULE`。
- `B2` 扫描窗口结构基（自有基）：文本扫描范围的承载结构，由片段引用、窗口边界、分隔标记和体量约束位构成。来源：`C2 + TM + SCOPE + LIMIT`。
- `B3` 结果对位结构基（自有基）：结构化清洗结果的承载结构，由片段引用、清洗后文本、原顺序锚点和结果状态构成。来源：`C3 + C4 + TM + RESULT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 规则先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：所有文本清洗都必须先形成稳定规则映射；没有规则映射不承认有效清洗入口。
  - `R1.3` 输出结构：`RULE_MAP`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`RULE`。
- `R2` 范围清洗组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：输入文本必须先落入扫描窗口结构，再按规则映射形成清洗结果；没有稳定处理范围直接全文替换的用法不承认有效。
  - `R2.3` 输出结构：`CLEAN_RESULT_SET`。
  - `R2.4` 输出能力：`C2 + C3`。
  - `R2.5` 边界绑定：`TM/RULE/SCOPE/LIMIT/RESULT`。
- `R3` 对位保持组合
  - `R3.1` 参与基：`B2 + B3`。
  - `R3.2` 组合方式：所有清洗结果都必须保留输入片段引用和原顺序锚点，再写入结构化清洗结果；在本层重新编号、重建段落归并规则或创造新顺序的用法不承认有效。
  - `R3.3` 输出结构：`STRUCTURED_CLEAN_RESULT`。
  - `R3.4` 输出能力：`C4`。
  - `R3.5` 边界绑定：`TM/SCOPE/RESULT`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：没有规则直接清洗、没有范围直接替换、结果无法保持输入片段对位，或把文件解析、输出格式编码、段落重组规则直接塞进本层的组合均无效。
  - `R4.3` 输出能力：`C5`。
  - `R4.4` 边界绑定：`TM/RULE/SCOPE/LIMIT/RESULT`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R3` 输出能力并集必须覆盖 `C1~C4`。
- `V4` 边界符合性：所有有效清洗都必须满足文本模型、规则、范围、体量和结果边界。
- `V5` 最小必要性：移除任一 `B*` 后，规则承接、范围控制、文本清洗生成或结果对位保持至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
