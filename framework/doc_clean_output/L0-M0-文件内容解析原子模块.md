# 文件内容解析原子模块:FileContentParsingAtoms

@framework

组件目标：判定输入文件类型，绑定对应解析器，并把解析结果统一为可清洗的文本片段结构。

## 1. 能力声明（Capability Statement）

- `C1` 文件类型判定能力：把输入文件稳定判定到可解释的文件类型。
- `C2` 解析器分派能力：把已判定文件类型稳定绑定到对应解析器。
- `C3` 解析结果承接能力：把解析后的文本结果维持为稳定可遍历结构。
- `C4` 非能力项：不负责文本替换规则，不负责输出格式封装。

## 2. 边界定义（Boundary / 参数）

- `SOURCE` 文件源边界：文件标识、文件内容载体和基础元信息必须稳定。来源：`C1 + C2`。
- `FORMAT` 文件类型边界：文件类型必须可判定，并能稳定落到对应解析路径。来源：`C1 + C2`。
- `PARSER` 解析器边界：每种文件类型必须能绑定到明确解析器，未绑定类型不承认有效解析。来源：`C2 + C3`。
- `TEXT` 文本结果边界：解析结果必须是带片段引用和原顺序锚点的统一文本片段集合。来源：`C3`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 文件类型判定结构基（自有基）：文件类型判定结果的承载结构，由文件源引用、类型标识和失败分流位构成。来源：`C1 + SOURCE + FORMAT`。
- `B2` 解析器映射结构基（自有基）：文件类型与解析器对应关系的承载结构，由类型键、解析器引用和映射约束构成。来源：`C2 + FORMAT + PARSER`。
- `B3` 统一文本结构基（自有基）：解析后统一文本结果的承载结构，由片段引用、原顺序锚点、文本内容和结果状态构成。来源：`C3 + PARSER + TEXT`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 类型判定先行
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：所有输入文件都必须先完成类型判定，再允许进入后续解析器绑定；类型不可判定的输入不承认有效。
  - `R1.3` 输出结构：`FORMAT_ROUTE`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`SOURCE/FORMAT`。
- `R2` 解析器分派组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：已判定文件类型必须绑定到对应解析器，再允许进入具体解析；没有稳定映射关系的类型不承认有效解析。
  - `R2.3` 输出结构：`BOUND_PARSER_ROUTE`。
  - `R2.4` 输出能力：`C1 + C2`。
  - `R2.5` 边界绑定：`FORMAT/PARSER`。
- `R3` 结果归一组合
  - `R3.1` 参与基：`B2 + B3`。
  - `R3.2` 组合方式：不同解析器的输出都必须归一为统一文本片段集合；无法被清洗层读取的解析结果不承认有效。
  - `R3.3` 输出结构：`NORMALIZED_TEXT_SET`。
  - `R3.4` 输出能力：`C3`。
  - `R3.5` 边界绑定：`PARSER/TEXT`。
- `R4` 禁止组合
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：类型不可判定、解析器未绑定、结果无法归一为统一文本片段集合，或把具体解析器内部算法和文本替换逻辑直接塞进本层的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SOURCE/FORMAT/PARSER/TEXT`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R3` 输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效解析都必须满足文件源、文件类型、解析器和统一文本边界。
- `V5` 最小必要性：移除任一 `B*` 后，类型判定、解析器分派或结果归一至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
