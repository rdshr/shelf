# 树组件:Tree

@framework

本组件定义通用树结构的根层抽象。
它只定义节点实体、连接接口和树形组合规则，不引入文件域、持久化或项目装配语义。

## 1. 能力声明（Capability Statement）

- `C1` 节点建立能力：在合法父节点下创建树节点，并承载节点附加属性。
- `C2` 树结构查询能力：输出当前树视图和节点快照，并保留节点附加属性。
- `C3` 节点修改能力：修改节点标签和附加属性而不破坏树结构。
- `C4` 节点重挂接能力：改变父子关系且不形成环。
- `C5` 节点删除能力：按删除模式和级联模式收敛删除结果。
- `C6` 非能力项：不定义文件域语义、不负责持久化介质、不负责鉴权本身。

## 2. 边界定义（Boundary / 参数）

- `ID` 节点标识（id）：每个节点必须有稳定唯一标识。来源：`C1 + C2 + C3 + C4 + C5`。
- `PID` 父节点标识（parent_id）：作为连接结果的父节点投影，非根节点必须稳定指向已存在父节点。来源：`C1 + C2 + C4`。
- `NM` 节点标签（name）：节点标签必须合法且非空。来源：`C1 + C3`。
- `AT` 节点附加属性（attributes）：键值映射，可选；组件必须原样承载、校验并在查询时返回。来源：`C1 + C2 + C3`。
- `RL` 关系约束（relation_rule）：节点连接必须满足单父、无环、无悬空。来源：`C2 + C4 + C5`。
- `DM` 删除模式（delete_mode）：`soft | hard`。来源：`C5`。
- `CM` 级联模式（cascade_mode）：`forbid | subtree`。来源：`C5`。
- `ST` 节点状态（state）：至少支持 `active | deleted`。来源：`C2 + C5`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 节点实体单元：节点作为实体记录的内部结构，由标识、标签、附加属性和状态槽位构成。来源：`C1 + C2 + C3 + C5 + ID + NM + AT + ST`。
- `B2` 连接接口单元：节点之间用于挂接与重绑定的接口结构，由父节点投影入口、连接位向和关系约束槽位构成。来源：`C1 + C2 + C4 + C5 + PID + RL`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 节点建立与挂接组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先形成节点实体，再通过连接接口完成挂接；根节点可无父节点，非根节点必须挂接到已存在父节点；节点附加属性必须随实体一起建立，新节点状态必须收敛为 active。
  - `R1.3` 输出结构：`NODE_SET + LINK_BINDING`。
  - `R1.4` 输出能力：`C1`。
  - `R1.5` 边界绑定：`ID/PID/NM/AT/RL/ST`。
- `R2` 树查询投影组合
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：树视图和节点快照只能由节点实体及其连接结果推导；查询结果必须满足单父、无环、无悬空约束，且不得丢失节点附加属性。
  - `R2.3` 输出结构：`TREE_VIEW + SNAPSHOT`。
  - `R2.4` 输出能力：`C2`。
  - `R2.5` 边界绑定：`ID/PID/AT/RL/ST`。
- `R3` 节点内容修改组合
  - `R3.1` 参与基：`B1`。
  - `R3.2` 组合方式：仅允许修改节点标签和附加属性；节点身份必须保持不变，修改不得隐式改写任何连接结果或删除状态。
  - `R3.3` 输出结构：`MUTATED_NODE`。
  - `R3.4` 输出能力：`C3`。
  - `R3.5` 边界绑定：`ID/NM/AT/ST`。
- `R4` 接口重绑定组合
  - `R4.1` 参与基：`B1 + B2`。
  - `R4.2` 组合方式：重挂接节点时必须先解除原有挂接，再通过连接接口重绑定到目标父节点；目标父节点必须已存在，重绑定后不得形成环或悬空关系，节点身份和附加属性必须保持连续。
  - `R4.3` 输出结构：`REBOUND_TREE`。
  - `R4.4` 输出能力：`C4`。
  - `R4.5` 边界绑定：`ID/PID/AT/RL`。
- `R5` 删除收敛组合
  - `R5.1` 参与基：`B1 + B2`。
  - `R5.2` 组合方式：删除动作必须作用于已存在节点实体及其连接结果；删除结果必须按删除模式和级联模式收敛为明确的节点状态或连接移除结果；未删除节点的附加属性不得被无故改写。
  - `R5.3` 输出结构：`DELETE_RESULT`。
  - `R5.4` 输出能力：`C5`。
  - `R5.5` 边界绑定：`ID/PID/AT/DM/CM/ST/RL`。
- `R6` 完整树组合
  - `R6.1` 参与基：`B1 + B2`。
  - `R6.2` 组合方式：通过节点实体承载节点信息，通过连接接口形成树拓扑；在此基础上开展创建、查询、修改、重挂接和删除；所有结果都必须保持父子关系可投影且节点附加属性可追踪。
  - `R6.3` 输出能力：`C1 + C2 + C3 + C4 + C5`。
  - `R6.4` 边界绑定：`ID/PID/NM/AT/RL/DM/CM/ST`。
- `R7` 禁止组合
  - `R7.1` 参与基：`B1 + B2`。
  - `R7.2` 组合方式：没有节点实体直接挂接、没有连接接口直接形成父子树、绕过连接接口直接改写父节点投影、重绑定形成环或悬空父节点、删除后状态与连接结果不收敛、静默丢失节点附加属性的组合均无效。
  - `R7.3` 输出能力：`C6`。
  - `R7.4` 边界绑定：`ID/PID/NM/AT/RL/DM/CM/ST`。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 和一个参数项推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确参与基、组合方式、输出能力、边界绑定。
- `V3` 目标覆盖性：`R1~R5` 输出能力并集必须覆盖 `C1~C5`。
- `V4` 边界符合性：所有有效组合都必须满足节点标识、连接约束、附加属性和删除收敛边界。
- `V5` 最小必要性：移除任一 `B*` 后，节点承载或连接挂接至少一项缺失，导致创建、查询、修改/重挂接、删除至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
- `V7` 能力落点一致性：`C1~C5` 必须同时出现在 `L3 文档能力声明`、`package.py capabilities` 和 `api.py/component.py` 的公开出口中。
- `V8` 属性一致性：节点附加属性在创建、查询、修改、重挂接、删除后都必须保持可追踪；不得静默丢失。

## 6. 能力暴露（Capability Exposure）

- `E1` 语义落点：本文件 `1. 能力声明`。
- `E2` 机器落点：组件包中的 `package.py::capabilities`。
- `E3` 接口落点：组件包中的 `api.py`。
- `E4` 运行时落点：组件包中的 `component.py`。
- `E5` 一致性要求：`E1~E4` 必须一一对应；新增能力时四处同时修改。
