# Frontend L1 正交性检查表

本文档不是 `framework/*.md` 真相源，而是面向 `framework/frontend/L1-*` 的边界治理与扩展检查表。
它服务于以下目标：

- 在新增 `L1` 原子前，判断是否真的存在新的独立结构角色
- 在已有 `L1` 原子之间，识别相邻边界是否开始重叠
- 在需要继续细分或上提到 `L2` 时，给出一致的检查口径

## 1. 判断口径

对任一候选 `L1` 模块，优先检查以下问题：

1. 它是否表达单一主关系，而不是多轴复合关系
2. 它是否可以主要由 `L0-M0 ~ L0-M2` 直接推出
3. 它是否不需要借现有多个 `L1` 才能成立
4. 它的 `Boundary` 是否有独立边界集，而不是旧模块边界的改写同义词
5. 它的 `N*` 是否能明确排除相邻模块的职责

若以上问题多数回答为“否”，则该对象更可能属于 `L2` 复合结构，而非新的 `L1` 原子。

## 2. 相邻带检查

### 2.1 触发 / 选择面 / 提示面

涉及模块：

- [L1-M0-触发原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M0-触发原子模块.md)
- [L1-M7-临时展开选择面原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M7-临时展开选择面原子模块.md)
- [L1-M8-锚点附着提示原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M8-锚点附着提示原子模块.md)

主关系划分：

- `L1-M0`：离散动作入口
- `L1-M7`：展开后可选择并闭合的临时选择面
- `L1-M8`：附着于锚点的提示或说明面

检查点：

- 若结构重点是“触发动作”，应归入 `L1-M0`
- 若结构重点是“候选项选择后闭合”，应归入 `L1-M7`
- 若结构重点是“说明、提示、注释或短信息附着”，应归入 `L1-M8`
- 不能因为都存在锚点和展开面，就把三者混写为同一类结构

高风险误入：

- 把 `dropdown/menu` 写成纯触发结构
- 把 `tooltip/popover` 写成临时选择面
- 把无选择闭合的说明面误写成 `L1-M7`

### 2.2 展示面 / 任务接管 / 媒体预览

涉及模块：

- [L1-M2-展示与容器原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M2-展示与容器原子模块.md)
- [L1-M5-局部任务接管容器原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M5-局部任务接管容器原子模块.md)
- [L1-M14-媒体预览原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M14-媒体预览原子模块.md)

主关系划分：

- `L1-M2`：承载内容并组织阅读
- `L1-M5`：临时接管局部任务流
- `L1-M14`：围绕当前媒体焦点形成预览闭环

检查点：

- 若结构重点是“阅读与展示承载”，应归入 `L1-M2`
- 若结构重点是“进入/退出/返回/背景策略”，应归入 `L1-M5`
- 若结构重点是“当前媒体对象 + 预览控制 + 预览反馈”，应归入 `L1-M14`
- 不能把媒体查看工作面无限扩张到资源治理、编辑或复杂浏览编排

高风险误入：

- 把 `dialog` 或 `sheet` 只写成容器展示
- 把媒体预览扩写成媒体工作台
- 把文档查看器总模块误压进 `L1-M14`

### 2.3 一般导航 / 页序列 / 路径链

涉及模块：

- [L1-M3-集合浏览原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M3-集合浏览原子模块.md)
- [L1-M10-页序列导航原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M10-页序列导航原子模块.md)
- [L1-M11-路径链导航原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M11-路径链导航原子模块.md)

主关系划分：

- `L1-M3`：一般集合项浏览、移动与当前项承接
- `L1-M10`：页窗序列与页间跳转
- `L1-M11`：上游路径链与当前位置回溯

检查点：

- 若结构重点是“列表、树、节点和当前项浏览”，应归入 `L1-M3`
- 若结构重点是“页号、页窗、前后页跳转”，应归入 `L1-M10`
- 若结构重点是“当前位置由上游路径链表达”，应归入 `L1-M11`
- 不能让 `L1-M3` 继续吞并一切具体导航族

高风险误入：

- 把分页写成一般集合浏览实例
- 把 breadcrumb 写成树形路径浏览
- 在 `L1-M3` 中重复定义页窗或路径链边界

### 2.4 连续文本输入 / 日期时间输入

涉及模块：

- [L1-M1-文本输入原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M1-文本输入原子模块.md)
- [L1-M13-日期时间输入原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M13-日期时间输入原子模块.md)

主关系划分：

- `L1-M1`：连续文本编辑
- `L1-M13`：时间值、粒度、选择面与范围闭合

检查点：

- 若结构重点是自由文本连续编辑，应归入 `L1-M1`
- 若结构重点是值粒度、时间选择面、范围与校验闭合，应归入 `L1-M13`
- 不能因为都有输入壳，就把时间输入写成普通文本编辑

高风险误入：

- 把 date/time picker 写成文本输入的实例 profile
- 把时间范围校验误写成一般文本校验

### 2.5 标记反馈的兜底风险

涉及模块：

- [L1-M4-标记与反馈原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M4-标记与反馈原子模块.md)
- [L1-M8-锚点附着提示原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M8-锚点附着提示原子模块.md)
- [L1-M12-文件摄取原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M12-文件摄取原子模块.md)
- [L1-M13-日期时间输入原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M13-日期时间输入原子模块.md)
- [L1-M14-媒体预览原子模块](/d:/xiawei/2026/shelf/framework/frontend/L1-M14-媒体预览原子模块.md)

主关系划分：

- `L1-M4`：附着于主体结构的标记与反馈
- 其余模块：围绕各自主体结构的专属状态回流

检查点：

- `L1-M4` 应只承接通用附着反馈，不应变成所有状态提示的总兜底
- 文件摄取、时间输入、媒体预览中的状态反馈，应优先留在各自主体结构闭环中
- 若反馈离开主体结构仍可独立成立，才需要判断是否回到 `L1-M4`

高风险误入：

- 把文件摄取状态完全抽空到 `L1-M4`
- 把媒体预览失败提示写成通用气泡反馈
- 把时间输入的范围校验全部下沉成统一错误气泡

## 3. 新增 L1 前的五项检查

新增 `L1` 原子前，至少完成以下检查：

1. 是否存在单一主关系
2. 是否存在独立边界集
3. 是否能写出清晰的 `N*`
4. 是否不会与现有 `L1` 形成大面积同义重叠
5. 是否不更适合上提为 `L2`

若第 2、3、4 项不能明确成立，应优先补边界治理，而不是继续新增模块。

## 4. 当前优先治理建议

按当前 `frontend` 现状，最值得优先治理的相邻带是：

1. `L1-M0 / L1-M7 / L1-M8`
2. `L1-M2 / L1-M5 / L1-M14`
3. `L1-M3 / L1-M10 / L1-M11`
4. `L1-M1 / L1-M13`
5. `L1-M4` 的兜底边界

治理方式优先级：

1. 先补强各模块 `N*`
2. 再统一写模块分界说明
3. 最后才考虑重新拆分或合并模块

## 5. 当前治理进度

截至当前版本，以下相邻带已完成首轮边界收敛：

- `L1-M0 / L1-M7 / L1-M8`
  - `L1-M0` 已从“触发与选择”收窄为“触发原子”。
  - `L1-M7` 与 `L1-M8` 已补强非职责声明，三者主关系已分离为“动作入口 / 选择闭合面 / 附着提示面”。
- `L1-M2 / L1-M5 / L1-M14`
  - 已通过 `N*` 明确分离“展示承载 / 任务接管 / 媒体预览闭环”。
- `L1-M3 / L1-M10 / L1-M11`
  - `L1-M3` 已从“集合与导航”收窄为“集合浏览”，不再作为导航总类。
- `L1-M1 / L1-M13`
  - 已通过 `N*` 明确分离“连续文本编辑”与“时间值、粒度、选择面、范围闭合输入”。
- `L1-M4` 兜底风险
  - `L1-M4` 已收窄为“附着反馈原子”，不再作为所有反馈的总出口。

当前剩余的主要治理重点是：

- 领域模块是否仍沿用旧的宽措辞，把实例结构回写进已收窄的 `L1` 原子。
- 新增 `L1` 原子前，是否先经过本表的相邻带检查，而不是直接开新模块。
- `L1-M4` 是否在实例层继续被误用为统一反馈出口。
