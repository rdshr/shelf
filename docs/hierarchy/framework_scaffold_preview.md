# 承载构件模块:LoadbearingKit

@framework

## 1. 能力声明（Capability Statement）

- `C1` 结构承载能力：提供可重复组装的承载结构能力。
- `C2` 形态生成能力：可生成地板、置物架、支架等目标形态。
- `C3` 适配能力：在给定参数边界内可按需扩展和收缩。
- `C4` 非能力项：不负责电控、装饰、非承载功能件。

## 2. 边界定义（Boundary / 参数）

- `N` 层数（int）：`N >= 1`。来源：`C1 + C3`。
- `P` 单层承重（number）：`P > 0`。来源：`C1`。
- `S` 单层净空（space）：宽/深/高均大于 0。来源：`C2 + C3`。
- `O` 开口尺寸（opening）：需满足可存取约束。来源：`C2`。
- `A` 占地尺寸（footprint）：需满足场地上限约束。来源：`C2 + C3`。
- `T` 连接公差（tolerance）：需满足装配稳定约束。来源：`C1 + C3`。
- `SF` 安全系数（number）：`SF >= 1`。来源：`C1`。

## 3. 最小可行基（Minimum Viable Bases）

- `B1` 骨架：提供主承载路径与结构稳定性。来源：`C1 + N + P + SF`。
- `B2` 连接接口：提供构件连接、定位与传力。来源：`C1 + C3 + T`。
- `B3` 承载面：提供可用放置/受力平面。来源：`C2 + S + O + A + T`。

## 4. 基组合原则（Base Combination Principles）

- `R1` 结构通路组合
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 骨架形态：骨架由立向构件与横向构件组成；稳定单元定义为骨架连接图中存在至少一个闭合环（cycle）。
  - `R1.3` 输出结构：生成连接点集合 `CP_set`，其中元素为 `CP`，每个 `CP` 绑定到骨架端点或骨架交点。
  - `R1.4` 组合方式：连接点按骨架拓扑装配，形成连续传力通路。
  - `R1.5` 几何约束：闭合环内构件关系仅允许平行或垂直；不允许斜接形成不稳定受力路径。
  - `R1.6` 输出能力：满足 `C1`。
  - `R1.7` 边界绑定：受 `N/P/T/SF` 约束。
- `R2` 承载单元组合
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：每个承载面边界至少连接到两个有效连接点 `CP`，且连接点必须来自 `CP_set`。
  - `R2.3` 形态约束：承载面投影需落在可用占地 `A` 内，且满足净空 `S` 与开口 `O` 的可存取要求。
  - `R2.4` 输出能力：满足 `C2` 的基础承载形态能力。
  - `R2.5` 边界绑定：受 `S/O/A/T` 约束。
- `R3` 完整功能组合
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：先完成 `R1` 骨架与连接点，再执行 `R2` 承载面挂接，最后执行整体稳定复核。
  - `R3.3` 闭环约束：每个承载面都必须回溯到至少一条有效传力通路。
  - `R3.4` 输出能力：同时满足 `C1/C2/C3`。
  - `R3.5` 边界绑定：受 `N/P/S/O/A/T/SF` 全量约束。
- `R4` 禁止组合
  - `R4.1` 参与基：任意 `B*` 组合。
  - `R4.2` 组合方式：缺少关键基、存在游离连接点、断裂传力路径或斜接不稳定关系。
  - `R4.3` 输出能力：仅满足 `C4`（非能力项/无效组合判定）。
  - `R4.4` 边界绑定：受 `N/P/S/O/A/T/SF` 约束。

## 5. 验证（Verification）

- `V1` 推导一致性：每个 `B*` 必须能由至少一个 `C*` 与一个参数边界项（如 `N/P/S/O/A/T/SF`）推导得到。
- `V2` 规则一致性：每个 `R*` 必须明确“参与基/组合方式/输出能力/边界绑定”四项。
- `V3` 目标覆盖性：`R1~R3` 的输出能力并集必须覆盖 `C1~C3`。
- `V4` 边界符合性：所有有效组合必须满足其绑定的 `N/P/S/O/A/T/SF`。
- `V5` 最小必要性：移除任一 `B*` 后，`V3` 或 `V4` 至少一项失败。
- `V6` 结论表达：逐条输出 `R* -> C* / 参数边界` 的通过或失败结论。
