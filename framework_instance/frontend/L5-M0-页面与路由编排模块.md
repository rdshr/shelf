# 工程实例层：页面路由编排模块:PageRouteOrchestration

@framework_instance

- source_standard: framework/frontend/L5-M0-页面与路由编排模块.md
- sync_rule: 当标准分层文件变更时，必须同步更新本实例分层文件。

## 1. 实例化能力声明（Capability Statement）

- `C1` 页面骨架组装能力（对应标准 `C1`）：将 `packages/ui` 组件与布局容器组装为可复用页面骨架。
- `C2` 路由访问控制能力（对应标准 `C2`）：统一鉴权、重定向、导航守卫与兜底路由策略。
- `C3` 基础页面复用能力（对应标准 `C3`）：沉淀登录类与信息展示类基础页面模板并提供扩展插槽。
- `C4` 非能力项（对应标准 `C4`）：本层不负责原子组件实现细节与后端认证协议实现。

## 2. 实例化边界定义（Boundary / 参数）

- `LAYOUT`（对应标准 `LAYOUT`）：固定页面槽位职责（`Header/Sidebar/Main/Footer`），业务组件仅挂载在 `Main` 区域。
- `ROUTE`（对应标准 `ROUTE`）：路由命名、层级和目录一一对应；嵌套深度受限，避免不可维护路由树。
- `GUARD`（对应标准 `GUARD`）：受限路由必须声明鉴权策略、重定向目标与未授权 fallback。
- `PAGEBASE`（对应标准 `PAGEBASE`）：基础模板至少包含 `login/register/forgot-password`，并提供统一的表单与反馈插槽。
- `NAV`（对应标准 `NAV`）：关键页面必须可经主导航或直达 URL 访问，且导航状态与当前路由一致。
- `SEO`（对应标准 `SEO`）：页面必须设置 title/description/canonical 等元信息并与路由绑定。
- `PERF`（对应标准 `PERF`）：关键页面首屏性能满足预算，路由级懒加载和资源预取策略可配置。

## 3. 实例化最小可行基（Minimum Viable Bases）

- `B1` 布局路由基（对应标准 `B1`）：布局骨架组件、路由配置树、导航映射表与默认跳转规则。
- `B2` 页面模板基（对应标准 `B2`）：登录/注册/找回密码模板、通用信息页模板、SEO 元信息注入器。
- `B3` 访问治理基（对应标准 `B3`）：路由守卫中间件、权限校验、性能预算检查与慢页面告警策略。

## 4. 实例化基组合原则（Base Combination Principles）

- `R1` 页面骨架编排（对应标准 `R1`）
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：按布局槽位和路由层级生成页面入口、导航关系与默认跳转。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`LAYOUT/ROUTE/NAV`。
- `R2` 基础页面拼装（对应标准 `R2`）
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在页面骨架中挂接基础模板，统一页面结构、元信息和入口路径。
  - `R2.3` 输出能力：`C1 + C3`。
  - `R2.4` 边界绑定：`PAGEBASE/LAYOUT/SEO/ROUTE`。
- `R3` 登录类页面落地（对应标准 `R3`）
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：在登录、注册、找回密码页面接入守卫策略与性能治理，形成可上线页面集。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`PAGEBASE/ROUTE/GUARD/PERF/NAV`。
- `R4` 禁止组合（对应标准 `R4`）
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：未配置守卫直接暴露受限路由、模板与路由不匹配、或超出性能预算的组合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`LAYOUT/ROUTE/GUARD/PAGEBASE/NAV/SEO/PERF`。

## 5. 实例化验证（Verification）

- `V1` 骨架一致性（对应标准 `V1`）：`R1` 产出路由结构必须满足 `LAYOUT/ROUTE/NAV` 约束。
- `V2` 模板覆盖性（对应标准 `V2`）：`PAGEBASE` 必须覆盖登录类基础页面模板。
- `V3` 访问正确性（对应标准 `V3`）：`GUARD` 必须覆盖所有受限页面入口并有明确重定向路径。
- `V4` 性能符合性（对应标准 `V4`）：关键页面首屏表现满足 `PERF` 预算阈值。
- `V5` 最小必要性（对应标准 `V5`）：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达（对应标准 `V6`）：输出 `R1~R4 -> C* / 边界` 的通过或失败清单及验证证据。
