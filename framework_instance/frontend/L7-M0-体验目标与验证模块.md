## 1. 实例化能力声明（Capability Statement）

- `C1` 体验目标建模能力（对应标准 `C1`）：将体验目标转为可量化指标体系并绑定采样口径。
- `C2` 多维验证能力（对应标准 `C2`）：联合验证性能、稳定性、可用性、一致性与流程转化。
- `C3` 反馈优化能力（对应标准 `C3`）：将异常告警、回归识别和修复复盘接入持续优化闭环。
- `C4` 非能力项（对应标准 `C4`）：本层不实现底层组件和业务规则逻辑。

## 2. 实例化边界定义（Boundary / 参数）

- `SPEED`（对应标准 `SPEED`）：定义首屏、关键交互、流程完成耗时阈值，并区分桌面与移动端基线。
- `STABLE`（对应标准 `STABLE`）：定义 JS Error 率、崩溃率、请求失败率、重试率等稳定性指标。
- `A11Y`（对应标准 `A11Y`）：定义键盘可达、语义标签、焦点管理与读屏可用性阈值。
- `CONSIST`（对应标准 `CONSIST`）：定义同类交互反馈、状态提示、视觉语义一致性检查项。
- `FUNNEL`（对应标准 `FUNNEL`）：定义登录等关键流程漏斗节点（进入、提交、成功、失败、回退）。
- `SAT`（对应标准 `SAT`）：定义任务完成率、关键流程中断率和用户反馈评分等满意代理指标。
- `ALERT`（对应标准 `ALERT`）：定义指标越界告警、升级路径、处理时限和责任归属。

## 3. 实例化最小可行基（Minimum Viable Bases）

- `B1` 指标模型基（对应标准 `B1`）：指标字典、采样口径、分位统计规则和跨环境对齐策略。
- `B2` 验证编排基（对应标准 `B2`）：自动化验证任务（性能/可访问性/流程回归）与漏斗追踪编排。
- `B3` 告警回归基（对应标准 `B3`）：告警规则、回归识别规则、处置记录模板与优化追踪看板。

## 4. 实例化基组合原则（Base Combination Principles）

- `R1` 指标建模组合（对应标准 `R1`）
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：统一指标定义、采样边界和统计口径，再接入验证执行链路。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`SPEED/STABLE/CONSIST`。
- `R2` 验证联动组合（对应标准 `R2`）
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：以统一指标驱动自动化验证，并对关键流程漏斗进行持续追踪。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`FUNNEL/A11Y/SPEED/CONSIST`。
- `R3` 反馈闭环组合（对应标准 `R3`）
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：将验证结果接入告警与回归治理，形成“发现-定位-修复-复验”闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`ALERT/SAT/STABLE/FUNNEL`。
- `R4` 禁止组合（对应标准 `R4`）
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失统一指标口径、验证路径不可复现、或无告警回路的方案均判定无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SPEED/STABLE/A11Y/CONSIST/FUNNEL/SAT/ALERT`。

## 5. 实例化验证（Verification）

- `V1` 指标一致性（对应标准 `V1`）：同一指标在开发、预发、生产环境口径一致且可比对。
- `V2` 采样有效性（对应标准 `V2`）：样本覆盖关键页面和关键流程节点，避免只采集局部流量。
- `V3` 闭环完整性（对应标准 `V3`）：每个异常指标必须有告警、处置、复验记录。
- `V4` 边界符合性（对应标准 `V4`）：验证结果满足 `SPEED/STABLE/A11Y` 阈值。
- `V5` 最小必要性（对应标准 `V5`）：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达（对应标准 `V6`）：输出 `R1~R4 -> C* / 边界` 的通过或失败清单及验证证据。
