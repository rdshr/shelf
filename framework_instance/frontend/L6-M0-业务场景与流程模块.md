# 工程实例层：业务流程场景模块:BusinessFlowScenarios

@framework_instance

- source_standard: framework/frontend/L6-M0-业务场景与流程模块.md

## 1. 实例化能力声明（Capability Statement）

- `C1` 场景流程编排能力（对应标准 `C1`）：将页面模板串联为完整业务流程（登录、注册、找回密码、登出）。
- `C2` 跨页状态一致能力（对应标准 `C2`）：统一会话、鉴权与中间态在跨页跳转中的一致性。
- `C3` 过程观测与回滚能力（对应标准 `C3`）：关键节点具备监控、告警、开关与快速回退能力。
- `C4` 非能力项（对应标准 `C4`）：本层不定义组件样式细节与底层路由实现细节。

## 2. 实例化边界定义（Boundary / 参数）

- `SCENE`（对应标准 `SCENE`）：场景清单至少包含 `login/register/forgot-password/logout`。
- `STEP`（对应标准 `STEP`）：每个场景定义关键步骤（进入、校验、提交、结果、回退）并可追踪。
- `AUTH`（对应标准 `AUTH`）：统一鉴权前置条件、令牌校验逻辑与失效处理策略。
- `SESSION`（对应标准 `SESSION`）：统一会话生命周期、续期机制、失效清理和跨页同步策略。
- `FALLBACK`（对应标准 `FALLBACK`）：超时、鉴权失败、提交失败等异常场景必须定义降级与重试入口。
- `METRIC`（对应标准 `METRIC`）：监控登录转化率、失败率、耗时分位、回退次数等关键指标。
- `ROLLBACK`（对应标准 `ROLLBACK`）：流程改动必须支持特性开关关闭与快速版本回滚。
- `LOGINDOMAINCONTRACT`（对应标准 `LOGINDOMAINCONTRACT`）：登录流程细化由 `framework_instance/frontend_login/L2-M0-登录业务流程场景模块.md` 承接，并与 `AUTH/SESSION/FALLBACK/METRIC` 语义一致。

## 3. 实例化最小可行基（Minimum Viable Bases）

- `B1` 场景骨架基（对应标准 `B1`）：基于页面路由模板定义主链路场景骨架、步骤节点与鉴权检查点。
- `B2` 跨页状态基（对应标准 `B2`）：会话容器、鉴权状态同步、异常回退入口与跨页状态桥接机制。
- `B3` 流程观测基（对应标准 `B3`）：关键节点埋点、告警阈值、发布开关与回滚执行策略。

## 4. 实例化基组合原则（Base Combination Principles）

- `R1` 主链路场景编排（对应标准 `R1`）
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：按业务顺序连接入口页、验证页、结果页，并绑定会话与鉴权状态流转。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`SCENE/STEP/AUTH/SESSION`。
- `R2` 异常链路编排（对应标准 `R2`）
  - `R2.1` 参与基：`B2 + B3`。
  - `R2.2` 组合方式：为超时、鉴权失败、提交失败配置重试、降级、告警与回退策略。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`FALLBACK/METRIC/ROLLBACK/SESSION`。
- `R3` 登录场景闭环编排（对应标准 `R3`）
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：合并主链路与异常链路，形成可监控、可回退的登录闭环流程。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`SCENE/STEP/AUTH/SESSION/FALLBACK/METRIC`。
- `R4` 禁止组合（对应标准 `R4`）
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失会话一致性、缺少失败回退定义、或无关键指标监控的流程均判定无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`SCENE/STEP/AUTH/SESSION/FALLBACK/METRIC/ROLLBACK`。

## 5. 实例化验证（Verification）

- `V1` 场景覆盖性（对应标准 `V1`）：`SCENE` 中每个场景都必须具备主链路定义与步骤清单。
- `V2` 状态一致性（对应标准 `V2`）：主链路与异常链路的会话状态可追踪且一致。
- `V3` 回退有效性（对应标准 `V3`）：`FALLBACK` 中每类异常必须具备可执行回退路径。
- `V4` 观测符合性（对应标准 `V4`）：关键节点必须输出 `METRIC` 并满足阈值约束。
- `V5` 最小必要性（对应标准 `V5`）：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达（对应标准 `V6`）：输出 `R1~R4 -> C* / 边界` 的通过或失败清单及验证证据。
