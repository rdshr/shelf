# 工程实例层：组件原子模块:ComponentAtoms

@framework_instance

- source_standard: framework/frontend/L3-M0-组件原子模块.md
- sync_rule: 当标准分层文件变更时，必须同步更新本实例分层文件。

## 1. 实例化能力声明（Capability Statement）

- `C1` 原子抽象能力（对应标准 `C1`）：将基础 UI 组件沉淀到 `packages/ui`，形成稳定可复用原子单元。
- `C2` 交互契约能力（对应标准 `C2`）：统一事件、状态、键盘行为和可访问性语义，避免组件行为漂移。
- `C3` 扩展发布能力（对应标准 `C3`）：支持主题扩展、文档化、测试门禁与版本化发布。
- `C4` 非能力项（对应标准 `C4`）：本层不负责路由策略、页面流程与后端接口编排。

## 2. 实例化边界定义（Boundary / 参数）

- `ATOM`（对应标准 `ATOM`）：原子清单至少包含 `Button/Input/Select/Checkbox`，并补齐 `FormField/Dialog/Toast`。
- `API`（对应标准 `API`）：组件对外接口遵循稳定等级；破坏性变更必须主版本升级并提供迁移说明。
- `A11Y`（对应标准 `A11Y`）：支持键盘可达、语义标签、aria 属性与状态提示，焦点顺序可预测。
- `THEME`（对应标准 `THEME`）：组件样式仅消费语义 token，颜色/间距/字号/圆角通过主题映射配置。
- `TEST`（对应标准 `TEST`）：关键交互路径必须覆盖单测与回归测试（点击、输入、禁用、错误、加载）。
- `I18N`（对应标准 `I18N`）：默认文案、提示语、占位符可替换，不在组件内部硬编码业务文案。
- `DOC`（对应标准 `DOC`）：每个原子组件必须有参数表、交互示例、无障碍说明与升级备注。

## 3. 实例化最小可行基（Minimum Viable Bases）

- `B1` 交互原子基（对应标准 `B1`）：`packages/ui/src/*` + 事件契约类型 + A11Y 行为规范。
- `B2` 展示原子基（对应标准 `B2`）：主题 token 映射层 + 国际化可替换文案入口 + 样式状态规范。
- `B3` 发布封装基（对应标准 `B3`）：组件文档站、测试流水线、变更日志与版本发布策略。

## 4. 实例化基组合原则（Base Combination Principles）

- `R1` 原子目录构建（对应标准 `R1`）
  - `R1.1` 参与基：`B1 + B2`。
  - `R1.2` 组合方式：先固定组件清单，再统一交互语义、主题映射和状态反馈表达。
  - `R1.3` 输出能力：`C1 + C2`。
  - `R1.4` 边界绑定：`ATOM/API/A11Y/THEME`。
- `R2` 规范封装发布（对应标准 `R2`）
  - `R2.1` 参与基：`B1 + B2 + B3`。
  - `R2.2` 组合方式：将组件接入文档与测试流水线，并按版本策略对外发布到 `packages/ui`。
  - `R2.3` 输出能力：`C2 + C3`。
  - `R2.4` 边界绑定：`API/TEST/DOC/I18N/THEME`。
- `R3` 页面可组装契约输出（对应标准 `R3`）
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：输出统一 props 约定与事件契约，供页面层直接编排登录等页面场景。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`ATOM/API/A11Y/TEST/DOC`。
- `R4` 禁止组合（对应标准 `R4`）
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：缺失核心原子、缺少测试/文档发布链路、与主题 token 脱节的组件集合均无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`ATOM/API/A11Y/THEME/TEST/I18N/DOC`。

## 5. 实例化验证（Verification）

- `V1` 清单覆盖（对应标准 `V1`）：`ATOM` 必须覆盖核心输入、选择、触发与反馈类组件。
- `V2` 契约一致（对应标准 `V2`）：`R1~R2` 产出的 props、事件与状态语义在组件间保持一致。
- `V3` 质量约束（对应标准 `V3`）：`TEST` 与 `DOC` 达到门禁阈值，未达标不得发布。
- `V4` 边界符合（对应标准 `V4`）：所有有效组合需满足 `A11Y/THEME/I18N` 约束。
- `V5` 最小必要性（对应标准 `V5`）：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达（对应标准 `V6`）：输出 `R1~R4 -> C* / 边界` 的通过或失败清单及验证记录。
