# 工程实例层：设计令牌主题模块:DesignTokenTheme

@framework_instance

- source_standard: framework/frontend/L1-M0-设计令牌与主题模块.md
- sync_rule: 当标准分层文件变更时，必须同步更新本实例分层文件。

## 1. 实例化能力声明（Capability Statement）

- `C1` 语义抽象能力（对应标准 `C1`）：将颜色、字体、间距、圆角等统一定义为语义令牌，不在组件中写死实例值。
- `C2` 主题映射能力（对应标准 `C2`）：支持同一语义令牌在多主题（brand）和多模式（light/dark/high-contrast）下映射。
- `C3` 扩展治理能力（对应标准 `C3`）：支持令牌版本演进、覆盖策略和可访问性门禁，保证扩展不破坏已有语义。
- `C4` 非能力项（对应标准 `C4`）：本层不编排业务页面布局与流程。

## 2. 实例化边界定义（Boundary / 参数）

- `TOKEN`（对应标准 `TOKEN`）：令牌按语义域分层，目录为 `packages/tokens/src/{color,typography,space,radius,shadow,motion}`。
- `THEME`（对应标准 `THEME`）：主题至少包含 `default` 与 `brandA`，并通过 `themeId` 统一注册。
- `MODE`（对应标准 `MODE`）：模式至少包含 `light/dark/high-contrast`，由 `data-theme` 与 `data-mode` 双维度切换。
- `SCALE`（对应标准 `SCALE`）：字号、间距、圆角、阴影采用统一尺度（如 `xs/sm/md/lg/xl`）并提供 token 索引表。
- `SEM`（对应标准 `SEM`）：语义令牌命名遵循 `semantic.<domain>.<intent>.<state>`，禁止直接引用原始值。
- `OVRD`（对应标准 `OVRD`）：覆盖优先级为 `component local -> feature -> app theme -> base token`，缺失时按上层回退。
- `A11Y`（对应标准 `A11Y`）：文本与背景对比度满足 WCAG AA（普通文本>=4.5:1，大文本>=3:1）。

## 3. 实例化最小可行基（Minimum Viable Bases）

- `B1` 语义令牌基（对应标准 `B1`）：`tokens.semantic.json` + `tokens.scale.json` + 语义命名校验脚本。
- `B2` 主题映射基（对应标准 `B2`）：`themes/{default,brandA}/{light,dark,high-contrast}.json` + 运行时主题解析器。
- `B3` 覆盖约束基（对应标准 `B3`）：覆盖优先级规则、fallback 机制、A11Y 对比度检测任务。

## 4. 实例化基组合原则（Base Combination Principles）

- `R1` 语义抽象组合（对应标准 `R1`）
  - `R1.1` 参与基：`B1`。
  - `R1.2` 组合方式：先固化语义 token 与尺度，再由组件层只消费语义 token。
  - `R1.3` 输出能力：`C1`。
  - `R1.4` 边界绑定：`TOKEN/SCALE/SEM`。
- `R2` 主题映射组合（对应标准 `R2`）
  - `R2.1` 参与基：`B1 + B2`。
  - `R2.2` 组合方式：在语义 token 上挂接主题与模式映射，确保语义稳定、实例可替换。
  - `R2.3` 输出能力：`C1 + C2`。
  - `R2.4` 边界绑定：`THEME/MODE/SEM/TOKEN`。
- `R3` 治理闭环组合（对应标准 `R3`）
  - `R3.1` 参与基：`B1 + B2 + B3`。
  - `R3.2` 组合方式：通过覆盖优先级、fallback 与 A11Y 检测构建稳定可扩展闭环。
  - `R3.3` 输出能力：`C1 + C2 + C3`。
  - `R3.4` 边界绑定：`OVRD/A11Y/THEME/MODE/SEM`。
- `R4` 禁止组合（对应标准 `R4`）
  - `R4.1` 参与基：`B1 + B2 + B3`。
  - `R4.2` 组合方式：在组件中硬编码颜色、绕过语义层直接绑定主题值、跳过 A11Y 检测均判定无效。
  - `R4.3` 输出能力：`C4`。
  - `R4.4` 边界绑定：`TOKEN/THEME/MODE/SCALE/SEM/OVRD/A11Y`。

## 5. 实例化验证（Verification）

- `V1` 抽象完整性（对应标准 `V1`）：扫描组件样式，禁止出现硬编码颜色/间距等实例值（允许白名单除外）。
- `V2` 映射一致性（对应标准 `V2`）：同一语义 token 在不同 theme/mode 的映射必须完整且可解释。
- `V3` 治理有效性（对应标准 `V3`）：覆盖优先级链与 fallback 路径必须可追踪并可复现。
- `V4` 约束符合性（对应标准 `V4`）：A11Y 对比度检查和 mode 切换回归测试必须通过。
- `V5` 最小必要性（对应标准 `V5`）：移除任一 `B*` 后，`V2~V4` 至少一项失败。
- `V6` 结论表达（对应标准 `V6`）：输出 `R1~R4 -> C* / 边界` 通过或失败清单及对应校验记录。
